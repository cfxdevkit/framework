# cfx-llm

The **cfx-llm** module provides a toolkit for integrating Large Language Model (LLM) capabilities into development workflows. It is structured as a monorepo package (`packages/llm-tools`) containing workers, agents, and utility suites designed for code analysis, automation, and generation.

## Architecture

The module is organized around a worker-based architecture, where specific tasks are encapsulated in isolated worker files. The `src` directory handles entry points and orchestration, while `workers` contains the domain-specific logic.

## Directory Structure

| Path | Description |
| :--- | :--- |
| `src/bin.ts` | CLI entry point for executing tools. |
| `src/run.test.ts` | Integration tests for the main execution flow. |
| `workers/` | Core logic modules executed as background tasks or commands. |
| `workers/agents/` | Runtime environment for LLM agents. |
| `workers/lemonade/` | Suite of development automation tools. |
| `workers/code-hotspots.ts` | Module for identifying and analyzing code performance hotspots. |
| `artifacts/llm/reports/` | Generated output reports (JSON/Markdown). |

## Key Components

### Workers
The `workers` directory contains the primary logic units. Each file represents a distinct capability or service.

*   **`workers/llm-agents.ts`**: Orchestrates LLM agent interactions.
*   **`workers/code-hotspots.ts`**: Handles static analysis or profiling data to identify critical code sections.
*   **`workers/agents/runtime/`**: Manages the runtime environment for agents.
    *   `constants.ts`: Configuration constants for agent execution.
    *   `index.ts`: Runtime initialization and lifecycle management.

### Lemonade Suite
The `lemonade` directory houses a specific set of development tools, likely focused on code lifecycle management.

*   **Commit Management**
    *   `workers/lemonade/commit/index.ts`: Logic for handling commit operations.
    *   `workers/lemonade/commit/index.test.ts`: Unit tests for commit functionality.
*   **Completion**
    *   `workers/lemonade/completion/index.ts`: Core completion service.
    *   `workers/lemonade/completion/complete-utils.ts`: Utility functions for completion logic.
    *   `workers/lemonade/completion/pi.ts`: Specific implementation or plugin for completion.
*   **Documentation & Testing**
    *   `workers/lemonade/docs/index.ts`: Documentation generation or retrieval.
    *   `workers/lemonade/tests/index.ts`: Test generation or management utilities.
*   **Shared**
    *   `workers/lemonade/shared/index.ts`: Common utilities shared across the Lemonade suite.
*   **Commands**
    *   `workers/lemonade/commands.test.ts`: Tests for command-line interactions within the suite.

### Artifacts
Generated reports and data outputs are stored in `artifacts/llm/reports/`.

*   `code-hotspots.json`: Structured data output from hotspot analysis.
*   `code-hotspots.md`: Human-readable report generated from hotspot data.

## Configuration

The module uses standard TypeScript and Vite configurations for building and bundling.

*   **`tsconfig.json`**: TypeScript compiler options.
*   **`vite.config.ts`**: Build configuration for Vite.
*   **`moon.yml`**: Task runner configuration (likely for `moon` build system).
*   **`package.json`**: Dependencies and scripts.
*   **`pnpm-workspace.template.yaml`**: Workspace configuration template for pnpm.

## Testing

Testing is distributed across the module structure:

*   **Unit Tests**: Located alongside source files (e.g., `*.test.ts`).
*   **Integration Tests**: Located in `src/run.test.ts`.
*   **Worker Tests**: Specific tests exist for Lemonade commands and commit logic.

## Usage

The module is designed to be executed via the CLI entry point defined in `src/bin.ts`. Specific workers are invoked based on the command or task context provided at runtime. Configuration is managed through the root `package.json` and `moon.yml` files.
