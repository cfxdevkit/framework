# Other — cfx-solidity

# cfx-solidity

The `cfx-solidity` workspace provides the Solidity pipeline for the Conflux development kit. It handles standard ABIs, contract bindings, and compiler tooling. This module was carved out from `cfx-core` to operate independently, allowing any layer of the stack (including `@cfxdevkit/cdk`) to consume ABI definitions without pulling in contract-execution machinery.

## Package Structure

The workspace is organized into distinct packages based on responsibility:

| Package | Purpose | Dependencies |
| :--- | :--- | :--- |
| `@cfxdevkit/abis` | Standard EVM ABI shapes (ERC-20/721/1155, Multicall3). Re-exported from `viem`. | `viem` |
| `@cfxdevkit/contracts` | Contract bindings, read/write/deploy surface, and bridge utilities. | `@cfxdevkit/abis`, `@cfxdevkit/cdk`, `viem` |
| `@cfxdevkit/compiler` | Solidity compilation pipeline. | *(Internal)* |
| `@cfxdevkit/contracts-extract` | Contract artifact extraction. | *(Internal)* |

## ABI Definitions (`@cfxdevkit/abis`)

The `@cfxdevkit/abis` package is the leaf package for ABI definitions. It ensures framework-stable aliases for standard EVM interfaces sourced from `viem`.

### Key Exports

*   **Standard ABIs:** `ERC20_ABI`, `ERC721_ABI`, `ERC1155_ABI`, `MULTICALL3_ABI`, `ERC2612_ABI`, `ERC4626_ABI`.
*   **CamelCase Aliases:** `erc20Abi`, `erc721Abi`, `erc1155Abi`, etc. (Point to uppercase constants).
*   **Constants:** `MULTICALL3_ADDRESS` (`0xcA11bde05977b3631167028862bE2a173976CA11`).

### Usage

```typescript
import { ERC20_ABI, erc20Abi } from '@cfxdevkit/abis';

// Prefer direct import for tree-shaking
import { ERC20_ABI } from '@cfxdevkit/abis';
```

## Contract Interaction (`@cfxdevkit/contracts`)

The `@cfxdevkit/contracts` package provides the execution layer. It abstracts the differences between Conflux **Core Space** (base32 addresses, `cfx_*` RPC) and **eSpace** (0x addresses, `eth_*` RPC).

### Sub-Paths

Import from sub-paths to maintain tree-shaking efficiency:

*   `@cfxdevkit/contracts/abis`: Back-compat re-exports of ABI shapes.
*   `@cfxdevkit/contracts/read`: Read-only contract calls.
*   `@cfxdevkit/contracts/write`: Transaction preparation and signing.
*   `@cfxdevkit/contracts/deploy`: Contract deployment logic.
*   `@cfxdevkit/contracts/bridge`: Cross-space transfer utilities.
*   `@cfxdevkit/contracts/erc20`: Typed helpers for ERC-20 tokens.
*   `@cfxdevkit/contracts/errors`: Custom error classes.

### Read Operations

The `readContract` function automatically detects the client family to select the correct RPC method.

*   **eSpace:** Uses `eth_call`.
*   **Core:** Uses `cfx_call` with `epochTag`.

```typescript
import { readContract } from '@cfxdevkit/contracts/read';
import { ERC20_ABI } from '@cfxdevkit/contracts/abis';

const balance = await readContract({
  client,
  address: '0x...', // eSpace) or 'cfxtest:...' (Core)
  abi: ERC20_ABI,
  functionName: 'balanceOf',
  args: ['0x...'],
});
```

**Validation:**
*   Rejects `0x` addresses on Core Space clients.
*   Rejects base32 addresses on eSpace clients.
*   Wraps decode errors in `ContractsError`.

### Write Operations

Write operations handle nonce management, gas estimation, and transaction signing.

*   **`prepareWrite`**: Encodes call data and returns a `SignableTx` object.
*   **`sendWrite`**: Fills missing fields (nonce, gas, fees), signs via `signer`, and broadcasts.

```typescript
import { sendWrite } from '@cfxdevkit/contracts/write';

const result = await sendWrite({
  client,
  signer,
  address: TOKEN_ADDRESS,
  abi: ERC20_ABI,
  functionName: 'transfer',
  args: [RECIPIENT, 100n],
  waitForReceipt: true, // Polls for receipt if true
});
```

**Core Space Specifics:**
*   Uses `cfx_sendRawTransaction`.
*   Requires `coreAddress` on the signer.
*   Calculates `gas`, `storageLimit`, and `epochHeight` based on `cfx_estimateGasAndCollateral`.
*   Applies a 25% safety buffer to gas limits.

### Deployment

The `deployContract` function supports deployment on both spaces.

*   **eSpace:** Broadcasts via `eth_sendRawTransaction`. Extracts `contractAddress` from receipt.
*   **Core:** Broadcasts via `cfx_sendRawTransaction`. Extracts `contractCreated` from receipt.

```typescript
import { deployContract } from '@cfxdevkit/contracts/deploy';

const result = await deployContract({
  client,
  signer,
  abi: ERC20_ABI,
  bytecode: '0x60006000',
  waitForReceipt: true,
});
// result.address contains the deployed contract address
```

### Cross-Space Bridge

The `bridge` namespace handles interactions with the `CrossSpaceCall` contract (`0x0888000000000000000000000000000000000006`).

*   **`transferToEspace`**: Encodes `transferEVM(bytes20)` and broadcasts from Core to eSpace.
*   **`callEspace`**: Encodes `callEVM(bytes20, bytes)` to invoke eSpace contracts from Core.
*   **`withdrawFromMapped`**: Encodes `withdrawFromMapped(uint256)` to move funds from eSpace back to Core.
*   **`mappedEspaceAddress`**: Derives the eSpace address from a Core hex address using `keccak256(coreHex)[12:32]`.
*   **`getMappedBalance`**: Queries the balance of a mapped address on Core.

```typescript
import { bridge } from '@cfxdevkit/contracts';

// Transfer from Core to eSpace
await bridge.transferToEspace({
  client,
  signer,
  to: '0x000000000000000000000000000000000000c0de',
  value: 1_000_000_000_000_000_000n,
});
```

## Development & Build

### Tooling

*   **Build:** `vite` with `vite-plugin-dts` for TypeScript declaration generation.
*   **Test:** `vitest` with `node` environment.
*   **Lint:** `@biomejs/biome`.
*   **Config:** `moon` for workspace project configuration.

### Scripts

Run the following from the package root:

```bash
pnpm build          # Compile to dist/
pnpm test           # Run vitest
pnpm typecheck      # Run tsc --noEmit
pnpm lint           # Run biome check
pnpm clean          # Remove dist, coverage, and cache
```

### Testing Utilities

The package exports mock utilities for testing contract interactions:

*   `makeMockClient`: Creates a mock RPC client with recorded calls.
*   `makeMockSigner`: Creates a mock signer that records transaction history.
*   `getRecordedCalls`: Retrieves the list of RPC methods invoked during a test.

```typescript
import { makeMockClient, makeMockSigner } from '@cfxdevkit/contracts/test/mocks';

const client = makeMockClient({ rpc: { eth_call: '0x...' } });
const signer = makeMockSigner();
```

## Error Handling

The module defines specific error codes within `ContractsError`:

*   `contracts/invalid-argument`: Thrown when address formats mismatch the client family (e.g., 0x address on Core).
*   `contracts/reverted`: Thrown when a transaction receipt indicates failure.
*   `contracts/timeout`: Thrown when `waitForReceipt` exceeds `receiptTimeoutMs`.

## Contributing

1.  Ensure `@cfxdevkit/abis` remains dependency-free of `cfxdevkit` logic.
2.  When adding new contract bindings, prefer sub-path exports (e.g., `./erc20`) over the main barrel export.
3.  Update `moon.yml` if adding new packages to the workspace.
4.  Verify dual-space compatibility (Core vs eSpace) for all new RPC interactions.
