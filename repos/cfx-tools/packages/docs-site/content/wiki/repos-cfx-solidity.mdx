# repos — cfx-solidity

# `cfx-solidity` Module Documentation

The `cfx-solidity` module provides a unified Solidity pipeline for the Conflux framework, separating standard ABI definitions from contract execution machinery. It delivers framework-stable ABI shapes, typed read/write/deploy surfaces, and cross-space bridge helpers that operate across both Conflux Core Space and eSpace.

## Package Architecture

The module is structured as a pnpm workspace containing two primary packages:

| Package | Scope | Dependencies | Purpose |
|---------|-------|--------------|---------|
| `@cfxdevkit/abis` | Leaf | `viem` only | Standard EVM & Conflux DEX ABI constants. Zero framework dependencies. |
| `@cfxdevkit/contracts` | Execution Surface | `@cfxdevkit/abis`, `@cfxdevkit/cdk`, `viem` | Typed contract reads, writes, deployments, bridge helpers, and error handling. |

Both packages use Vite with `vite-plugin-dts` for library bundling and Vitest for testing. Sub-paths are exported separately to enable precise tree-shaking.

## Core Subsystems

### ABI Definitions (`@cfxdevkit/abis`)
Exports canonical EVM interface shapes as `as const` arrays, re-exported from `viem` under framework-stable aliases. Provides both uppercase constants and camelCase variants:
- `ERC20_ABI`, `ERC20_EXTENDED_ABI`, `ERC2612_ABI`
- `ERC721_ABI`, `ERC721_ENUMERABLE_ABI`, `ERC2981_ABI`, `ERC721_EXTENDED_ABI`
- `ERC1155_ABI`
- `ERC4626_ABI`
- `MULTICALL3_ABI`, `MULTICALL3_ADDRESS`
- Swappi V2 interfaces: `SWAPPI_FACTORY_ABI`, `SWAPPI_PAIR_ABI`, `SWAPPI_ROUTER_ABI` + chain-specific addresses (`SWAPPI_FACTORY_ADDRESS`, `SWAPPI_ROUTER_ADDRESS`, `WCFX_ADDRESS`)

### Contract Reads (`@cfxdevkit/contracts/read`)
The `readContract<TAbi, TName>()` function encodes view/pure calls via `encodeFunctionData`, dispatches to the chain-appropriate RPC, and decodes results using `decodeFunctionResult`.
- **eSpace**: Routes to `eth_call` with `blockTag` support. Validates `0x`-prefixed addresses.
- **Core Space**: Routes to `cfx_call` with `epochTag` support. Validates base32 addresses (`cfx:…` / `cfxtest:…`).
- Decoding failures wrap in `ContractsError` with code `contracts/decode-failure`.

### Contract Writes (`@cfxdevkit/contracts/write`)
Handles state-modifying transactions through `sendWrite<TAbi, TName>()` and `prepareWrite()`.
- **eSpace**: Uses `eth_getTransactionCount`, `estimateGas`, and `getBlock` for base fee calculation. Builds EIP-1559 transactions (`maxFeePerGas`, `maxPriorityFeePerGas`).
- **Core Space**: Uses `cfx_getNextNonce`, `cfx_estimateGasAndCollateral`, `cfx_gasPrice`, and `cfx_epochNumber`. Builds CIP-2930/CIP-1559 transactions with explicit `gas`, `storageLimit`, `gasPrice`, and `epochHeight`.
- Signs via `Signer.signTransaction()` and broadcasts via `eth_sendRawTransaction` or `cfx_sendRawTransaction`.
- Optional receipt polling via `waitForReceipt()` with configurable `pollIntervalMs` and `receiptTimeoutMs`. Reverted transactions throw `ContractsError` with code `contracts/reverted`.

### Contract Deployment (`@cfxdevkit/contracts/deploy`)
The `deployContract<TAbi>()` entry point routes to chain-specific deployers:
- `deployEspaceContract()`: Encodes bytecode with `encodeDeployData`, estimates gas, signs, and broadcasts. Extracts `contractAddress` from successful receipts.
- `deployCoreContract()`: Requires `signer.account.coreAddress`. Encodes bytecode, estimates gas/collateral, signs with Core-specific fields (`coreType`, `storageLimit`, `epochHeight`), and extracts `contractCreated` from receipts.
- Both implementations return `DeployContractResult` containing `hash`, `request`, `rawTransaction`, and optionally `address`/`receipt`.

### Cross-Space Bridge (`@cfxdevkit/contracts/bridge`)
Abstracts interactions with the CrossSpaceCall contract (`0x0888000000000000000000000000000000000006`). All bridge functions require a Core Space client and validate target addresses.
- `mappedEspaceAddress(coreHexAddress)`: Derives a deterministic eSpace address via `keccak256(coreHex)[12:32]`.
- `transferToEspace(opts)`: Encodes `transferEVM(bytes20)` with value. Converts target to base32 via `crossSpaceCallAddress()` and broadcasts.
- `callEspace(opts)`: Encodes `callEVM(bytes20, bytes)` with optional value.
- `withdrawFromMapped(opts)`: Encodes `withdrawFromMapped(uint256)`.
- `getMappedBalance(input)` / `getMappedNonce(input)`: Reads via `staticCallEVM`/`readContract` pattern, padding the Core hex address to 20 bytes.
- Utility helpers: `uint256Hex()`, `hexToUint256()` for manual ABI encoding.

### ERC20 Helpers & Errors (`@cfxdevkit/contracts/erc20` & `errors`)
- **ERC20**: `erc20` namespace provides thin wrappers over `readContract`/`sendWrite` (`balanceOf`, `symbol`, `decimals`, `totalSupply`, `allowance`, `transfer`, `approve`). Pre-binds `ERC20_ABI` and function names.
- **Errors**: `ContractsError` extends `CfxError` with stable string codes:
  - `contracts/unsupported-family`
  - `contracts/decode-failure`
  - `contracts/receipt-timeout`
  - `contracts/reverted`
  - `contracts/invalid-argument`

## Execution Flow & Routing

All execution surfaces (`read`, `write`, `deploy`) use `client.family` as a runtime discriminator. The routing pattern ensures chain-specific RPC calls, gas models, and address formats are handled transparently:

<Mermaid chart={`flowchart TD
    A[Input: Client + ABI + Function] --> B{client.family}
    B -->|espace| C[eSpace Handler]
    B -->|core| D[Core Handler]
    C --> E["eth_call / eth_sendRawTransaction / eth_estimateGas"]
    D --> F["cfx_call / cfx_sendRawTransaction / cfx_estimateGasAndCollateral"]
    E --> G["Decode Result / Wait Receipt / Return"]
    F --> G
`} />

**Address Validation:**
- eSpace expects `0x`-prefixed 20-byte hex. Invalid formats throw `contracts/invalid-argument`.
- Core Space expects base32 strings. Hex addresses throw on Core routing.
- Bridge module explicitly converts between Core hex and eSpace hex via `mappedEspaceAddress()`, then resolves the CrossSpaceCall contract to a base32 address via `hexToBase32()`.

## Testing & Mock Infrastructure

The contracts package includes comprehensive mock utilities in `src/test/mocks.ts` for offline testing:
- `makeMockClient(opts)`: Creates an in-memory `Client` with configurable `family`, `rpc` method stubs, `gas` estimates, and `receipt` returns. Records all RPC calls via `__calls` array.
- `makeMockSigner(opts)`: Creates a `Signer` that captures signed transactions into a `history` array and returns configurable `rawTx` payloads.
- `getRecordedCalls(client)`: Retrieves the exact RPC methods and parameters invoked during tests.
- Tests verify ABI shapes, function selector encoding, address validation, receipt parsing, and Core/eSpace branching logic without requiring live nodes.

## Build & Workspace Validation

### Build Configuration
- **Vite**: Configured in `vite.config.ts` with library mode. Exports separate ES modules per sub-path (`index`, `abis/index`, `read/index`, `write/index`, `deploy/index`, `erc20/index`, `bridge/index`, `errors/index`).
- **TypeScript**: Extends `@cfxdevkit/tsconfig/lib.json`. Emits declaration files only (`emitDeclarationOnly: true`). Excludes test files and `node_modules`.
- **Vitest**: Configured in `vitest.config.ts` with V8 coverage thresholds (80% lines/functions/statements, 65% branches). Excludes test files and type-only definitions from coverage.

### Workspace Scripts
- `scripts/check-deps.mjs`: Scans `packages/*/package.json`, validates `workspace:*` runtime dependencies match declared packages, and warns about external tooling devDeps. Exits with code 1 on mismatches.
- `scripts/validate-workspace.mjs`: Parses `pnpm-workspace.template.yaml` glob patterns and verifies they match the actual `packages/` directory structure. Ensures workspace consistency before releases.

## Integration Notes for Contributors
- Prefer direct sub-path imports (`import { readContract } from '@cfxdevkit/contracts/read'`) over the barrel export to maintain tree-shaking efficiency.
- ABI constants in `@cfxdevkit/abis` are framework-stable and safe to depend on from any stack layer, including `@cfxdevkit/cdk`.
- Bridge functions strictly require a Core Space client. Passing an eSpace client will throw `contracts/unsupported-family` during `assertCoreClient()`.
- When extending `@cfxdevkit/abis` with new interfaces, add them to the relevant source file (`erc20.ts`, `erc721.ts`, etc.), export both uppercase and camelCase variants, and update `src/index.ts` barrel re-exports.
- Coverage thresholds in `vitest.config.ts` are enforced during CI. Ensure new logic includes unit tests that exercise both eSpace and Core Space branches.
