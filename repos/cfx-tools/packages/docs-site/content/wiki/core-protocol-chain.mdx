# Core Protocol & Chain Module

## Overview

The Core Protocol & Chain module provides a unified TypeScript SDK for interacting with Conflux blockchain networks. It supports both **Core Space** (Conflux-native) and **eSpace** (EVM-compatible) with consistent APIs, type safety, and error handling.

### Key Capabilities

- **Dual-chain support**: Core Space and eSpace clients from a single interface
- **Address encoding**: Base32 ↔ hex conversion for Core Space addresses
- **Chain registry**: Pre-configured mainnet, testnet, and local networks
- **Client abstraction**: Unified RPC interface with transport layer
- **Wallet management**: BIP-39 mnemonic derivation and transaction signing
- **Protocol utilities**: Precompile contracts, ABI bindings, and transaction helpers

## Chain Configuration

The module maintains a static catalog of supported Conflux networks in `chains/index.ts`.

### Chain Families

| Family | Description | Chain IDs |
|--------|-------------|-----------|
| `core` | Conflux Core Space (native) | 1029 (mainnet), 1 (testnet), 2029 (local) |
| `espace` | EVM-compatible space | 1030 (mainnet), 71 (testnet), 2030 (local) |

### Available Chains


## Address Handling

Core Space uses base32 encoding with network prefixes, while eSpace uses standard EVM hex addresses.

### Address Conversion

```typescript
import {
  hexToBase32,
  base32ToHex,
  isBase32Address,
  getCoreAddress,
} from '@cfxdevkit/cdk/address';

// Hex to base32 (requires networkId)
const base32 = hexToBase32('0x1234...', 1029); // cfx:mainnet:1234...

// Base32 to hex
const hex = base32ToHex('cfx:mainnet:1234...'); // 0x1234...

// Validate base32 address
const valid = isBase32Address('cfx:mainnet:1234...'); // true

// Normalize address (canonical prefix + casing)
const normalized = getCoreAddress('cfx:mainnet:1234...');
```

|---------|------------|--------|

## Client API

Clients are created via `createClient()` with a chain configuration and transport.

### Transport Layer

```typescript
import { http, ws, fallback } from '@cfxdevkit/cdk/client';

// HTTP transport
const transport = http({
  url: 'https://main.confluxrpc.com',
  timeoutMs: 30000,
  retries: 3,
});

// WebSocket transport
const wsTransport = ws({
  url: 'wss://main.confluxrpc.com/ws',
  reconnect: true,
});

// Fallback (multiple endpoints)
const fallbackTransport = fallback([http(), wsTransport]);
```

### Client Creation

```typescript
import { createClient, http } from '@cfxdevkit/cdk/client';
import { coreSpaceMainnet } from '@cfxdevkit/cdk/chains';

const client = createClient({
  chain: coreSpaceMainnet,
  transport: http({ url: 'https://main.confluxrpc.com' }),
});
```

## Wallet & Accounts

### Mnemonic Generation & Validation

```typescript
import { generateMnemonic, validateMnemonic } from '@cfxdevkit/cdk/wallet';

// Generate new mnemonic (128-256 bit strength)
const mnemonic = generateMnemonic(128);

// Validate existing mnemonic

```

### Account Derivation

```typescript
import {
  deriveAccount,
  deriveAccounts,
  deriveDualAccount,
  deriveDualAccounts,
} from '@cfxdevkit/cdk/wallet';

// Single account derivation (Core Space default)
const account = deriveAccount({
  mnemonic: '...',
  path: "m/44'/503'/0'/0/0",
  passphrase: 'optional',
});

// Multiple accounts
const accounts = deriveAccounts({
  mnemonic: '...',
  basePath: "m/44'/503'/0'/0",
  count: 10,
});

// Dual address account (Core + eSpace)
const dual = deriveDualAccount({
  mnemonic: '...',
  index: 0,
  accountType: 'standard', // or 'mining'
  coreNetworkId: 1029,
});

// Multiple dual accounts
const duals = deriveDualAccounts({
  mnemonic: '...',
  count: 5,
  startIndex: 0,
  coreNetworkId: 1029,
});
```

### Signer Creation

```typescript
import { signerFromPrivateKey } from '@cfxdevkit/cdk/wallet';

const signer = signerFromPrivateKey('0x...', 1029);

// Sign transaction (auto-detects family)
const signedTx = await signer.signTransaction({
  family: 'core',
  chainId: 1029,
  to: 'cfx:mainnet:...',
  value: 100n,
  gas: 21000n,
  nonce: 0,
  storageLimit: 1024n,
  epochHeight: 1000n,
  coreType: 'cip2930', // 'legacy' | 'cip2930' | 'cip1559'
  gasPrice: 1000000000n, // for legacy/cip2930
  maxFeePerGas: 1000000000n, // for cip1559
  maxPriorityFeePerGas: 100000000n, // for cip1559
});

// Sign message
const sig = await signer.signMessage('Hello');

// Sign typed data (EIP-712)
const typedSig = await signer.signTypedData({
  domain: { name: 'MyApp', chainId: 1029 },
  types: { ... },
  primaryType: 'MyType',
  message: { ... },
});
```

## Token Units

All arithmetic uses `bigint` to avoid floating-point errors.

```typescript
import {
  parseCFX,
  formatCFX,
  parseDrip,
  formatDrip,
  parseGDrip,
  formatGDrip,
  parseUnits,
  formatUnits,
  MAX_UINT256,
  ZERO_ADDRESS,
  stringifyBigInt,
} from '@cfxdevkit/cdk/units';

// CFX (18 decimals)
const wei = parseCFX('1.5'); // 1.5 * 10^18
const cfx = formatCFX(wei); // '1.5'

// Drip (18 decimals, alias for CFX)
const wei = parseDrip('1.5');
const drip = formatDrip(wei);

// Gdrip (9 decimals, gas price unit)
const gdrip = parseGDrip('1000'); // 1000 * 10^9 drip
const str = formatGDrip(gdrip); // '1000'

// Generic token formatting
const token = { symbol: 'USDT', decimals: 6 };
const formatted = formatToken(1000000n, token); // '1.000000 USDT'

// JSON serialization with bigint support
const json = stringifyBigInt({ balance: 100n }); // '{"balance":"100"}'
```

## Error Handling

All errors extend `CfxError` with structured codes and metadata.

```typescript
import { CfxError, RpcError, WalletError, KeystoreError, isCfxError } from '@cfxdevkit/cdk/errors';

// Catch and inspect
try {
  await client.getBalance('invalid');
} catch (err) {
  if (isCfxError(err)) {
    console.log(err.code); // 'core/rpc/get-balance'
    console.log(err.meta); // { address: 'invalid' }
    console.log(err.cause); // underlying error
  }
}

// Error hierarchy
- CfxError (base)
  - RpcError (transport/network failures)
  - ContractError (reverts, decode failures)
  - WalletError (signing, derivation)
  - KeystoreError (vault operations)
```

## Common Patterns

### Sending a Core Space Transaction

```typescript
import { createClient, http, signerFromPrivateKey } from '@cfxdevkit/cdk/client';
import { coreSpaceMainnet } from '@cfxdevkit/cdk/chains';
import { waitForTransactionReceipt } from '@cfxdevkit/protocol';

const client = createClient({
  chain: coreSpaceMainnet,
  transport: http({ url: 'https://main.confluxrpc.com' }),
});

const signer = signerFromPrivateKey(process.env.PRIVATE_KEY!, 1029);

// Build transaction
const tx = {
  family: 'core',
  chainId: 1029,
  to: signer.account.coreAddress!,
  value: parseCFX('1'),
  gas: 21000n,
  nonce: await client.getTransactionCount(signer.account.coreAddress!),
  storageLimit: 1024n,
  epochHeight: await client.getEpochNumber(),
  coreType: 'cip2930',
  gasPrice: await client.getGasPrice(),
};

// Sign and send
const signedTx = await signer.signTransaction(tx);
const hash = await client.sendRawTransaction(signedTx);

// Wait for receipt
const receipt = await waitForTransactionReceipt(client, hash);
```

## Dependencies

- **cive**: Core Space RPC client library
- **viem**: EVM-compatible client library (eSpace)
- **@scure/bip39**: BIP-39 mnemonic handling
- **@scure/bip32**: HD key derivation

## Version Notes

- Core Space uses `cfx_*` RPC methods
- eSpace uses `eth_*` RPC methods
- Addresses in Core Space must be base32-encoded
- Transaction fields differ between families (epochHeight, storageLimit for Core)
