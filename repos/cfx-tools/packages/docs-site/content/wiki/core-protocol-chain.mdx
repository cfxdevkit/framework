# Core Protocol & Chain Module

The `@cfxdevkit/cdk` module provides foundational utilities for interacting with Conflux networks, including address encoding, chain configuration, client creation, and protocol-level helpers.

## Purpose

This module serves as the core protocol and chain abstraction layer for Conflux-based applications. It provides:
- Address encoding/decoding between base32 (Conflux native) and hex (EVM-compatible) formats
- Static chain configuration catalog with lookup utilities
- Client creation factories for both Core Space and eSpace
- Protocol-level helpers for transaction receipt handling, gas estimation, and log collection
- Wallet utilities for account derivation and signing

## Key Components

### Address Codec (`@cfxdevkit/cdk/address`)

Provides base32 address encoding for Conflux Core Space, compatible with network IDs:
- Mainnet: `cfx:`
- Testnet: `cfxtest:`
- Custom networks: `net<id>:`

```ts
import { hexToBase32, base32ToHex, isBase32Address, getCoreAddress } from '@cfxdevkit/cdk/address';

// Convert hex to base32
const base32 = hexToBase32('0x1234567890123456789012345678901234567890', 1029);
// Result: "cfx:ac12345678901234567890123456789012345678"

// Convert base32 to hex
const hex = base32ToHex('cfx:ac12345678901234567890123456789012345678');
// Result: "0x1234567890123456789012345678901234567890"
```

### Chain Configuration (`@cfxdevkit/cdk/chains`)

Static catalog of Conflux chain configurations with:
- Network tier identification (mainnet, testnet, devnet, local)
- Chain family discrimination (core, espace)
- RPC endpoints and explorer URLs
- Native token information

```ts
import { getChain, listChains } from '@cfxdevkit/cdk/chains';

// Get chain by ID or name
const mainnet = getChain(1029); // Core mainnet
const testnet = getChain('espace-testnet'); // eSpace testnet

// List chains by filter
const coreChains = listChains({ family: 'core' });
```

### Client Creation (`@cfxdevkit/cdk/client`)

Factory functions for creating typed clients for Core Space and eSpace:
- `createClient()` - Factory that returns either `CoreSpaceClient` or `EspaceClient`
- Transport layer abstraction with HTTP/WebSocket support

```ts
import { createClient, http } from '@cfxdevkit/cdk/client';
import { coreSpaceMainnet } from '@cfxdevkit/cdk/chains';

const client = createClient({
  chain: coreSpaceMainnet,
  transport: http({ url: 'https://main.confluxrpc.com' }),
});
```

### Protocol Helpers (`@cfxdevkit/protocol`)

Protocol-level utilities for common operations:
- `waitForTransactionReceipt()` - Waits for transaction confirmation
- `estimateTransaction()` - Estimates gas and storage requirements
- `collectLogs()` - Collects Core Space logs with filtering

```ts
import { waitForTransactionReceipt, estimateTransaction } from '@cfxdevkit/protocol';

// Wait for receipt
const receipt = await waitForTransactionReceipt(client, txHash);

// Estimate gas
const estimate = await estimateTransaction(client, txRequest);
```

### Wallet Utilities (`@cfxdevkit/cdk/wallet`)

Account derivation and signing utilities:
- BIP-39 mnemonic generation and validation
- HD path derivation for Core and eSpace accounts
- Private key to address conversion
- Transaction signing for both chain families

```ts
import { deriveAccount, signerFromPrivateKey } from '@cfxdevkit/cdk/wallet';

// Derive account from mnemonic
const account = deriveAccount({
  mnemonic: 'test test test test test test test test test test test junk',
  path: "m/44'/503'/0'/0/0"
});

// Create signer
const signer = signerFromPrivateKey(account.privateKey);
```

## Integration Points

### Client Usage

The client system provides a unified interface for both Core Space and eSpace operations:

```ts
// Core Space operations
const balance = await client.getBalance('cfx:ac12345678901234567890123456789012345678');
const logs = await client.getLogs({ fromEpoch: 1000n, toEpoch: 2000n });

// eSpace operations
const blockNumber = await client.getBlockNumber();
const gas = await client.estimateGas({ to: '0x...', data: '0x...' });
```

### Chain Configuration

Chain configurations are used to initialize clients and provide metadata:

```ts
const chain = getChain('core-mainnet');
const client = createClient({
  chain,
  transport: http({ url: chain.rpc.http[0] })
});
```

### Transport Layer

The transport layer abstracts network communication:

```ts
import { http, ws, fallback } from '@cfxdevkit/cdk/client';

// HTTP transport
const transport = http({ url: 'https://main.confluxrpc.com' });

// WebSocket transport
const wsTransport = ws({ url: 'wss://main.confluxrpc.com/ws' });

// Fallback transport
const fallbackTransport = fallback([http(), ws()]);
```

## Error Handling

All framework errors extend `CfxError` with stable error codes:

```ts
import { CfxError, isCfxError } from '@cfxdevkit/cdk/errors';

try {
  const client = createClient({ chain, transport });
} catch (error) {
  if (isCfxError(error)) {
    console.error(`Error code: ${error.code}`);
    console.error(`Message: ${error.message}`);
  }
}
```

## Key Features

### 1. Multi-chain Support
Support for both Core Space (Conflux native) and eSpace (EVM-compatible) chains with unified client interfaces.

### 2. Type Safety
Strong typing throughout the system using TypeScript interfaces and type guards.

### 3. Extensible Architecture
Modular design allows for easy extension and customization of chain configurations, transports, and client behavior.

### 4. Error Handling
Consistent error handling with structured error codes and metadata for debugging.

### 5. Protocol Abstraction
High-level protocol helpers simplify common operations like transaction waiting and gas estimation.
