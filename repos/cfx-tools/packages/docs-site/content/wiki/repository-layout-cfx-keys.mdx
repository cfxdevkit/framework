# Repository Layout — cfx-keys

# cfx-keys

**Tier 0b — audit-grade trust boundary.**

This repository contains all code that touches private keys. The repository perimeter **is** the audit perimeter. By isolating cryptographic operations, key management, and signing logic from UI and protocol code, the audit scope is reduced approximately 10×.

## Architecture

The module is split into two primary packages with a strict dependency hierarchy. Consumers should never depend on `wallet` directly for low-level crypto; use `services` for primitives and `wallet` for signer orchestration.

### Dependency Rules

- **`@cfxdevkit/services`**: MAY depend on `@cfxdevkit/cdk`. MUST NOT depend on `cfx-ui`, `cfx-domain`, or `cfx-tools`.
- **`@cfxdevkit/wallet`**: MAY depend on `@cfxdevkit/cdk` and `@cfxdevkit/services`.
- **Consumers**: SHOULD use **tilde** ranges (`~x.y.z`) to force conscious patch upgrades.

## Package: `@cfxdevkit/services`

**Scope:** Stateless service primitives sitting just above `@cfxdevkit/cdk`.

This package provides the foundational building blocks for key storage, encryption, and authentication. It is designed to be pluggable; backends (file, memory, hardware) are exposed as sub-paths.

### Key Sub-paths

| Sub-path | Responsibility |
|----------|----------------|
| `.` | Core interfaces (`KeystoreProvider`, `AuditLogger`) and re-exports. |
| `./keystore` | `KeystoreProvider` interface, `StoredSecret`, `Capability` types. |
| `./keystore-file` | File-backed encrypted keystore (cfx-v1 envelope: Argon2id KEK + AES-256-GCM). |
| `./keystore-memory` | In-memory keystore for testing. |
| `./keystore-ledger` | Ledger hardware wallet adapter. |
| `./crypto` | AES-256-GCM encryption, Argon2id/HKDF derivation, encoding utilities. |
| `./auth` | Session token management (`signSessionToken`, `verifySessionToken`). |
| `./embedded-wallet` | `EmbeddedWalletManager` for keystore-backed wallet lifecycle. |

### Core Interfaces

#### `KeystoreProvider`
The abstraction for secret storage. Implementations include `KeystoreFileProvider`, `KeystoreLedgerProvider`, and `KeystoreMemoryProvider`.

```ts
export interface KeystoreProvider {
  list(options?: KeystoreListOptions): Promise<StoredSecret[]>;
  get(id: string): Promise<StoredSecret | null>;
  put(input: KeystorePutInput): Promise<void>;
  delete(id: string): Promise<void>;
}
```

#### `AuditLogger`
All keystore operations must be logged. Use `createAppendOnlyAuditLogger` to instantiate a logger that writes to a file or external sink.

```ts
export interface AuditLogger {
  log(entry: AuditEntry): void;
}

export interface AuditEntry {
  timestamp: Timestamp;
  operation: string;
  keystoreId?: string;
  secretId?: string;
  caller?: string;
  success: boolean;
  error?: string;
}
```

### Crypto Primitives

The `./crypto` sub-path exposes low-level primitives. Prefer high-level wrappers where available.

- `encryptAesGcm(input: EncryptInput)`: Encrypts data using AES-256-GCM.
- `decryptAesGcm(input: DecryptInput)`: Decrypts data.
- `deriveKeyArgon2id(input: Argon2idInput)`: Derives keys from passwords.
- `generateAesGcmKey()`: Generates a random key.

## Package: `@cfxdevkit/wallet`

**Scope:** Focused wallet primitives — session keys, capability-scoped signers, batched transactions.

**Security Note:** This package is the **only blessed entrypoint for automated signers**. Any project requiring a non-interactive signer MUST use `wallet/session-key` or `wallet/signers/from-hardware`.

### Key Features

#### 1. Local Wallet Initialization
Convenience functions for managing filesystem keystores.

```ts
import { initLocalWallet, openLocalWallet } from '@cfxdevkit/wallet';

// Initialize a new keystore
const result = await initLocalWallet({
  keystorePath: './keys/keystore.json',
  passphrase: 'secure-passphrase'
});

// Open an existing keystore
const wallet = await openLocalWallet({
  keystorePath: './keys/keystore.json',
  passphrase: 'secure-passphrase'
});
```

#### 2. Signers
Create signers from keystores or hardware devices.

```ts
import { signerFromKeystore, readonlySigner } from '@cfxdevkit/wallet';

// From file keystore
const signer = await signerFromKeystore({
  keystorePath: './keys/keystore.json',
  passphrase: 'secure-passphrase'
});

// Watch-only
const readOnly = readonlySigner('0x...');
```

#### 3. Session Keys & Capabilities
Generate ephemeral session keys with restricted capabilities (e.g., gas limits, allowed methods).

```ts
import { createSessionKey, withCapability, checkCapability } from '@cfxdevkit/wallet';

// Create a session key with specific capabilities
const session = await createSessionKey({
  parentSigner: mainSigner,
  capability: { name: 'transfer', args: { maxGas: 100000 } }
});

// Wrap a signer with capability enforcement
const restrictedSigner = withCapability(mainSigner, capability);
```

#### 4. Hardware Wallets
Adapters for Ledger, OneKey, and Satochip are available under `./hardware`.

```ts
import { signerFromLedger, createLedgerHardwareAdapter } from '@cfxdevkit/wallet/hardware/ledger';

const adapter = await createLedgerHardwareAdapter({ path: "m/44'/60'/0'/0/0" });
const signer = await signerFromLedger({ adapter });
```

#### 5. Batched Transactions
Execute multiple transactions in a single call using `TransactionBatcher`.

```ts
import { TransactionBatcher } from '@cfxdevkit/wallet/batcher';

const batcher = new TransactionBatcher();
batcher.add({ to: '0x...', data: '0x...' });
batcher.add({ to: '0x...', data: '0x...' });

const result = await batcher.execute();
```

## Security & Compliance

This repository is subject to stricter CI and release requirements than the rest of the monorepo.

### CI Requirements
- **Audit:** `pnpm audit --prod` + `socket.dev` advisory scan on every PR.
- **Reproducibility:** `SOURCE_DATE_EPOCH` pinned for reproducible builds.
- **SBOM:** CycloneDX Software Bill of Materials attached to every release.
- **Sign-off:** `Signed-off-by` enforced on all commits.
- **Review:** `main` branch requires two reviewers.

### Dependency Constraints
- **Allowed:** `@noble/*`, `@scure/*`, `viem`, `cive`, `@cfxdevkit/cdk`.
- **Forbidden:** React, Phaser, Three.js, `cfx-ui`, `cfx-domain`, `cfx-tools`.

### Audit Logging
All keystore operations (`list`, `get`, `put`, `delete`) must be logged via the `AuditLogger` interface.

```ts
import { createAppendOnlyAuditLogger } from '@cfxdevkit/services/keystore-audit';

const logger = createAppendOnlyAuditLogger({
  onLog: (entry) => {
    // Write to secure log file or SIEM
    fs.appendFileSync('./audit.log', JSON.stringify(entry));
  }
});
```

## Worked Example

The canonical flow for consuming this module in a project:

```ts
// 1. Initialize local keystore (one-time setup)
await initLocalWallet({ keystorePath: './keys.json', passphrase: '...' });

// 2. Open wallet and create signer
const signer = await signerFromKeystore({ keystorePath: './keys.json', passphrase: '...' });

// 3. Create a session key for limited operations
const session = await createSessionKey({
  parentSigner: signer,
  capability: { name: 'swap', args: { maxAmount: '100' } }
});

// 4. Execute transaction via session signer
await session.signTransaction(tx);
```
