# Solidity Compiler & Contracts

# @cfxdevkit/contracts

The `@cfxdevkit/contracts` package provides a unified TypeScript interface for interacting with Solidity contracts on Conflux. It abstracts the differences between **Core Space** (native Conflux) and **eSpace** (EVM-compatible), handling address formatting, transaction construction, and RPC dispatch automatically based on the client's chain family.

## Overview

This module sits on top of `@cfxdevkit/cdk` and `viem`. It standardizes contract reads, writes, deployments, and cross-space bridging.

*   **Dual-Space Support:** Automatically detects `client.family` (`'core'` vs `'espace'`) and routes calls to the appropriate RPC methods (`cfx_call` vs `eth_call`).
*   **ABI Handling:** Uses `viem` for encoding/decoding function data.
*   **Transaction Management:** Handles nonce fetching, gas estimation, and receipt polling internally.
*   **Bridge Utilities:** Provides helpers for moving assets and calling contracts across the Core ↔ eSpace boundary.

## Core Concepts

### Chain Families
All operations rely on a `Client` instance from `@cfxdevkit/cdk`. The `family` property dictates the execution path:

| Feature | Core Space (`'core'`) | eSpace (`'espace'`) |
| :--- | :--- | :--- |
| **Address Format** | Base32 (`cfx:…`) | Hex (`0x…`) |
| **Read RPC** | `cfx_call` | `eth_call` |
| **Write RPC** | `cfx_sendRawTransaction` | `eth_sendRawTransaction` |
| **Gas Estimation** | `cfx_estimateGasAndCollateral` | `eth_estimateGas` |
| **Transaction Type** | CIP-2930 / CIP-1559 | EIP-1559 |

### Address Validation
The module enforces strict address formats to prevent routing errors:
*   **Core:** Must be a valid Base32 string.
*   **eSpace:** Must be a 20-byte hex string (`0x...`).
*   **Bridge:** Core addresses are mapped to eSpace addresses via `mappedEspaceAddress` (Keccak256 hash suffix).

## Interaction Patterns

### Reading Contract State
Use `readContract` for view/pure functions. It handles ABI encoding and RPC dispatch.

```typescript
import { readContract } from '@cfxdevkit/contracts';

const balance = await readContract({
  client, // Core or eSpace client
  address: 'cfx:aaejua...', // Base32 for Core, 0x... for eSpace
  abi: MY_ABI,
  functionName: 'balanceOf',
  args: ['0x...'],
  epochTag: 'latest_state', // Optional for Core
});
```

### Writing Contract State
Use `sendWrite` for state-changing transactions. It constructs the transaction, signs it via the `signer`, broadcasts it, and optionally waits for the receipt.

```typescript
import { sendWrite } from '@cfxdevkit/contracts';

const result = await sendWrite({
  client,
  signer,
  address: 'cfx:...',
  abi: MY_ABI,
  functionName: 'transfer',
  args: ['recipient', amount],
  waitForReceipt: true, // Polls until mined
});
// result.receipt contains the transaction receipt
```

### Deploying Contracts
Use `deployContract`. It abstracts the differences in deployment data encoding and gas estimation between spaces.

```typescript
import { deployContract } from '@cfxdevkit/contracts';

const deployment = await deployContract({
  client,
  signer,
  abi: MY_ABI,
  bytecode: MY_BYTECODE,
  args: [constructorArg],
  waitForReceipt: true,
});
// deployment.address is populated after receipt confirmation
```

## Cross-Space Bridging

The `bridge` module manages interactions with the `CrossSpaceCall` contract (`0x0888...06`), enabling value and code transfer between Core and eSpace.

### Key Functions
*   **`transferToEspace`**: Send native CFX from Core to an eSpace address.
*   **`callEspace`**: Execute arbitrary eSpace code from Core (requires `data` payload).
*   **`withdrawFromMapped`**: Withdraw funds from a mapped eSpace account back to Core.
*   **`getMappedBalance` / `getMappedNonce`**: Query state for a mapped Core address within eSpace context.

### Address Mapping
Core addresses are mapped to eSpace addresses by taking the last 20 bytes of their Keccak256 hash.

```typescript
import { mappedEspaceAddress, transferToEspace } from '@cfxdevkit/contracts/bridge';

const coreAddress = 'cfx:aaejua...';
const espaceAddress = mappedEspaceAddress(coreAddress);

await transferToEspace({
  client, // Must be Core Space client
  signer,
  to: espaceAddress,
  value: 1000000000000000000n, // 1 CFX
});
```

## Standard Token Helpers

The `erc20` namespace provides typed convenience wrappers for standard ERC-20 operations. These work on both spaces automatically.

```typescript
import { erc20 } from '@cfxdevkit/contracts';

const bind = { client, address: '0x...' };

// Read
const name = await erc20.name(bind);
const balance = await erc20.balanceOf(bind, ownerAddress);

// Write
await erc20.transfer({ ...bind, signer }, toAddress, amount);
await erc20.approve({ ...bind, signer }, spenderAddress, amount);
```

## Error Handling

All contract operations throw `ContractsError` (extending `CfxError`) with stable error codes for programmatic handling.

| Code | Description |
| :--- | :--- |
| `contracts/unsupported-family` | Client family is not Core or eSpace. |
| `contracts/invalid-argument` | Address format mismatch or missing signer data. |
| `contracts/decode-failure` | ABI decoding failed on response. |
| `contracts/receipt-timeout` | Transaction not mined within timeout. |
| `contracts/reverted` | Transaction execution reverted on-chain. |

```typescript
import { ContractsError } from '@cfxdevkit/contracts';

try {
  await sendWrite(...);
} catch (err) {
  if (err instanceof ContractsError && err.code === 'contracts/reverted') {
    // Handle revert
  }
}
```

## Testing Utilities

The `test/mocks` submodule provides in-memory clients and signers for unit testing without an RPC endpoint.

*   **`makeMockClient`**: Creates a `Client` stub. Configure `rpc` handlers to simulate responses.
*   **`makeMockSigner`**: Creates a `Signer` stub. Captures signed transactions in `history` for assertions.
*   **`getRecordedCalls`**: Inspects RPC calls made by a mock client.

```typescript
import { makeMockClient, makeMockSigner, getRecordedCalls } from '@cfxdevkit/contracts/test/mocks';

const client = makeMockClient({ family: 'core' });
const signer = makeMockSigner({ coreAddress: 'cfx:...' });

// Execute logic...

// Verify RPC interaction
const calls = getRecordedCalls(client);
expect(calls[0].method).toBe('cfx_call');
```

## API Reference

### Exports

*   **`readContract`**: Generic read function.
*   **`sendWrite`**: Generic write function.
*   **`prepareWrite`**: Offline transaction preparation (returns `SignableTx`).
*   **`deployContract`**: Generic deployment function.
*   **`erc20`**: ERC-20 helper object.
*   **`bridge`**: Cross-space namespace.
*   **`ContractsError`**: Error class.
*   **`ERC20_ABI`**, **`ERC721_ABI`**, **`MULTICALL3_ABI`**: Standard ABIs.

### Types

*   **`DeployContractInput`**: Configuration for `deployContract`.
*   **`SendWriteInput`**: Configuration for `sendWrite`.
*   **`ReadContractInput`**: Configuration for `readContract`.
*   **`BridgeBaseOptions`**: Common options for bridge functions.
