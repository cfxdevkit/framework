# Solidity Compiler & Contracts

# @cfxdevkit/contracts

The `@cfxdevkit/contracts` module provides a typed, unified interface for interacting with smart contracts on the Conflux network. It abstracts the differences between **Core Space** (Conflux native) and **eSpace** (EVM-compatible), allowing developers to write code that works across both environments without managing low-level RPC details.

## Architecture

The module relies on the `client.family` property to determine the target environment. All high-level functions (`deployContract`, `readContract`, `sendWrite`) dispatch to space-specific implementations based on this flag.

## Core Operations

### Deployment

Contract deployment is handled by `deployContract`. It automatically selects the correct deployment strategy based on the client's chain family.

```typescript
import { deployContract } from '@cfxdevkit/contracts/deploy';

const result = await deployContract({
  client,
  signer,
  abi,
  bytecode,
  args: [constructorArg1, constructorArg2],
  waitForReceipt: true,
});
```

**Key Behaviors:**
*   **Core Space:** Uses `deployCoreContract`. Requires `signer.account.coreAddress` (base32). Estimates gas via `cfx_estimateGasAndCollateral`. Applies a 25% buffer to estimated gas/storage limits.
*   **eSpace:** Uses `deployEspaceContract`. Uses `eth_estimateGas`. Fetches `baseFeePerGas` from the latest block to calculate `maxFeePerGas` (EIP-1559 style).
*   **Result:** Returns `DeployContractResult` containing the transaction hash, raw transaction, and optionally the deployed contract address if `waitForReceipt` is true.

### Reading Contract State

Read operations use `readContract` to execute view/pure functions. It handles ABI encoding/decoding via `viem` and dispatches the correct RPC method.

```typescript
import { readContract } from '@cfxdevkit/contracts/read';

const balance = await readContract({
  client,
  address: 'cfx:aa...123', // Base32 for Core, 0x... for eSpace
  abi,
  functionName: 'balanceOf',
  args: [ownerAddress],
});
```

**Address Validation:**
*   **Core Space:** Addresses must be base32 (`cfx:`, `cfxtest:`, or `net<id>:`). Passing a `0x` hex address throws `ContractsError` with code `contracts/invalid-argument`.
*   **eSpace:** Addresses must be `0x`-prefixed 20-byte hex.

**RPC Dispatch:**
*   **Core:** Calls `cfx_call` with an `epochTag` (default `latest_state`).
*   **eSpace:** Calls `eth_call` with a `blockTag` (default `latest`).

### Writing Contract State

Write operations use `sendWrite` to sign and broadcast transactions.

```typescript
import { sendWrite } from '@cfxdevkit/contracts/write';

const result = await sendWrite({
  client,
  signer,
  address,
  abi,
  functionName: 'transfer',
  args: [to, amount],
  waitForReceipt: true,
});
```

**Transaction Construction:**
*   **Core Space:** Uses `sendCoreWrite`. Requires `signer.account.coreAddress`. Supports `coreType` ('legacy', 'cip2930', 'cip1559'). Defaults to `cip2930`.
*   **eSpace:** Uses `sendEspaceWrite`. Uses EIP-1559 fee structure (`maxFeePerGas`, `maxPriorityFeePerGas`).
*   **Receipt Polling:** If `waitForReceipt` is enabled, `waitForReceipt` polls `cfx_getTransactionReceipt` or `eth_getTransactionReceipt` until the transaction is confirmed or a timeout occurs.

## Cross-Space Bridging

The `bridge` module facilitates interactions between Core Space and eSpace using the `CROSS_SPACE_CALL_ADDRESS` (`0x0888000000000000000000000000000000000006`).

### Transferring to eSpace

Moves value from Core Space to a mapped eSpace address.

```typescript
import { transferToEspace } from '@cfxdevkit/contracts/bridge';

await transferToEspace({
  client,
  signer,
  to: '0x...eSpaceAddress',
  value: 1000000000000000000n, // 1 CFX
});
```

### Calling eSpace Contracts from Core

Executes arbitrary EVM calldata from Core Space.

```typescript
import { callEspace } from '@cfxdevkit/contracts/bridge';

await callEspace({
  client,
  signer,
  to: '0x...eSpaceContract',
  data: '0x...', // Encoded calldata
  value: 0n,
});
```

### Mapped Account State

Retrieves the balance and nonce of a Core Space address as seen within eSpace.

```typescript
import { getMappedBalance, getMappedNonce } from '@cfxdevkit/contracts/bridge';

const balance = await getMappedBalance({ client, coreHexAddress: '0x...' });
const nonce = await getMappedNonce({ client, coreHexAddress: '0x...' });
```

**Address Mapping:**
Use `mappedEspaceAddress` to calculate the eSpace address corresponding to a Core Space hex address.
```typescript
import { mappedEspaceAddress } from '@cfxdevkit/contracts/bridge';

const espaceAddr = mappedEspaceAddress('0x...coreHex');
```

## ERC20 Helpers

The `erc20` namespace provides typed wrappers for standard ERC20 functions, pre-binding the ABI.

```typescript
import { erc20 } from '@cfxdevkit/contracts/erc20';

// Read
const name = await erc20.name({ client, address });
const balance = await erc20.balanceOf({ client, address }, owner);

// Write
await erc20.transfer({ client, address, signer }, to, amount);
await erc20.approve({ client, address, signer }, spender, amount);
```

## Error Handling

All contract operations throw `ContractsError` derived from `@cfxdevkit/cdk`. Errors include stable codes for programmatic handling.

| Code | Description |
| :--- | :--- |
| `contracts/unsupported-family` | Client family is not 'core' or 'espace'. |
| `contracts/invalid-argument` | Address format mismatch (e.g., hex in Core Space) or missing signer fields. |
| `contracts/decode-failure` | ABI decoding failed on response data. |
| `contracts/receipt-timeout` | Transaction receipt not found within timeout. |
| `contracts/reverted` | Transaction status indicates a revert. |

```typescript
import { ContractsError } from '@cfxdevkit/contracts/errors';

try {
  await sendWrite(...);
} catch (err) {
  if (err instanceof ContractsError && err.code === 'contracts/reverted') {
    // Handle revert
  }
}
```

## Testing

The module includes mock utilities for unit testing without an RPC endpoint.

### Mock Client

Creates an in-memory client that records RPC calls and returns stubbed data.

```typescript
import { makeMockClient, mockChain } from '@cfxdevkit/contracts/test/mocks';

const client = makeMockClient({
  family: 'espace',
  chainId: 71,
  rpc: {
    'eth_call': () => '0x...',
  },
});
```

### Mock Signer

Creates a signer that records signed transactions for verification.

```typescript
import { makeMockSigner } from '@cfxdevkit/contracts/test/mocks';

const signer = makeMockSigner({
  address: '0x...',
  coreAddress: 'cfx:...', // Required for Core Space tests
});

// After operation
expect(signer.history).toHaveLength(1);
```

### Recording Calls

Verify which RPC methods were invoked during a test.

```typescript
import { getRecordedCalls } from '@cfxdevkit/contracts/test/mocks';

const calls = getRecordedCalls(client);
expect(calls[0].method).toBe('eth_sendRawTransaction');
```
