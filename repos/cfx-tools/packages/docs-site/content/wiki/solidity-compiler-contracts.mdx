# Solidity Compiler & Contracts

# Solidity Compiler & Contracts Module

This module provides a complete toolchain for compiling Solidity smart contracts and interacting with them on the Conflux network — supporting both **Core Space** and **eSpace**. It is split into two main packages:

- `@cfxdevkit/compiler`: Compiles Solidity sources to artifacts (ABI, bytecode, metadata) using `solc`, with robust import resolution and error handling.
- `@cfxdevkit/contracts`: High-level helpers for deploying, reading, and writing to contracts on Conflux — abstracting away RPC family differences and encoding/decoding.

Both packages are designed to be **framework-agnostic**, **type-safe**, and **testable**, with first-class support for mocking and integration testing.

### Key Capabilities

| Feature | Description |
|--------|-------------|
| **Pinned solc versions** | Downloads and caches exact `soljson-vX.Y.Z+commit.*.js` binaries from `binaries.soliditylang.org`, with SHA-256 verification. |
| **Import resolution** | Supports `npm`-style imports (`@openzeppelin/...`) and Foundry-style remappings (`@oz/=node_modules/@openzeppelin/`). |
| **Dual-space support** | Transparently handles Core Space (`cfx:...`) and eSpace (`0x...`) addresses, RPC methods, and transaction formats. |
| **Typed contract calls** | Uses viem’s ABI encoding/decoding for reads and writes — no manual ABI parsing. |
| **Test-friendly** | Includes in-memory mocks (`mockClient`, `mockSigner`) for unit testing without RPC. |

## `@cfxdevkit/compiler`

### Purpose

Compiles Solidity source files into standardized artifacts (ABI, bytecode, metadata), with:

- Standard JSON input generation
- Async import resolution (via pluggable resolvers)
- Deterministic output (stable `inputHash`)
- Rich error reporting with stable error codes

### Core Types

```ts
interface Source {
  path: string;
  content: string;
}

interface CompileInput {
  sources: readonly Source[];
  solcVersion: string;
  optimizer?: { enabled: boolean; runs: number };
  evmVersion?: string;
  remappings?: readonly string[];
  resolver?: ImportResolver;
  signal?: AbortSignal;
}

interface Artifact {
  path: string;
  contractName: string;
  abi: Abi;
  bytecode: Hex;
  deployedBytecode: Hex;
  metadata: string;
  sourceMap?: string;
}

interface CompileOutput {
  artifacts: readonly Artifact[];
  warnings: readonly CompileDiagnostic[];
  inputHash: string; // SHA-256 of normalized JSON input
  solcVersion: string;
}
```

### Main Entry Point: `compile()`

```ts
import { compile } from '@cfxdevkit/compiler';

const output = await compile({
  solcVersion: '0.8.26',
  sources: [{ path: 'MyToken.sol', content: 'pragma solidity ^0.8.0; contract MyToken {}' }],
  optimizer: { enabled: true, runs: 200 },
});
```

#### How It Works

1. **Build Standard JSON**  
   `buildStandardJson()` normalizes sources, applies optimizer/remappings, and sets default output selection (`abi`, `bytecode`, `deployedBytecode`, `metadata`, `sourceMap`).

2. **Pre-resolve Imports**  
   If a resolver is provided, `preResolve()` walks the import graph and caches resolved sources before calling solc.

3. **Call solc**  
   `runSolcWithResolvedImports()` repeatedly calls `solc.compile()` with an `import` callback until all imports are resolved (up to 8 attempts to handle transitive dependencies).

4. **Parse Output**  
   `artifactsFrom()` converts raw solc output into typed `Artifact[]`.

5. **Error Handling**  
   `diagnostics()` separates hard errors (from `severity: 'error'`) and warnings. Hard errors throw `CompileError`.

### Import Resolvers

| Resolver | Purpose |
|---------|---------|
| `npmResolver({ rootDir })` | Resolves `@scope/pkg/file.sol` via `node_modules/` (Node-style). |
| `remappingResolver(remappings)` | Supports Foundry-style `prefix=path` rewrites (e.g., `@oz/=node_modules/@openzeppelin/`). |
| `compose(resolvers)` | Chains resolvers — returns first non-null result. |

### Artifact Helpers

```ts
// Extract 4-byte function selectors (constructor/fallback excluded)
const selectors = selectorsOf(abi); // Hex[]

// Read/write artifacts to disk
const artifact = await readArtifact('build/MyToken.json');
await writeArtifact('dist/MyToken.json', artifact);
```

### solc Binary Management (`ensureSolc`)

- **Caching**: Defaults to `$XDG_CACHE_HOME/cfxdevkit/solc/` (or `~/.cache/...`)
- **Version pinning**: Accepts clean semver (`"0.8.26"`), looks up exact build via `list.json`
- **Concurrency-safe**: Concurrent `ensureSolc("0.8.26")` calls share the same in-flight download promise
- **Verification**: SHA-256 checksums are validated after download

```ts
import { ensureSolc, listInstalledSolc } from '@cfxdevkit/compiler/solc/loader';

const solc = await ensureSolc('0.8.26');
const versions = await listInstalledSolc(); // e.g., ['0.8.25', '0.8.26']
```

### Error Handling

All errors derive from `CompileError` with stable `code`s:

| Code | Meaning |
|------|---------|
| `compiler/solc/syntax` | solc emitted one or more errors |
| `compiler/resolver/not-found` | Import could not be resolved |
| `compiler/solc/binary-unavailable` | solc binary missing or checksum mismatch |
| `compiler/io-failure` | Artifact read/write failed |
| `compiler/invalid-argument` | Input validation failure |

## `@cfxdevkit/contracts`

### Purpose

Provides **typed, family-aware helpers** for:

- Deploying contracts (Core/eSpace)
- Reading state (view/pure calls)
- Sending transactions (nonpayable/payable)
- Bridge-specific operations (Core ↔ eSpace transfers)

All helpers automatically select the correct RPC method and address format based on `client.family`.

### Core Helpers

| Helper | Family | Description |
|--------|--------|-------------|
| `deployContract()` | Both | Deploys a contract, returns `hash`, `receipt`, and `address` |
| `readContract()` | Both | Calls view/pure functions, decodes return value |
| `sendWrite()` | Both | Sends a transaction, waits for receipt if requested |
| `prepareWrite()` | Both | Builds a `SignableTx` without sending (for signing externally) |

#### Example: Deploy

```ts
import { deployContract } from '@cfxdevkit/contracts';

const result = await deployContract({
  client, // CoreSpaceClient or EspaceClient
  signer,
  abi: MyTokenABI,
  bytecode: '0x6080...',
  args: ['MyToken', 'MTK', 18n, 1_000_000n],
  waitForReceipt: true,
});

console.log(result.address); // e.g., 'cfx:...` or '0x...'
```

#### Example: Read

```ts
import { readContract } from '@cfxdevkit/contracts';

const totalSupply = await readContract({
  client,
  address: '0x1234...5678', // eSpace or base32 Core
  abi: ERC20_ABI,
  functionName: 'totalSupply',
});
```

#### Example: Write

```ts
import { sendWrite } from '@cfxdevkit/contracts';

const result = await sendWrite({
  client,
  signer,
  address: '0x1234...5678',
  abi: ERC20_ABI,
  functionName: 'transfer',
  args: ['0x9999...0000', 100n],
  waitForReceipt: true,
});
```

### Family-Specific Behavior

| Operation | eSpace (`family: 'espace'`) | Core Space (`family: 'core'`) |
|----------|------------------------------|-------------------------------|
| Address format | `0x`-prefixed 20-byte hex | Base32 (`cfx:...`, `cfxtest:...`) |
| Read RPC | `eth_call` | `cfx_call` |
| Write RPC | `eth_sendRawTransaction` | `cfx_sendRawTransaction` |
| Gas estimation | `estimateGas()` | `cfx_estimateGasAndCollateral()` |
| Nonce | `eth_getTransactionCount` | `cfx_getNextNonce` |
| Fee model | EIP-1559 (`maxFeePerGas`, `maxPriorityFeePerGas`) | Legacy/CIP-2930 (`gasPrice`, `storageLimit`) |

### Bridge Helpers (`@cfxdevkit/contracts/bridge`)

Specialized helpers for **Core ↔ eSpace transfers** using the `CrossSpaceCall` contract (`0x0888...0006`).

| Function | Purpose |
|---------|---------|
| `transferToEspace()` | Transfer tokens from Core to eSpace |
| `callEspace()` | Call a contract on eSpace from Core |
| `withdrawFromMapped()` | Withdraw from mapped eSpace balance |
| `mappedEspaceAddress()` | Derive eSpace address from Core address |
| `getMappedBalance()` / `getMappedNonce()` | Query mapped state |

```ts
import { transferToEspace, mappedEspaceAddress } from '@cfxdevkit/contracts/bridge';

const eSpaceAddr = mappedEspaceAddress('0x1234...5678'); // e.g., '0x...'

await transferToEspace({
  client, // CoreSpaceClient
  signer,
  to: eSpaceAddr,
  value: 1n,
});
```

### ERC-20 Helpers (`@cfxdevkit/contracts/erc20`)

Thin wrappers over `readContract`/`sendWrite` for common ERC-20 operations.

```ts
import { erc20 } from '@cfxdevkit/contracts/erc20';

const name = await erc20.name({ client, address });
const balance = await erc20.balanceOf({ client, address }, owner);
await erc20.transfer({ client, address, signer }, to, amount);
```

### Error Handling

All errors derive from `ContractsError` with stable codes:

| Code | Meaning |
|------|---------|
| `contracts/unsupported-family` | Client family not supported (e.g., `core` required) |
| `contracts/invalid-argument` | Malformed address, missing field, etc. |
| `contracts/receipt-timeout` | Transaction receipt not found within timeout |
| `contracts/reverted` | Transaction reverted |
| `contracts/decode-failure` | Failed to decode return data |
