# Solidity Compiler & Contracts Module

## Overview

The `@cfxdevkit/contracts` module provides a set of utilities and helpers for interacting with Ethereum-compatible smart contracts on both eSpace and Core Space within the Conflux ecosystem. It abstracts low-level RPC interactions, transaction signing, and contract deployment, offering a typed and ergonomic interface for developers.

This module is designed to work with the `@cfxdevkit/cdk` client and signer abstractions, supporting both eSpace (`eth_*`) and Core Space (`cfx_*`) RPC methods.

## Key Features

- **Typed Contract Reads**: Perform `eth_call` or `cfx_call` with ABI decoding.
- **Contract Writes**: Send transactions to eSpace or Core Space with automatic gas estimation.
- **Contract Deployments**: Deploy contracts on either eSpace or Core Space with full transaction lifecycle management.
- **Bridge Helpers**: Utilities for cross-space communication, including mapping Core addresses to eSpace equivalents.
- **ERC-20 Convenience**: Pre-bound helpers for common ERC-20 operations.
- **Error Handling**: Standardized error codes and messages for predictable error handling.

## Core Components

### 1. `readContract` — Typed Contract Reads

Reads contract state using `eth_call` (eSpace) or `cfx_call` (Core Space).

```ts
import { readContract } from '@cfxdevkit/contracts/read';

const balance = await readContract({
  client,
  address: '0x1234...', // eSpace address or Core base32
  abi: ERC20_ABI,
  functionName: 'balanceOf',
  args: ['0x5678...'],
});
```

### 2. `sendWrite` — Typed Contract Writes

Sends a transaction to a contract function, with automatic gas estimation and receipt polling.

```ts
import { sendWrite } from '@cfxdevkit/contracts/write';

const result = await sendWrite({
  client,
  signer,
  address: '0x1234...',
  abi: ERC20_ABI,
  functionName: 'transfer',
  args: ['0x5678...', 100n],
  waitForReceipt: true,
});
```

### 3. `deployContract` — Contract Deployment

Deploys a contract on either eSpace or Core Space, handling transaction signing and receipt polling.

```ts
import { deployContract } from '@cfxdevkit/contracts/deploy';

const result = await deployContract({
  client,
  signer,
  abi: MyContractABI,
  bytecode: '0x...',
  waitForReceipt: true,
});
```

### 4. Bridge Helpers

Utilities for interacting with the cross-space call contract (`0x0888...`).

#### `mappedEspaceAddress`

Maps a Core Space address to an eSpace address using keccak256.

```ts
import { mappedEspaceAddress } from '@cfxdevkit/contracts/bridge';

const espaceAddr = mappedEspaceAddress('cfx:...'); // e.g., '0x...'
```

#### `transferToEspace`

Transfers CFX from Core Space to eSpace.

```ts
import { transferToEspace } from '@cfxdevkit/contracts/bridge';

await transferToEspace({
  client,
  signer,
  to: '0x1234...',
  value: 1000000000000000000n,
});
```

#### `callEspace`

Calls an eSpace contract from Core Space.

```ts
import { callEspace } from '@cfxdevkit/contracts/bridge';

await callEspace({
  client,
  signer,
  to: '0x1234...',
  data: '0x...',
});
```

#### `getMappedBalance`

Fetches the balance of a mapped eSpace address.

```ts
import { getMappedBalance } from '@cfxdevkit/contracts/bridge';

const balance = await getMappedBalance({
  client,
  coreHexAddress: '0x1234...',
});
```

### 5. ERC-20 Helpers

Pre-bound helpers for common ERC-20 operations.

```ts
import { erc20 } from '@cfxdevkit/contracts/erc20';

const balance = await erc20.balanceOf({ client, address: '0x...' }, '0x...');
const transferResult = await erc20.transfer(
  { client, address: '0x...', signer },
  '0x...',
  100n
);
```

## Error Handling

All errors derive from `CfxError` and include a stable `code` for programmatic handling.

```ts
import { ContractsError } from '@cfxdevkit/contracts/errors';

try {
  await readContract(...);
} catch (err) {
  if (err instanceof ContractsError) {
    switch (err.code) {
      case 'contracts/decode-failure':
        // Handle decoding errors
        break;
      case 'contracts/reverted':
        // Handle reverted transactions
        break;
    }
  }
}
```

## Integration with CDK

This module integrates with `@cfxdevkit/cdk`'s `Client` and `Signer` abstractions:

- `Client.family` determines whether to use `eth_*` or `cfx_*` RPC methods.
- `Signer` handles transaction signing and account management.
- `SignableTx` is used for transaction construction and signing.

## Testing Utilities

The `test/mocks.ts` module provides:

- `makeMockClient`: A mock `Client` for testing without an RPC endpoint.
- `makeMockSigner`: A mock `Signer` that records signed transactions.

These are used in unit tests to isolate contract logic.

## Re-exports

The module barrel exports are available for convenience:

```ts
import * as Contracts from '@cfxdevkit/contracts';

// Access via:
Contracts.bridge.transferToEspace(...)
Contracts.deployContract(...)
Contracts.erc20.balanceOf(...)
```

For new code, prefer importing directly from sub-paths to enable tree-shaking:

```ts
import { transferToEspace } from '@cfxdevkit/contracts/bridge';
import { deployContract } from '@cfxdevkit/contracts/deploy';
import { erc20 } from '@cfxdevkit/contracts/erc20';
```
