# Solidity Compiler & Contracts

# @cfxdevkit/contracts

The `@cfxdevkit/contracts` package provides a unified interface for interacting with Solidity contracts on Conflux networks. It abstracts the differences between **Core Space** (native Conflux) and **eSpace** (EVM-compatible) chains, handling ABI encoding, transaction construction, signing, and receipt polling.

## Architecture

The module routes operations based on the `client.family` property (`'core'` or `'espace'`). Underlying RPC methods differ (`cfx_call`/`cfx_sendRawTransaction` vs `eth_call`/`eth_sendRawTransaction`), but the public API remains consistent.

### Key Dependencies
- **`viem`**: Used for ABI encoding/decoding (`encodeFunctionData`, `decodeFunctionResult`) and hex utilities.
- **`@cfxdevkit/cdk`**: Provides `Client`, `Signer`, and `TxReceipt` abstractions.

## Core Operations

### Reading Contract State

Use `readContract` for view/pure functions. It automatically selects `eth_call` (eSpace) or `cfx_call` (Core Space).

```typescript
import { readContract } from '@cfxdevkit/contracts/read';
import { ERC20_ABI } from '@cfxdevkit/contracts/abis';

const balance = await readContract({
  client,
  address: 'cfx:aaejuaaaaaaaaaaaaaaaaaaaaaaaaaaaa81557', // Core Space base32
  // OR address: '0x0888000000000000000000000000000000000006' // eSpace hex
  abi: ERC20_ABI,
  functionName: 'balanceOf',
  args: ['0x...'],
  epochTag: 'latest_state', // Core Space only
  blockTag: 'latest',       // eSpace only
});
```

**Address Formats:**
- **eSpace**: `0x`-prefixed 20-byte hex.
- **Core Space**: Base32 format (`cfx:…`, `cfxtest:…`, or `net<id>:…`).

### Writing Contract State

Use `sendWrite` to execute non-payable/payable functions. This function constructs the transaction, signs it via the `signer`, sends it to the node, and optionally waits for the receipt.

```typescript
import { sendWrite } from '@cfxdevkit/contracts/write';

const result = await sendWrite({
  client,
  signer,
  address: '0x...',
  abi: ERC20_ABI,
  functionName: 'transfer',
  args: ['0xrecipient', 1000n],
  waitForReceipt: true,
});
```

**Transaction Construction:**
- **eSpace**: Uses EIP-1559 fields (`maxFeePerGas`, `maxPriorityFeePerGas`). Gas is estimated via `client.estimateGas`.
- **Core Space**: Uses native fields (`gas`, `storageLimit`, `gasPrice`, `epochHeight`). Gas is estimated via `cfx_estimateGasAndCollateral`.
- **Signer Requirement**: Core Space writes require `signer.account.coreAddress` (base32) to be present.

### Deploying Contracts

The `deployContract` function handles bytecode deployment. It inspects the client family to invoke the appropriate deployment logic.

```typescript
import { deployContract } from '@cfxdevkit/contracts/deploy';

const deployment = await deployContract({
  client,
  signer,
  abi: MyContractABI,
  bytecode: '0x...',
  args: [constructorArg1],
  waitForReceipt: true,
});

console.log(deployment.address); // Populated if waitForReceipt is true
```

**Gas Estimation:**
- **eSpace**: Estimates gas and fetches `baseFeePerGas` from the latest block.
- **Core Space**: Estimates gas and collateral. Applies a 125% buffer to the estimated gas limit by default.

## Convenience Helpers

### ERC20 Wrapper

The `erc20` object provides typed shortcuts for standard ERC20 functions. It binds the ABI internally.

```typescript
import { erc20 } from '@cfxdevkit/contracts/erc20';

// Read
const name = await erc20.name({ client, address: '0x...' });
const balance = await erc20.balanceOf({ client, address: '0x...' }, '0xowner');

// Write
const tx = await erc20.transfer(
  { client, address: '0x...', signer },
  '0xrecipient',
  1000n
);
```

### Cross-Space Bridge

The `bridge` namespace facilitates interaction between Core Space and eSpace using the `CROSS_SPACE_CALL` contract (`0x0888000000000000000000000000000000000006`).

```typescript
import { bridge } from '@cfxdevkit/contracts/bridge';

// Transfer native CFX from Core to eSpace
await bridge.transferToEspace({
  client, // Must be Core Space client
  signer,
  to: '0x0888000000000000000000000000000000000006', // eSpace address
  value: 1000000000000000000n,
});

// Query mapped balance on Core Space
const balance = await bridge.getMappedBalance({
  client, // Must be Core Space client
  coreHexAddress: '0x...', // The eSpace address mapped to Core
});

// Convert eSpace address to Core mapped address (last 40 chars of keccak hash)
const mapped = bridge.mappedEspaceAddress('0x0888...');
```

## Error Handling

All contract operations throw `ContractsError` on failure. Errors include a stable `code` for programmatic handling.

| Code | Description |
| :--- | :--- |
| `contracts/unsupported-family` | Client family is not supported by the operation. |
| `contracts/invalid-argument` | Address format mismatch (e.g., hex on Core Space) or missing signer fields. |
| `contracts/decode-failure` | ABI decoding failed on the return data. |
| `contracts/receipt-timeout` | Transaction receipt not found within timeout. |
| `contracts/reverted` | Transaction executed but reverted on-chain. |

```typescript
import { ContractsError } from '@cfxdevkit/contracts/errors';

try {
  await sendWrite({ ... });
} catch (err) {
  if (err instanceof ContractsError && err.code === 'contracts/reverted') {
    // Handle revert
  }
}
```

## Testing & Mocks

The package includes a test utilities module for mocking clients and signers without an RPC endpoint.

```typescript
import { makeMockClient, makeMockSigner } from '@cfxdevkit/contracts/test/mocks';

const client = makeMockClient({
  family: 'espace',
  chainId: 71,
  rpc: { 'eth_call': '0x...' }, // Mock specific RPC responses
});

const signer = makeMockSigner({
  address: '0x...',
  coreAddress: 'cfx:...', // Required for Core Space tests
  rawTx: '0xsigned...',
});
```

**Mock Client Features:**
- Records RPC calls for assertion (`getRecordedCalls`).
- Stubbed `estimateGas` and `getBlock`.
- Supports both `core` and `espace` families.
