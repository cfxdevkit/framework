# Solidity Compiler & Contracts

# `@cfxdevkit/contracts`

A contract interaction layer for the Conflux network that works across both **eSpace** (EVM-compatible) and **Core Space**. It handles contract reads, writes, deployments, and cross-space bridging, automatically routing to the correct RPC methods based on `client.family`.

## Architecture Overview

<Mermaid chart={`graph TD
    A[readContract] --> B{client.family}
    C[sendWrite] --> B
    D[deployContract] --> B
    B -->|espace| E["eth_call / eth_sendRawTransaction"]

    C --> I[waitForReceipt]

All operations share the same pattern: validate inputs, encode with viem's pure helpers, dispatch the appropriate RPC method, decode or surface results.

## Reading Contracts

```ts
import { readContract } from '@cfxdevkit/contracts/read';

const balance = await readContract({
  client,
  address: '0xTokenAddress',   // 0x hex for eSpace; cfx:… base32 for Core
  abi: ERC20_ABI,
  functionName: 'balanceOf',
  args: ['0xOwnerAddress'],
});
```

`readContract` encodes the call with `encodeFunctionData`, dispatches `eth_call` (eSpace) or `cfx_call` (Core), then decodes with `decodeFunctionResult`. Address format is validated per family — passing a 0x hex address to a Core client throws `contracts/invalid-argument`.

**Options:**
- `blockTag` — eSpace block selector (`'latest'`, `'pending'`, a block number as `bigint`, etc.)
- `epochTag` — Core Space epoch selector (`'latest_state'`, `'latest_checkpoint'`, etc.)
- `from` — optional caller address
- `signal` — `AbortSignal` to cancel the underlying RPC call

## Writing to Contracts

```ts
import { sendWrite } from '@cfxdevkit/contracts/write';

const result = await sendWrite({
  client,
  signer,
  address: '0xContractAddress',
  abi: MY_ABI,
  functionName: 'setOwner',
  args: ['0xNewOwner'],
  waitForReceipt: true,
});
// result.hash, result.receipt, result.rawTransaction
```

`sendWrite` routes to `sendEspaceWrite` or `sendCoreWrite` based on `client.family`.

Both paths follow the same sequence:
1. Encode calldata with `encodeFunctionData`
2. Fetch nonce, estimate gas, and fetch current gas price (skipped for any explicitly provided values)
3. Build a `SignableTx` and call `signer.signTransaction`
4. Submit via `eth_sendRawTransaction` / `cfx_sendRawTransaction`
5. Optionally poll for receipt via `waitForReceipt`

**Core-specific fields:** `storageLimit`, `epochHeight`, `gasPrice`, `coreType` (`'legacy' | 'cip2930' | 'cip1559'`). The signer's `account.coreAddress` (base32) must be set for Core writes.

**eSpace-specific fields:** `maxFeePerGas`, `maxPriorityFeePerGas`. Base fee is fetched from the latest block; `maxFeePerGas` defaults to `baseFee * 2 + maxPriorityFeePerGas`.

If neither `gas` nor `storageLimit` are provided for Core, the module calls `cfx_estimateGasAndCollateral` and applies a 1.25x buffer. eSpace applies the same buffer on `estimateGas`.

### Offline transaction preparation

If you need to sign and submit separately:

```ts
import { prepareWrite } from '@cfxdevkit/contracts/write';

const tx = prepareWrite({
  chainId: 71,
  address: '0xContract',
  abi: MY_ABI,
  functionName: 'transfer',
  args: ['0xRecipient', 100n],
  gas: 50_000n,
});
// tx is a SignableTx — pass to signer.signTransaction
```

### Receipt polling

`waitForReceipt` polls every `pollIntervalMs` (default 1500 ms) until either the receipt appears or `timeoutMs` (default 60 000 ms) elapses. It throws `contracts/receipt-timeout` on timeout and `contracts/reverted` if the receipt's status is not `'success'` / `1` / `0x0`.

## Deploying Contracts

```ts
import { deployContract } from '@cfxdevkit/contracts/deploy';

const result = await deployContract({
  client,
  signer,
  abi: MyContract.abi,
  bytecode: MyContract.bytecode,
  args: [constructorArg1],
  waitForReceipt: true,
});

console.log(result.address); // populated when waitForReceipt: true
```

`deployContract` dispatches to `deployCoreContract` or `deployEspaceContract`. Both encode constructor calldata with `encodeDeployData`, estimate gas if not provided, build and sign the transaction, then broadcast it. With `waitForReceipt: true` the deployed contract address is read from `receipt.contractCreated` (Core) or `receipt.contractAddress` (eSpace) and surfaced as `result.address`.

`DeployContractInput` accepts the same gas/fee fields as `SendWriteInput`, plus `coreType` for Core transaction type selection.

## ERC-20 Helpers

```ts
import { erc20 } from '@cfxdevkit/contracts/erc20';

const bind = { client, address: '0xToken' };

const [name, symbol, decimals] = await Promise.all([
  erc20.name(bind),
  erc20.symbol(bind),
  erc20.decimals(bind),
]);

const balance = await erc20.balanceOf(bind, '0xOwner');

await erc20.transfer({ ...bind, signer }, '0xRecipient', 1000n, { waitForReceipt: true });
await erc20.approve({ ...bind, signer }, '0xSpender', 500n);
```

`erc20` is a plain object of pre-bound helpers. All methods work on eSpace and Core Space. Reads delegate to `readContract`; writes delegate to `sendWrite`.

Available methods: `name`, `symbol`, `decimals`, `totalSupply`, `balanceOf`, `allowance`, `transfer`, `approve`.

## Cross-Space Bridge

The bridge module wraps the `CrossSpaceCall` precompile at `0x0888000000000000000000000000000000000006` (Core address `crossSpaceCallAddress(client)` resolved via `hexToBase32`). All bridge operations require a **Core Space client**.

```ts
import { bridge } from '@cfxdevkit/contracts';

// Send CFX from Core to an eSpace address
await bridge.transferToEspace({ client, signer, to: '0xEspaceAddr', value: 10n ** 18n });

// Call an eSpace contract from Core
await bridge.callEspace({ client, signer, to: '0xEspaceContract', data: '0x...', value: 0n });

// Withdraw CFX from the mapped eSpace balance back to Core
await bridge.withdrawFromMapped({ client, signer, value: 10n ** 18n });

// Query the mapped balance/nonce for a Core hex address
const balance = await bridge.getMappedBalance({ client, coreHexAddress: '0xCoreHex' });
const nonce   = await bridge.getMappedNonce({ client, coreHexAddress: '0xCoreHex' });

// Derive the eSpace mirror address for a Core hex address
const mirror = bridge.mappedEspaceAddress('0xCoreHex');
```

`mappedEspaceAddress` derives the deterministic eSpace mirror address by taking the last 20 bytes of `keccak256(coreHexAddress)`.

Both `coreHexAddress` inputs and eSpace `to` addresses are validated with regex guards; mismatched formats throw `contracts/invalid-argument`.

## ABIs

Standard ABIs live in the leaf package `@cfxdevkit/abis` and are re-exported here for back-compat:

```ts
// Preferred (new code)
import { ERC20_ABI, ERC721_ABI, ERC1155_ABI, MULTICALL3_ABI, MULTICALL3_ADDRESS } from '@cfxdevkit/abis';

// Back-compat
import { ERC20_ABI } from '@cfxdevkit/contracts/abis';
```

## Error Handling

All errors are `ContractsError` instances (extends `CfxError`) with a stable string `code`:

| Code | When thrown |
|---|---|
| `contracts/unsupported-family` | Bridge helper called with a non-Core client |
| `contracts/invalid-argument` | Malformed address, missing `coreAddress` on signer, etc. |
| `contracts/decode-failure` | Return data doesn't match the ABI |
| `contracts/receipt-timeout` | Receipt not seen within `receiptTimeoutMs` |
| `contracts/reverted` | Transaction receipt status indicates revert |

```ts
import { ContractsError } from '@cfxdevkit/contracts/errors';

try {
  await sendWrite({ ... });
} catch (err) {
  if (err instanceof ContractsError && err.code === 'contracts/reverted') {
    // handle revert
  }
}
```

## Testing

`src/test/mocks.ts` provides in-memory fixtures:

```ts
import { makeMockClient, makeMockSigner, getRecordedCalls } from '@cfxdevkit/contracts/test/mocks';

const client = makeMockClient({
  family: 'espace',
  rpc: {
    'eth_getTransactionCount': '0x1',
    'eth_sendRawTransaction': '0xdeadbeef',
  },
  receipt: { status: 'success', contractAddress: '0xDeployed' },
});

const signer = makeMockSigner({ address: '0xCaller' });

await deployContract({ client, signer, abi, bytecode, waitForReceipt: true });

const calls = getRecordedCalls(client);
// inspect which RPC methods were called and with what params
```

`makeMockClient` supports both `'espace'` and `'core'` families. RPC handlers can be static values or functions `(req: RpcRequest) => unknown` for dynamic responses. `makeMockSigner` records every signed transaction in `signer.history` for assertions.
