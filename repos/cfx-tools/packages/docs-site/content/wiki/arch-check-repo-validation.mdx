# Architecture Check & Repo Validation

## Overview

The `arch-check` package provides a comprehensive suite of repository validation tools for the CFXDevKit monorepo. It enforces architectural rules, validates documentation contracts, detects code hotspots, scans for secrets, and generates structured reports for CI/CD pipelines and LLM agents.

The companion `cdk-repo-check` package wraps these checks with structured JSON output suitable for automated tooling and LLM integration.

## Core Components

### API Extraction (`api/extract.ts`)

Extracts public exports from compiled `.d.ts` files for API documentation generation.

```typescript
export type ExtractedExport = {
  name: string;
  kind: ExportKind; // 'type' | 'interface' | 'class' | 'function' | 'const' | 'enum' | 'other'
  signature: string;
};

export async function extractSubpathExports(
  distDir: string,
  subpathDtsPath: string,
): Promise<ExtractedExport[]>;
```

**Key Features:**
- Resolves `export * from` re-exports (depth-limited to 2)
- Handles multiple `.d.ts` resolution strategies
- Filters duplicate exports

### Package Discovery (`api/filter.ts`)

Identifies public packages that require API documentation.

```typescript
export async function discoverPublicPackages(): Promise<PublicPackageInfo[]>;
```

**Excluded packages:**
- Private packages (`"private": true`)
- Config-only packages (`@cfxdevkit/biome-config`, `@cfxdevkit/tsconfig`, etc.)
- Packages without `dist/` directory
- Application packages without exports

### Documentation Generation

#### README Skeleton (`api/readme.ts`)

```typescript
export function renderReadmeSkeleton(
  pkg: Pick<PublicPackageInfo, 'name' | 'description' | 'rel' | 'subpaths'>,
): string;

export function backfillReadmeSections(
  existing: string,
  pkg: Pick<PublicPackageInfo, 'name' | 'description' | 'rel' | 'subpaths'>,
): string;
```

**Hash Tracking:** Embeds `` to detect skeleton changes and skip LLM enrichment when stable.

#### API.md Rendering (`api/render.ts`)

```typescript
export function renderApiMd(
  packageName: string,
  description: string,
  sections: SubpathSection[],
): string;
```

Groups exports by kind and generates TypeScript code blocks for each subpath.

#### Structure.md Rendering (`api/structure.ts`)

```typescript
export async function walkStructureTree(
  dir: string,
  prefix = '',
  depth = 0,
  maxDepth = 3,
): Promise<string[]>;

export function renderStructureSkeleton(
  pkg: StructurePackageInfo,
  treeLines: readonly string[],
): string;
```

**Hash Tracking:** Uses `structure-hash` and `structure-status` markers to track deterministic vs. enriched documents.

## Check Types

### Architecture Check (`checks/arch.ts`)

Validates package tier relationships and source file rules.

```typescript
export async function runArchCheck(): Promise<ArchCheckResult>;
```

**Validations:**
- **Tier Policy:** Enforces no upward imports across tier boundaries
- **CLI Framework:** Requires `clipanion` + `@inquirer/prompts` for tooling CLI
- **Required Files:** `moon.yml`, `src/index.ts`
- **Source Rules:** No `@ts-nocheck`, no JS source files, file size limits (300 lines)

```typescript
export type ArchCheckResult = {
  status: 'ok' | 'error';
  lifecycle: string;
  packageCount: number;
  findings: Finding[];
};
```

### Documentation Check (`checks/docs.ts`)

```typescript
export async function runDocsCheck(opts: { silent?: boolean }): Promise<AgentSummary>;
```

**Validations:**
- **API.md Staleness:** Compares embedded hash with current `.d.ts` contents
- **Package Exports:** Verifies `package.json` exports match `vite.config.ts` entries
- **Moon Registration:** Ensures packages with `moon.yml` are registered in workspace
- **Path References:** Detects broken markdown links
- **Document Contracts:** Validates README/STRUCTURE.md sections

### Secrets Check (`checks/secrets.ts`)

```typescript
export async function runSecretsCheck(): Promise<SecretScanResult>;
```

**Scanned Patterns:**
- VS Code workspace state persistence with secrets
- Console/output channel secret exposure
- Hardcoded API keys, private keys, JWTs
- Recovery mnemonic labels

**Exclusions:** Test files (`.test.`, `.spec.`, `__tests__/`)

### Hotspots Check (`checks/hotspots.ts`)

Identifies high-churn, high-line-count files.

```typescript
export async function runHotspotsCheck(options: HotspotOptions): Promise<HotspotReport>;

export type HotspotOptions = {
  softLimit: number;      // Default: 250
  hardLimit: number;      // Default: 300
  since: string;          // Default: '90 days ago'
  failOnHard: boolean;
  json: boolean;
};
```

**Hotspot Score Formula:**
```
score = lines + (addedLines + deletedLines) * 2 + commits * 25
```

**Output:**
- JSON report at `artifacts/llm/reports/code-hotspots.json`
- Markdown report with Mermaid charts at `artifacts/llm/reports/code-hotspots.md`

### CI/CD Check (`checks/ci.ts`)

```typescript
export async function runCiCheck(opts: { silent?: boolean }): Promise<AgentSummary>;
```

**Validations:**
- Required workflow files (build, deploy, release, changeset)
- Docker image publishing to GHCR
- VPS deployment configuration
- Ansible infrastructure files

### Corpus Check (`checks/corpus.ts`)

Indexes documentation for LLM context.

```typescript
export async function runCorpusCheck(opts: { silent?: boolean }): Promise<AgentSummary>;
```

**Output:**
- `corpus/files.jsonl` - File metadata with SHA-256
- `corpus/chunks.jsonl` - Chunked source files
- `corpus/docs-index.jsonl` - Heading structure
- `corpus/manifest.json` - Summary statistics

### Unit Configs Check (`checks/unit-configs.ts`)

Validates monorepo unit configuration alignment.

```typescript
export async function runUnitConfigsCheck(
  opts: { json?: boolean; write?: boolean; failOnDrift?: boolean }
): Promise<UnitConfigReport>;
```

**Monorepo Units:** `delivery`, `implementation`, `operations`

### Full Report (`checks/report.ts`)

Aggregates all checks into a single status report.

```typescript
export async function runFullReport(): Promise<FullReport>;
```

**Includes:** Architecture, secrets, hotspots, kebab-groups, unit-configs, docs, CI, corpus

## CLI Entry Points

### Main Architecture Check

```bash
pnpm run arch-check
```

Runs `runArchCheck()` and exits with code 1 if errors found.

### Individual Checks

```bash

pnpm run check:hotspots  # Code hotspots
pnpm run check:secrets   # Secret scanning
```

**Structured Output:**

## Usage Examples

### Running a Specific Check

```typescript
import { runDocsCheck } from '@cfxdevkit/arch-check';

const result = await runDocsCheck({ silent: false });
console.log(`Docs check: ${result.status} (${result.findings} findings)`);
```

### Generating API Documentation

```typescript
import { discoverPublicPackages } from '@cfxdevkit/arch-check/api/filter';
import { extractSubpathExports } from '@cfxdevkit/arch-check/api/extract';
import { renderApiMd } from '@cfxdevkit/arch-check/api/render';
import { computeApiHash, embedHash } from '@cfxdevkit/arch-check/api/hash';

const packages = await discoverPublicPackages();
for (const pkg of packages) {
  const allExports: ExtractedExport[] = [];
  for (const [, dtsRelative] of Object.entries(pkg.subpaths)) {
    const exports = await extractSubpathExports(pkg.distDir, dtsRelative);
    allExports.push(...exports);
  }
  
  const md = renderApiMd(pkg.name, pkg.description, /* sections */);
  const hash = computeApiHash(allExports.map(e => e.signature));
  const final = embedHash(md, hash);
  
  await writeFile(pkg.apiMdPath, final, 'utf8');
}
```

### Custom Hotspot Analysis

```typescript
import { runHotspotsCheck } from '@cfxdevkit/arch-check';

const report = await runHotspotsCheck({
  softLimit: 200,
  hardLimit: 250,
  since: '30 days ago',
  failOnHard: true,
  json: false,
});

console.log(`Scanned ${report.totals.scannedFiles} files`);
console.log(`Hard violations: ${report.totals.hardViolations}`);
console.log(`Soft warnings: ${report.totals.softWarnings}`);
```

## Contribution Guidelines

### Adding New Checks

1. Create `src/checks/<name>.ts` with a `run<Name>Check()` function
2. Return `AgentSummary` or specific result type
3. Add to `runFullReport()` in `checks/report.ts`
4. Update `src/index.ts` exports
5. Add CLI entry in `src/bin/check-<name>.ts`

### Adding New API Utilities

1. Create module in `src/api/<name>.ts`
2. Export types and functions from `src/index.ts`
3. Add tests in `<name>.test.ts`
4. Document usage in this documentation

### Hash Stability

All generated documents use embedded SHA-256 hashes:
- `api-hash` for API.md
- `readme-hash` for README.md
- `structure-hash` + `structure-status` for STRUCTURE.md

This enables LLM agents to skip regeneration when skeletons are stable.
