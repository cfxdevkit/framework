# LLM Agent Workflows

## Overview

The LLM Agent Workflows module orchestrates automated repository operations using large language models. It provides three primary agent types:

1. **Review Agent** - Analyzes changed files for quality issues
2. **Check Agent** - Validates repository state and generates OpenSpec artifacts
3. **Commit Agent** - Manages commit workflow with quality gates and changeset generation

All agents integrate with execution context resolution, LLM completion providers, and repository validation tools (CDK repo-check, GitNexus).

## Core Agent Types

### Review Agent (`runReviewAgent`)

Analyzes git changes for quality issues without requiring an LLM.

**Entry Point:** `runReviewAgent(opts: { silent?: boolean })`

**Workflow:**
1. Resolve execution context (LLM not used)
2. Collect git changed files
3. Check for generated/artifact paths
4. Check for security-sensitive files
5. Run code hotspot analysis
6. Suggest validation commands
7. Write reports and print summary

**Findings Categories:**
- Generated/artifact paths changed
- Security-sensitive surfaces modified
- Code files exceeding size budgets (hard: 300 lines, soft: 250 lines)

**Reports:**
- `reports/review.json` - JSON report
- `reports/review.md` - Markdown report

### Check Agent (`runAgentCheck`)

Validates repository state and generates OpenSpec remediation artifacts.

**Entry Point:** `runAgentCheck(rawArgs: readonly string[], opts: { silent?: boolean })`

**Workflow:**

**Flags:**
| Flag | Description |
|------|-------------|
| `--quick` | Reduced token budgets |
| `--dry-run` | No file writes |
| `--create-branch` | Create git branch |
| `--draft-pr` | Create draft PR (implies `--create-branch`) |

**OpenSpec Artifacts Generated:**
- `proposal.md` - Change proposal
- `design.md` - Implementation design
- `specs/{name}/spec.md` - Specification files
- `tasks.md` - Implementation tasks

### Commit Agent (`runCommitWorkflow`)

Manages full commit workflow with quality gates, changeset generation, and commit message creation.

**Entry Point:** `runCommitWorkflow(args, options: CommitWorkflowOptions)`

**Workflow Steps:**
1. **Incremental validation sequence** - Quality gates (format, lint, typecheck, tests, hotspots, kebab-groups, repo check)
2. **Repository policy gates** - Hotspots, kebab-groups, repo check
3. **Preflight checks** - GitNexus ensure, git status/diff collection
4. **Scope detection** - Identify changed scopes
5. **Changeset plan** - Generate changeset if publishable packages changed
6. **Commit message generation** - LLM-generated conventional commit
7. **Approval** - Interactive or `--yes` flag
8. **Post-checks** - Validate changeset generation didn't break gates
9. **Commit** - Stage files and create commit

**Quality Gates:**
```typescript
const QUALITY_GATE_SPECS = [
  { id: 'gitnexus-analyze', label: 'GitNexus analyze', required: true },
  { id: 'format', label: 'Format', required: true },
  { id: 'lint', label: 'Lint', required: true },
  { id: 'typecheck', label: 'Typecheck', required: true },
  { id: 'test', label: 'Test', required: false },
  { id: 'build', label: 'Build', required: false },
  { id: 'hotspots', label: 'Hotspots', required: true },
  { id: 'kebab-groups', label: 'Kebab groups', required: false },
  { id: 'check', label: 'Repo check', required: true },
];
```

**Commit Flags:**
| Flag | Description |
|------|-------------|
| `--model <name>` | Override LLM model |
| `--quick` | Reduced token budgets |
| `--dry-run` | Preview only, no changes |
| `--yes` / `-y` | Auto-approve commit |
| `--force` / `-f` | Skip gate blocking |
| `--skip-checks` | Skip quality gates |
| `--skip-post-checks` | Skip post-generation validation |
| `--skip-changeset` | Skip changeset generation |
| `--changeset-bump <level>` | Force bump level (patch/minor/major) |
| `--with-tests` | Include test gate (default) |
| `--skip-tests` | Exclude test gate |

## Runtime Utilities

### Execution Context

Resolved via `resolveExecutionContext({ useLlm, action, modelOverride })`:

```typescript
{
  unit: { name?: string; rootDir?: string } | null;
  llm: {
    status: 'not-used' | 'ready' | 'unavailable';
    provider?: string;
    model?: string | null;
    baseUrl?: string | null;
    error?: string;
    configPath?: string;
  };
}
```

### Report Writing

```typescript
// JSON reports
await writeJsonReport('reports/agent-check.json', report);

// Markdown reports
await writeMarkdownReport('reports/agent-check.md', content);

// JSONL logs
await writeJsonl('logs/events.jsonl', records);
```

### Corpus Collection

Collects files for LLM context:

```typescript
const files = await collectCorpusFiles();
// Filters: generated paths, secrets, size > 512KB, non-text extensions
```

### Path Utilities

```typescript
isSecuritySensitive(file: string): boolean;  // keystore, wallet, secret, etc.
isGeneratedPath(file: string): boolean;      // artifacts/, dist/, coverage/, node_modules/
tierForPath(path: string): string;           // core, keys, solidity, ui, domain, tools, project, etc.
packageOwner(path: string): string;          // repos/cfx-tools/packages/name
languageForPath(path: string): string;       // typescript, javascript, markdown, etc.
```

## Failure Analysis

When quality gates fail, the system can provide LLM-based analysis:

```typescript
const analysis = await analyzeGateFailures({
  command: 'commit' | 'precommit',
  executionContext,
  reports: [qualityReport, policyReport],
  modelOverride?: string,
});
// Returns: { attempted, usedLlm, status, content, error? }
```

**Analysis Sections:**
- Summary
- Root causes
- Minimal fixes
- Next commands

## Smoke Testing

Test LLM model viability before committing to workflows:

```typescript
await runAgentSmoke(['--quick', '--no-thinking']);
```

**Probes:**
1. **Cold** - Model load time
2. **Hot** - Warm model response
3. **Task+Think** - Tier-appropriate JSON with thinking
4. **Task-NoThink** - Same without thinking mode

**Tier Probes:**
| Tier | Label | Actions |
|------|-------|---------|
| 1 | Lightweight/structured output | validation |
| 2 | Documentation generation | docs-api, readme-upkeep, structure-upkeep |
| 3 | Reasoning/code review | review, commit, changeset, release-readiness, ci-cd |

## Error Handling

### LLM Response Validation

All LLM responses are validated and retried on failure:

```typescript
try {
  return normalizeAgentCheckPlan(response.content);
} catch {
  const retry = await completeStructuredAgent({
    systemPrompt: `${basePrompt} The previous response was invalid. Return exactly one compact JSON object.`,
    userPrompt: [
      'Previous invalid response excerpt:',
      response.content.slice(0, 1200),
      userPrompt.slice(0, 8000),
    ].join('\n'),
  });
  return normalizeAgentCheckPlan(retry.content);
}
```

### Gate Failure States

```typescript
status: 'ok' | 'warning' | 'error' | 'skipped';
required: boolean;  // If required && error, workflow blocks
```

### Commit Workflow States

```typescript
status: 'committed' | 'dry-run' | 'approval-required' | 'aborted' | 'blocked';
phase: 'quality-gates' | 'repository-policy-gates' | 'approval' | 'post-checks' | 'completed';
blockedBy?: 'quality-gates' | 'repository-policy' | 'post-checks';
```

## Contributing

### Adding New Agents

1. Create agent file in `workers/agents/`
2. Export from `workers/agents/all.ts`
3. Add to `runAll()` orchestration
4. Implement report writing via `writeJsonReport`/`writeMarkdownReport`
5. Add test coverage in `workers/agents/*.test.ts`

### Adding New Gates

1. Extend `QUALITY_GATE_SPECS` in `workers/shared/index.ts`
2. Implement gate execution in `workers/commit/gates.ts`
3. Add deterministic hints in `workers/commit/gate-output.ts`
4. Update terminal UI in `workers/commit/terminal-ui.ts`

### Testing
