# LLM Automation & Tooling (`llm-tools`)

## Overview

The `llm-tools` package provides the core infrastructure for Large Language Model (LLM) automation within the `cfx-llm` ecosystem. It serves as a unified library and worker suite for executing AI-driven tasks, ranging from code review and documentation generation to commit validation and test baseline management.

The module is divided into two primary subsystems:
1.  **Agents:** Specialized background workers for domain-specific tasks (corpus, docs, review, evaluation).
2.  **Lemonade:** A CLI-driven workflow engine for development lifecycle automation (commits, completions, docs, tests).

## Core Execution

The package entry points reside in `src/`. These modules initialize the runtime environment and dispatch tasks to the appropriate workers.

*   **`src/index.ts`**: Primary export point for library consumers.
*   **`src/run.ts`**: Execution orchestrator for running agent tasks and CLI commands.

## Agent Workers

The `workers/agents/` directory contains specialized agents designed for long-running or background tasks. These agents handle specific domains of LLM interaction.

### Domain Agents
*   **`corpus.ts`**: Manages corpus ingestion and processing for RAG (Retrieval-Augmented Generation) contexts.
*   **`docs.ts`**: Handles documentation-specific agent tasks.
*   **`review.ts`**: Executes code review logic using LLM analysis.
*   **`eval-serve.ts`**: Serves evaluation endpoints for model performance testing.
*   **`all.ts`**: Aggregates or registers all available agents.

### Runtime Utilities
Located in `workers/agents/runtime/`, these modules provide shared infrastructure for agent execution:
*   **`checks.ts`**: Runtime validation and health checks.
*   **`models.ts`**: Model configuration and selection logic.
*   **`paths.ts`**: File system path resolution for agent artifacts.
*   **`reports.ts`**: Report generation and formatting for agent outputs.
*   **`corpus.ts`**, **`docs.ts`**: Runtime-specific implementations for domain agents.

## Lemonade CLI Engine

The `workers/lemonade/` directory implements the "Lemonade" CLI tool, which automates developer workflows. It is structured around specific lifecycle stages.

### Command Interface
*   **`cli.ts`**: Main CLI entry point and argument parsing.
*   **`commands.ts`**: Command registration and routing.
*   **`help.ts`**: Help text generation and documentation.
*   **`shared/logging.ts`**: Unified logging configuration for CLI output.

### Commit Workflow
The `commit/` subdirectory enforces commit standards and changelog management.
*   **`validate.ts`**: Validates commit message format against project standards.
*   **`message.ts`**: Generates or formats commit messages.
*   **`scope.ts`**: Manages commit scope definitions.
*   **`changelog.ts`**: Generates changelog entries from commit history.
*   **`gates.ts`**: Implements pre-commit or CI gates for commit validation.

### Completion Engine
The `completion/` subdirectory handles LLM inference requests and response formatting.
*   **`runner.ts`**: Orchestrates completion execution.
*   **`client.ts`**: Manages connections to LLM providers.
*   **`complete.ts`**: Core completion logic.
*   **`context.ts`**: Context window management and prompt construction.
*   **`direct.ts`**: Direct completion mode (streaming/synchronous).
*   **`json.ts`**: JSON-mode completion handling for structured output.
*   **`pi-rpc.ts`**: Remote Procedure Call handling for completion services.

### Documentation & Testing
Modules for generating and managing project documentation and test suites.

**Documentation (`docs/`)**
*   **`discover.ts`**: Scans codebase for documentation opportunities.
*   **`generate.ts`**: Generates documentation content.
*   **`validate.ts`**: Checks documentation for accuracy or completeness.
*   **`write.ts`**: Writes generated content to the file system.

**Testing (`tests/`)**
*   **`baseline.ts`**: Manages test baseline snapshots.
*   **`discover.ts`**: Identifies areas requiring test coverage.
*   **`generate.ts`**: Generates test cases.
*   **`write.ts`**: Writes test files to the repository.

## Integration

To integrate `llm-tools` into a project:

1.  **Library Usage**: Import from `src/index.ts` to access agent runners programmatically.
2.  **CLI Usage**: Execute `lemonade` commands via `workers/lemonade/cli.ts` for workflow automation.
3.  **Agent Extension**: Add new agents to `workers/agents/` and register them in `all.ts`.
4.  **Runtime Configuration**: Utilize `workers/agents/runtime/` modules to configure model paths, checks, and reporting standards.
