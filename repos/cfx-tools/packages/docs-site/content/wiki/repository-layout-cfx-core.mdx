# Repository Layout — cfx-core

# cfx-core Repository

**Tier 0a — chain primitives.** This repository carves out stable, low-level chain primitives to decouple their release cadence from higher-level application logic. It serves as the foundation for the DevKit ecosystem, providing types, protocol bindings, execution primitives, and local development tooling.

## Overview

`cfx-core` is a monorepo containing seven distinct npm packages. These packages are designed to have a small dependency tree (primarily `viem`, `cive`, `@scure/*`) and a long support window.

### Design Goals
- **Stability:** Chain primitives change rarely; this repo allows other slices to pin stable versions.
- **Decoupling:** Separates core logic from `cfx-keys`, `cfx-ui`, `cfx-domain`, and `cfx-tools`.
- **Type Safety:** All packages publish typed entrypoints + sourcemaps.

### Dependency Boundaries
- **MUST NOT** depend on: `cfx-keys`, `cfx-ui`, `cfx-domain`, `cfx-tools`.
- **MAY** depend on: Third-party crypto libs (`@noble/*`, `@scure/*`, `viem`, `cive`).

## Package Details

### `@cfxdevkit/devnode`
**Purpose:** Local Conflux dev node lifecycle. Wraps `@xcfx/node` to bring up a dual-space (Core + eSpace) chain on deterministic ports with pre-funded genesis accounts.

**Scope:** Dev / test only — Node.js host required. Never bundled into a runtime application.

#### Key Features
- **Dual-Space Support:** Core Space (HTTP/WS) and eSpace (HTTP/WS) on default ports.
- **Pre-funded Accounts:** 10 genesis accounts derived from a BIP-44 standard branch.
- **Auto-mining:** Packs pending transactions into blocks on a timer.
- **Snapshot/Restore:** Supports test isolation via state snapshots.

#### Default Configuration
| Field | Value | Matches |
|-------|-------|---------|
| Core HTTP / WS | `12537` / `12536` | `coreSpaceLocal` |
| eSpace HTTP / WS | `8545` / `8546` | `espaceLocal` |
| Core Chain ID | `2029` | `coreSpaceLocal.id` |
| eSpace Chain ID | `2030` | `espaceLocal.id` |
| Genesis Accounts | `10` | BIP-44 standard branch |
| Initial Balance | `10_000` CFX | Hardcoded by `@xcfx/node` |
| Auto-miner Tick | `2000` ms | `mine({numTxs:1})` |

#### API Surface
```ts
import { createDevNode, DevNode } from '@cfxdevkit/devnode';

const node = createDevNode();
await node.start();

// Access URLs
node.urls.core    // http://127.0.0.1:12537
node.urls.espace  // http://127.0.0.1:8545

// Lifecycle
await node.stop();
await node.restart();
await node.mine(1);       // Advance empty blocks
await node.packMine();    // Pack pending txs into next block

// Mining Control
node.startMining(2000);   // Re-arm auto-miner
node.stopMining();        // Disarm auto-miner
```

#### CLI Usage
```sh
cfxdevkit-devnode --accounts 4 --balance 1000
cfxdevkit-devnode --help
```

#### Internal Structure
```
devnode/
├── lifecycle/      # start, stop, health, ports
├── genesis/        # deterministic genesis, funded accounts
├── snapshot/       # take, restore, store
├── docker/         # optional Docker driver
└── internal/       # child_process helpers
```

### `@cfxdevkit/executor`
**Purpose:** Generic execution primitives for keeper / off-chain automation systems.

**Scope:** No domain logic (strategies live in `@cfxdevkit/automation`).

#### Key Features
- **Job Queue:** Pluggable backends (memory, postgres, redis).
- **Retry Policy:** Exponential backoff with jitter.
- **Gas Awareness:** `gasPrice`, `maxFeePerGas`, `maxPriorityFeePerGas` in context.
- **Idempotency:** Support via `idempotencyKey`.
- **Distributed Locking:** `withLock` for critical sections.
- **Polling:** Periodic checks via `createPoller`.

#### API Surface
```ts
import { execute, createTaskQueue, withLock } from '@cfxdevkit/executor';

// Single execution with retry
const result = await execute(
  async (ctx) => { /* on-chain op */ },
  { maxAttempts: 3, idempotencyKey: 'tx-0x123' }
);

// Task Queue
const queue = createTaskQueue({ concurrency: 5 });
queue.start();
await queue.enqueue(async (ctx) => { /* task */ });

// Distributed Lock
const result = await withLock('dca:account-0x...', async () => {
  // Critical section
});
```

#### Internal Structure
```
executor/
├── job/            # Job lifecycle, context
├── queue/          # memory, postgres, redis adapters
├── scheduler/      # cron, interval triggers
├── policy/         # retry, circuit-breaker, rate-limit
├── gas/            # estimate, price-strategy, replace
└── observability/  # metrics, trace
```

### `@cfxdevkit/protocol`
**Purpose:** Conflux protocol helpers, precompile ABIs, and reusable DevKit contract metadata.

**Scope:** Pure Conflux protocol helpers. No RPC side effects.

#### Key Exports
- **Precompiles:** `AdminControl`, `SponsorWhitelist`, `Staking`, `CrossSpaceCall`, `PoSRegister`.
- **DevKit Contracts:** `AutomationManager`, `PermitHandler`, `SwappiPriceAdapter`.
- **WCFX:** ABI and address constants for wrap/unwrap flows.
- **Helpers:** `waitForTransactionReceipt`, `getChainProgress`, `estimateTransaction`.

#### API Surface
```ts
import { 
  crossSpaceCallAbi, 
  waitForTransactionReceipt, 
  getChainProgress 
} from '@cfxdevkit/protocol';

// Chain Progress
const progress = await getChainProgress(client);
// { epochNumber, epochIndex, blockNumber }

// Receipt Handling
const receipt = await waitForTransactionReceipt(client, hash, {
  timeout: 30_000,
  retryInterval: 1000
});
```

### `@cfxdevkit/testing`
**Purpose:** Shared test fixtures and matchers.

**Scope:** Dev-dependency only. Used by every package and project.

#### Key Features
- **Mock Client:** `createMockClient` (Viem-shaped).
- **DevNode Fixture:** `createDevNodeFixture` wraps `framework/devnode`.
- **Async Coordination:** `createDeferred`, `waitFor`.
- **Vitest Matchers:** `toBeHexHash`, `toBeHexAddress`, `toEmitEvent`.

#### API Surface
```ts
import { createMockClient, createDevNodeFixture } from '@cfxdevkit/testing';

// Mock Client
const client = createMockClient({ blockNumber: 1n });

// DevNode Fixture
const node = await createDevNodeFixture({
  startupTimeout: 10_000
});
```

### `@cfxdevkit/cdk` (core)
**Purpose:** Base types, errors, chains, units, client, and wallet primitives.

**Scope:** Foundation for all other packages.

**Exports:** Types, errors, chains, units, client, wallet primitives.

### `@cfxdevkit/contracts` & `@cfxdevkit/compiler`
- **contracts:** Shared ABIs + deployments.
- **compiler:** solc + revive bridge.

## Development Guidelines

### Building
All packages use `vite` for node-only builds and `moon` for task orchestration.

```sh
pnpm --filter @cfxdevkit/devnode build
pnpm --filter @cfxdevkit/executor build
```

### Testing
Tests rely on `@cfxdevkit/testing` for fixtures and `@cfxdevkit/devnode` for integration.

```sh
pnpm test --filter @cfxdevkit/protocol
```

### Contribution
1. **Boundaries:** Ensure no forbidden dependencies (`cfx-ui`, `cfx-domain`, etc.) are introduced.
2. **Types:** All public entrypoints must be typed.
3. **Sourcemaps:** Ensure sourcemaps are published for debugging.
4. **API Stability:** Changes to public exports in `core` or `protocol` require version bumping and migration notes.

### Sub-path Exports
Most packages support sub-path imports for tree-shaking and specific functionality:

```ts
// Example: Importing specific queue backend
import { createTaskQueue } from '@cfxdevkit/executor/queue/postgres';

// Example: Importing lifecycle helpers
import { start } from '@cfxdevkit/devnode/lifecycle';
```
