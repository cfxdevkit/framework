# DevNode & Runtime

# DevNode & Runtime

The **DevNode & Runtime** module provides infrastructure for local Conflux development and general execution utilities. It consists of two packages:

- **`@cfxdevkit/devnode`** — Spawns and manages local Conflux nodes for testing and development
- **`@cfxdevkit/executor`** — Generic execution primitives (retry, batching, polling, locking)

---

## @cfxdevkit/devnode

Spawns a local Conflux node with dual-space support (Core + eSpace) for development and testing. Wraps the native `@xcfx/node` binary to provide deterministic ports, pre-funded genesis accounts, and configurable auto-mining.

### Quick Start

```typescript
import { createDevNode } from '@cfxdevkit/devnode';

const node = createDevNode({
  accounts: 5,
  balanceCfx: '1000000',
  miningIntervalMs: 2000,
});

await node.start();
console.log(node.urls.core);   // http://127.0.0.1:12537
console.log(node.urls.espace); // http://127.0.0.1:8545

// Mine transactions
await node.packMine();

await node.stop();
```

### CLI Usage

The package ships a CLI for standalone node spawning:

```bash
cfxdevkit-devnode
cfxdevkit-devnode --accounts 4 --balance 1000 --logging
cfxdevkit-devnode --no-mining
```

| Flag | Default | Description |
|------|---------|-------------|
| `--core-port` | `12537` | Core JSON-RPC HTTP port |
| `--espace-port` | `8545` | eSpace JSON-RPC HTTP port |
| `--core-ws-port` | `12536` | Core WebSocket port |
| `--espace-ws-port` | `8546` | eSpace WebSocket port |
| `--chain-id` | `2029` | Core Space chain ID |
| `--evm-chain-id` | `2030` | eSpace chain ID |
| `--accounts` | `10` | Pre-funded genesis accounts |
| `--balance` | `1000000` | Initial CFX per account |
| `--mining-interval` | `2000` | Auto-miner tick (ms) |
| `--no-mining` | — | Disable auto-miner |
| `--mnemonic` | — | Override BIP-39 mnemonic |
| `--data-dir` | — | Override data directory |
| `--logging` | `false` | Forward node logs to stdout |

### Core Lifecycle

The `DevNode` class manages node lifecycle through explicit state transitions:

```typescript
type DevNodeStatus = 'stopped' | 'starting' | 'running' | 'stopping' | 'error';
```

**Start flow:**
1. `createDevNode()` — Construct with optional config
2. `node.start()` — Spawn `@xcfx/node` server, start auto-miner if configured
3. Node listens on deterministic ports (Core: `12537`, eSpace: `8545`)
4. Pre-funded accounts available at `node.accounts` and `node.faucet`

**Stop flow:**
1. `node.stop()` — Stop auto-miner, shut down server
2. Graceful cleanup on `SIGINT`/`SIGTERM` (CLI mode)

### Configuration

```typescript
interface DevNodeConfig {
  chainId?: number;           // Default: 2029
  evmChainId?: number;        // Default: 2030
  coreRpcPort?: number;       // Default: 12537
  evmRpcPort?: number;        // Default: 8545
  coreWsPort?: number;        // Default: 12536
  evmWsPort?: number;         // Default: 8546
  dataDir?: string;           // Default: ~/.cfxdevkit/devnode/<random>
  mnemonic?: string;          // Random BIP-39 by default
  accounts?: number;          // Default: 10
  balanceCfx?: string;        // Default: '10000'
  miningIntervalMs?: number;  // Default: 2000, 0 to disable
  logging?: boolean;          // Default: false
}
```

### Accounts & Faucet

Each dev node provides:
- **`node.accounts`** — Array of pre-funded genesis accounts (index 0-9 by default)
- **`node.faucet`** — Dedicated mining/faucet account

```typescript
interface DevNodeAccount {
  index: number;
  evmAddress: string;
  coreAddress: string;
  publicKey: string;
  privateKey: string;
  paths: { core: string; evm: string };
  initialBalanceCfx: string;
}
```

Accounts are derived from the configured mnemonic using `deriveDualAccount()` from `@cfxdevkit/cdk`.

### Mining Control

```typescript
// Manual block mining (empty Core blocks)
await node.mine(1);

// Pack pending transactions into next block
await node.packMine();

// Start/stop auto-miner
await node.startMining(2000);  // Mine every 2000ms
await node.stopMining();

// Check mining status
const status = node.getMiningStatus();
// { enabled: true, intervalMs: 2000, ticks: 42, startedAt: Date }
```

Auto-miner uses `cive` to call `mine({ numTxs: 1 })` at the configured interval. Concurrent calls are serialized to prevent container lockups.

### Error Handling

All operations throw `DevNodeError` with specific codes:

```typescript
type DevNodeErrorCode =
  | 'devnode/already-running'
  | 'devnode/not-running'
  | 'devnode/start-failed'
  | 'devnode/stop-failed'
  | 'devnode/mining-already-running'
  | 'devnode/mining-not-running'
  | 'devnode/invalid-config';
```

```typescript
class DevNodeError extends CfxError {
  readonly code: DevNodeErrorCode;
  readonly cause?: unknown;
  readonly meta?: Record<string, unknown>;
}
```

### Integration with viem/ethers

Defaults match `@cfxdevkit/cdk/chains`'s `coreSpaceLocal` and `espaceLocal`:

```typescript
import { createClient, http } from 'viem';
import { coreSpaceLocal, espaceLocal } from '@cfxdevkit/cdk/chains';

const node = createDevNode();
await node.start();

const coreClient = createClient({
  chain: coreSpaceLocal,
  transport: http(node.urls.core),
});

const espaceClient = createClient({
  chain: espaceLocal,
  transport: http(node.urls.espace),
});
```

## @cfxdevkit/executor

Generic execution primitives for retry logic, batching, polling, and concurrency control.

### Execution with Retry

```typescript
import { execute } from '@cfxdevkit/executor';

const result = await execute(
  async ({ attempt }) => {
    // task runs with attempt context
    return await fetch('https://api.example.com');
  },
  {
    attempts: 3,
    delayMs: 1000,
    timeoutMs: 5000,
    shouldRetry: (error) => error instanceof NetworkError,
  },
);

if (result.ok) {
  console.log(result.value);  // { ok: true, value: ..., attempts: 1, durationMs: 120 }
} else {
  console.error(result.error); // { ok: false, error: ..., attempts: 3, durationMs: 5100 }
}
```

### Batch Execution

```typescript
import { executeBatch, createTaskQueue } from '@cfxdevkit/executor';

// Parallel batch with concurrency limit
const results = await executeBatch(
  tasks.map((t) => async () => t()),
  { concurrency: 4, stopOnError: true },
);

// Task queue for sequential processing
const queue = createTaskQueue({ concurrency: 1, onResult: (r, i) => { /* ... */ } });
queue.add(async () => { /* task 1 */ });
queue.add(async () => { /* task 2 */ });
await queue.run();
```

### Polling

```typescript
import { createPoller } from '@cfxdevkit/executor';

const poller = createPoller(
  async ({ signal }) => {
    if (signal.aborted) return;
    await checkCondition();
  },
  1000, // interval in ms
);

poller.start();
// ...
await poller.stop();
```

### Concurrency Locking

```typescript
import { withLock } from '@cfxdevkit/executor';

// Ensures only one task runs per key at a time
await withLock('my-resource', async () => {
  await criticalSection();
});
```

### API Reference

| Function | Description |
|----------|-------------|
| `execute<T>(task, options)` | Execute single task with retry, timeout, and abort support |
| `executeBatch<T>(tasks, options)` | Execute array of tasks with configurable concurrency |
| `createTaskQueue(options)` | Create FIFO task queue with batch execution |
| `createPoller(task, intervalMs)` | Create polling loop with abort support |
| `withLock<T>(key, task)` | Execute task with exclusive access per key |

### Types

```typescript
interface ExecutionContext {
  signal?: AbortSignal;
  attempt: number;
  metadata?: Record<string, unknown>;
}

type ExecutionTask<T> = (context: ExecutionContext) => Promise<T> | T;

interface RetryPolicy {
  attempts?: number;
  delayMs?: number | ((attempt: number, error: unknown) => number);
  shouldRetry?: (error: unknown, attempt: number) => boolean;
}

interface ExecutionResult<T> {
  ok: true;
  value: T;
  attempts: number;
  durationMs: number;
} | {
  ok: false;
  error: unknown;
  attempts: number;
  durationMs: number;
};
```

## Notes

- **Node.js only** — `@xcfx/node` binary does not resolve in browser environments
- **Deterministic defaults** — Ports and chain IDs match `@cfxdevkit/cdk/chains` for seamless integration
- **Cleanup required** — Always call `node.stop()` to release ports and resources
- **Auto-miner safety** — Concurrent `packMine()` and auto-mine ticks are serialized to prevent lockups
