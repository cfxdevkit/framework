# DevNode & Runtime Module

The `@cfxdevkit/devnode` module provides a minimal, Node.js-only interface to spawn and manage a local Conflux node for development and testing. It wraps the native `@xcfx/node` binary to offer a dual-space (Core + eSpace) chain with pre-funded genesis accounts, deterministic ports, and optional auto-mining.

## Purpose

This module is intended for **development and testing only**. It allows developers to quickly spin up a local Conflux node with:
- Core Space and eSpace JSON-RPC endpoints
- Pre-funded genesis accounts
- Auto-mining capability
- Deterministic configuration matching `@cfxdevkit/cdk`'s local chain definitions

## Key Components

### `DevNode` Class

The core of the module is the `DevNode` class, which encapsulates the lifecycle of a local Conflux node.

#### Properties

- `config`: Resolved configuration with defaults applied.
- `accounts`: Array of pre-funded genesis accounts.
- `faucet`: Dedicated mining/faucet account.
- `urls`: RPC endpoint URLs for Core and eSpace.

#### Methods

- `start()`: Initializes and starts the node.
- `stop()`: Stops the node and auto-miner.
- `restart()`: Convenience method to stop and start the node.
- `mine(blocks?: number)`: Mines empty Core blocks.
- `packMine()`: Packs pending transactions into the next block.
- `startMining(intervalMs?: number)`: Begins auto-mining.
- `stopMining()`: Stops the auto-miner.
- `getStatus()`: Returns the current lifecycle status.
- `isRunning()`: Checks if the node is running.
- `getMiningStatus()`: Returns the auto-miner's status.

### `createDevNode(config: DevNodeConfig)`

Factory function to instantiate a `DevNode` with the given configuration.

### CLI Interface

The module includes a CLI (`cfxdevkit-devnode`) for quick node startup from the command line.

#### Usage

```bash
cfxdevkit-devnode                 # default ports + 10 funded accounts
cfxdevkit-devnode --accounts 4 --balance 1000 --logging
cfxdevkit-devnode --no-mining     # disable auto-miner
```

#### Options

| Option | Description |
|--------|-------------|
| `--core-port` | Core JSON-RPC HTTP port (default 12537) |
| `--espace-port` | eSpace JSON-RPC HTTP port (default 8545) |
| `--core-ws-port` | Core JSON-RPC WS port (default 12536) |
| `--espace-ws-port` | eSpace JSON-RPC WS port (default 8546) |
| `--chain-id` | Core chain id (default 2029) |
| `--evm-chain-id` | eSpace chain id (default 2030) |
| `--accounts` | Pre-funded genesis accounts (default 10) |
| `--balance` | Initial CFX balance per account (default 1000000) |
| `--mining-interval` | Auto-miner tick (default 2000, set 0 to disable) |
| `--no-mining` | Shortcut for `--mining-interval 0` |
| `--mnemonic` | Override the random BIP-39 mnemonic |
| `--data-dir` | Override the data directory |
| `--logging` | Forward @xcfx/node logs to stdout |
| `-h, --help` | Show help |

## Configuration

The `DevNodeConfig` interface defines the configuration options for the node.

```ts
interface DevNodeConfig {
  chainId?: number;           // Core chain id (default 2029)
  evmChainId?: number;        // eSpace chain id (default 2030)
  coreRpcPort?: number;       // Core JSON-RPC HTTP port (default 12537)
  evmRpcPort?: number;        // eSpace JSON-RPC HTTP port (default 8545)
  coreWsPort?: number;        // Core JSON-RPC WS port (default 12536)
  evmWsPort?: number;         // eSpace JSON-RPC WS port (default 8546)
  dataDir?: string;           // Data directory (default ~/.cfxdevkit/devnode/<random>)
  mnemonic?: string;          // BIP-39 mnemonic (random by default)
  accounts?: number;          // Number of pre-funded accounts (default 10)
  balanceCfx?: string;        // Initial balance per account (display only)
  miningIntervalMs?: number;  // Auto-miner interval in ms (default 2000)
  logging?: boolean;          // Forward logs to stdout (default false)
}
```

## Error Handling

The module throws `DevNodeError` for various failure conditions, such as:
- Node already running
- Node not running
- Start/stop failures
- Invalid configuration
- Mining already/already not running

## Integration with Other Modules

### `@xcfx/node`

The `DevNode` class uses `@xcfx/node` to create and manage the actual Conflux node server.

### `cive`

Used to create a test client for mining operations.

### `@cfxdevkit/cdk`

The default ports and chain IDs are aligned with `@cfxdevkit/cdk`'s local chain definitions (`coreSpaceLocal`, `espaceLocal`).

## Lifecycle Management

The `DevNode` class manages its lifecycle through the following states:
1. `stopped`: Node is not running.
2. `starting`: Node is initializing.
3. `running`: Node is active.
4. `stopping`: Node is shutting down.
5. `error`: Node encountered an error.

## Usage Example

```ts
import { createDevNode } from '@cfxdevkit/devnode';

const node = createDevNode({
  accounts: 5,
  balanceCfx: '1000',
  miningIntervalMs: 1000,
});

await node.start();

console.log('Core RPC:', node.urls.core);
console.log('eSpace RPC:', node.urls.espace);

// Mine a block
await node.mine(1);

// Pack and mine pending transactions
await node.packMine();

// Stop the node
await node.stop();
```

## CLI Usage Example

```bash
# Start a dev node with custom settings
cfxdevkit-devnode --accounts 3 --balance 5000 --mining-interval 500

# Start without auto-mining
cfxdevkit-devnode --no-mining
```

## Notes

- The module is **Node.js-only** and cannot be used in browser environments.
- The `@xcfx/node` binary is required and must be available in the environment.
- The default configuration matches `@cfxdevkit/cdk`'s local chain definitions for seamless integration.
- Auto-mining is enabled by default with a 2000ms interval.
- The mnemonic is randomly generated unless explicitly provided.
- The data directory is automatically created if it doesn't exist.
