# DevNode & Runtime

# DevNode & Runtime Module

The **DevNode & Runtime** module provides a minimal, self-contained local Conflux node for development and testing. It wraps `@xcfx/node` to spawn a fully functional Core + eSpace node with pre-funded genesis accounts, auto-mining, and programmatic control over mining behavior.

This module is used by:
- CLI (`cfxdevkit-devnode`)
- VS Code extension (via `src/devnode/manager.ts`)
- Integration tests (`compiler/src/integration.test.ts`)
- Local showcase apps (`pnpm exec moon run :test`)

---

## Core Purpose

The DevNode module enables:
- **Instant local node startup** with sensible defaults matching `core-local` / `espace-local`
- **Deterministic genesis** via configurable mnemonic or auto-generated BIP-39 phrase
- **Pre-funded accounts** (Core + eSpace addresses) derived from the mnemonic
- **Auto-mining** with configurable interval, or manual mining via RPC
- **Clean lifecycle management**: `start()`, `stop()`, `restart()`, `mine()`, `packMine()`

It is *not* intended for production use — it trades security and persistence for simplicity and speed.

## Key Components

### 1. `DevNode` Class (`src/node.ts`)

The central abstraction. Manages the lifecycle of the underlying `@xcfx/node` server and auto-miner.

#### Public API

| Method | Description |
|--------|-------------|
| `new DevNode(config?)` | Instantiate with optional config (defaults applied in `resolveConfig`) |
| `urls` | Read-only `DevNodeUrls` (`core`, `espace`, `coreWs`, `espaceWs`) |
| `getStatus()` | Returns `'stopped' | 'starting' | 'running' | 'stopping' | 'error'` |
| `isRunning()` | `true` iff status is `'running'` |
| `getMiningStatus()` | Returns `{ enabled, intervalMs, ticks, startedAt? }` |
| `start()` | Starts the node and (if `miningIntervalMs > 0`) auto-miner |
| `stop()` | Stops auto-miner and shuts down the node |
| `restart()` | `stop()` + `start()` |
| `mine(blocks?)` | Mine `blocks` empty Core blocks (requires running node) |
| `packMine()` | Mine next block with pending Core + eSpace txs (1 tx per block) |
| `startMining(intervalMs?)` | Start/adjust auto-miner (requires running node) |
| `stopMining()` | Stop auto-miner (safe to call when not mining) |

#### Internal State

- `config`: Resolved config (all fields required)
- `accounts`: Array of `DevNodeAccount` (genesis accounts)
- `faucet`: Dedicated mining/funding account (derivation path `m/44'/60'/1'/0/0`)
- `server`: `XcfxServer` instance (from `@xcfx/node`)
- `testClient`: `CiveTestClient` (for mining RPCs)
- `mining`: `{ enabled, intervalMs, ticks, startedAt? }`
- `packing`: Boolean flag to prevent concurrent `mine()` calls during `packMine()`

### 2. Configuration (`src/types.ts`, `src/internals.ts`)

#### `DevNodeConfig` (user-facing, all optional)

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| `chainId` | `number` | `2029` | Core Space chain ID |
| `evmChainId` | `number` | `2030` | eSpace chain ID |
| `coreRpcPort` | `number` | `12537` | Core HTTP RPC |
| `evmRpcPort` | `number` | `8545` | eSpace HTTP RPC |
| `coreWsPort` | `number` | `12536` | Core WS RPC |
| `evmWsPort` | `number` | `8546` | eSpace WS RPC |
| `dataDir` | `string` | `~/.cfxdevkit/devnode/<random>` | Node data directory |
| `mnemonic` | `string` | auto-generated | BIP-39 mnemonic |
| `accounts` | `number` | `10` | Number of genesis accounts |
| `balanceCfx` | `string` | `"10000"` | *Display-only* balance per account |
| `miningIntervalMs` | `number` | `2000` | Auto-miner tick (0 = disabled) |
| `logging` | `boolean` | `false` | Forward `@xcfx/node` logs to stdout |

> ⚠️ `balanceCfx` is **not** used to set genesis balances — `@xcfx/node` hardcodes `10_000` CFX per `genesisSecrets` entry. This field is only for display.

#### `ResolvedDevNodeConfig` (internal, all required)

Generated by `resolveConfig()`:
- Applies defaults
- Validates mnemonic (throws `DevNodeError` if invalid)
- Generates `dataDir` if missing
- Ensures `miningIntervalMs >= 100` (else throws)

---

### 3. Account Generation (`src/internals.ts`)

#### `createAccounts(config)`

- Derives `accounts` (default 10) from path `m/44'/60'/0'/0/{i}`
- Derives `faucet` from path `m/44'/60'/1'/0/0`
- Each account includes:
  - `evmAddress`, `coreAddress`
  - `publicKey`, `privateKey`
  - `paths.evm`, `paths.core`
  - `initialBalanceCfx` (display-only)

Uses `@cfxdevkit/core`:
- `generateMnemonic(128)` → 12-word mnemonic
- `deriveAccount({ mnemonic, path })`
- `coreAddressFromPrivateKey(privateKey, chainId)`

### 4. Server Setup (`src/server.ts`)

#### `createXcfxServer(config, accounts, faucet)`

Builds `@xcfx/node` config:
```ts
{
  jsonrpcHttpPort: config.coreRpcPort,
  jsonrpcHttpEthPort: config.evmRpcPort,
  jsonrpcWsPort: config.coreWsPort,
  jsonrpcWsEthPort: config.evmWsPort,
  chainId: config.chainId,
  evmChainId: config.evmChainId,
  confluxDataDir: config.dataDir,
  genesisSecrets: [...accounts.map(a => a.privateKey), faucet.privateKey],
  genesisEvmSecrets: [...accounts.map(a => a.privateKey), faucet.privateKey],
  miningAuthor: faucet.coreAddress,
  devPackTxImmediately: false,
  log: config.logging,
  timeout: 60_000,
  retryInterval: 300,
}
```

> ⚠️ `devPackTxImmediately: false` ensures `packMine()` is needed to include pending txs.

#### `createMiningClient(urls)`

Creates a `cive` test client targeting Core Space:
```ts
cive.createTestClient({
  transport: cive.http(urls.core, { timeout: 60_000 }),
})
```

Used for:
- `mine({ blocks })` → empty blocks
- `mine({ numTxs: 1 })` → pack 1 tx per block

### 5. CLI (`src/cli.ts`)

Command-line entrypoint (`cfxdevkit-devnode`).

#### Usage

```bash
cfxdevkit-devnode                 # default config
cfxdevkit-devnode --accounts 4 --balance 1000 --logging
cfxdevkit-devnode --no-mining     # disable auto-miner
```

#### Flags

| Flag | Description |
|------|-------------|
| `--core-port`, `--espace-port`, `--core-ws-port`, `--espace-ws-port` | Override ports |
| `--chain-id`, `--evm-chain-id` | Override chain IDs |
| `--accounts`, `--balance` | Genesis account count & display balance |
| `--mining-interval`, `--no-mining` | Auto-miner control |
| `--mnemonic`, `--data-dir` | Advanced config |
| `--logging` | Enable `@xcfx/node` logs |
| `-h`, `--help` | Show help |

### 6. Error Handling (`src/errors.ts`)

All errors are `DevNodeError` instances (extends `CfxError`).

#### Error Codes

| Code | Description |
|------|-------------|
| `devnode/already-running` | `start()` called while running |
| `devnode/not-running` | Operation requires running node |
| `devnode/start-failed` | `start()` threw |
| `devnode/stop-failed` | `stop()` threw |
| `devnode/mining-already-running` | `startMining()` while mining |
| `devnode/mining-not-running` | `stopMining()` while not mining |
| `devnode/invalid-config` | Invalid mnemonic, interval < 100, etc. |

## Execution Flow: `start()`

```mermaid
flowchart TD
    A[start()] --> B{status check}
    B -->|running| E[throw already-running]
    B -->|ok| C[mkdir dataDir]
    C --> D[createXcfxServer]
    D --> F[server.start()]
    F --> G[status = running]
    G --> H{miningIntervalMs > 0?}
    H -->|yes| I[startMining()]
    H -->|no| J[skip miner]
    I --> K[requireTestClient]
    K --> L[setInterval → mine({ numTxs:1 })]
```

## Integration Points

### VS Code Extension (`vscode-extension/src/extension-node-helpers.ts`)

- `startNode()` → `createDevNode()` → `node.start()`
- `wipeNodeAndRestart()` → `stop()` → `start()`

### Integration Tests (`compiler/src/integration.test.ts`)

- `createDevNode()` → `node.start()` → deploy contracts → `node.stop()`

## Usage Examples

### CLI

```bash
# Start with 4 accounts, 1000 CFX each, no mining
cfxdevkit-devnode --accounts 4 --balance 1000 --no-mining

# Start with custom mnemonic (for reproducibility)
cfxdevkit-devnode --mnemonic "abandon ... abandon"
```

### Programmatic

```ts
import { createDevNode } from '@cfxdevkit/devnode';

const node = createDevNode({
  accounts: 2,
  balanceCfx: '5000',
  logging: true,
});

await node.start();
console.log(node.urls.core); // http://127.0.0.1:12537

// Mine 3 empty blocks
await node.mine(3);

// Pack pending txs
await node.packMine();

// Stop
await node.stop();
```

## Limitations & Gotchas

- **Genesis balance is fixed**: `@xcfx/node` ignores `balanceCfx` and uses `10_000` CFX per account.
- **No persistence**: `dataDir` is used for chain state, but mnemonic is *not* persisted — regenerate on restart unless `mnemonic` is set.
- **Auto-miner is Core-only**: eSpace blocks are packed only via `packMine()` (which mines Core first).
- **No transaction signing**: DevNode does *not* provide a wallet — accounts are for genesis only.

## Summary

The **DevNode & Runtime** module is the backbone of local Conflux development. It abstracts away the complexity of `@xcfx/node` and provides a clean, typed API for spinning up, controlling, and tearing down a local node — ideal for testing, demos, and tooling like the VS Code extension.
