# DevNode & Runtime

The `cfxdevkit-devnode` module provides a minimal, programmatic interface to spin up a local Conflux node for development and testing. It wraps `@xcfx/node` to manage the underlying server lifecycle, genesis configuration, and auto-mining capabilities.

The module defaults match `@cfxdevkit/cdk/chains` (`core-local` / `espace-local`), ensuring compatibility with standard tooling (e.g., `moon run :test`) immediately upon boot.

## Quick Start

### CLI Usage
The module exposes a CLI entry point (`cfxdevkit-devnode`) for quick local node startup.

```bash
# Start with defaults (10 accounts, ports 12537/8545)
cfxdevkit-devnode

# Custom configuration
cfxdevkit-devnode --accounts 4 --balance 1000 --logging

# Disable auto-mining
cfxdevkit-devnode --no-mining
```

### Programmatic Usage
For integration into test suites or custom tooling, use the `DevNode` class.

```typescript
import { createDevNode } from '@cfxdevkit/devnode';

const node = createDevNode({
  accounts: 5,
  miningIntervalMs: 2000,
  logging: true,
});

await node.start();
console.log(node.urls.core); // http://127.0.0.1:12537

// Manually mine a block
await node.mine(1);

await node.stop();
```

## Architecture

The module abstracts the underlying `@xcfx/node` server and `cive` test client into a unified `DevNode` class.

*   **CLI (`cli.ts`)**: Handles argument parsing and process lifecycle (SIGINT/SIGTERM).
*   **DevNode (`node.ts`)**: Manages state, lifecycle, mining timers, and exposes RPC URLs.
*   **Internals (`internals.ts`)**: Handles configuration resolution, account derivation, and defaults.
*   **Server (`server.ts`)**: Bridges `DevNode` to `@xcfx/node` (server) and `cive` (mining client).

## Configuration

Configuration is defined via `DevNodeConfig`. All fields are optional; defaults are applied in `resolveConfig`.

| Field | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `chainId` | `number` | `2029` | Conflux Core Space chain ID. |
| `evmChainId` | `number` | `2030` | eSpace chain ID. |
| `coreRpcPort` | `number` | `12537` | Core JSON-RPC HTTP port. |
| `evmRpcPort` | `number` | `8545` | eSpace JSON-RPC HTTP port. |
| `coreWsPort` | `number` | `12536` | Core JSON-RPC WS port. |
| `evmWsPort` | `number` | `8546` | eSpace JSON-RPC WS port. |
| `accounts` | `number` | `10` | Number of pre-funded genesis accounts. |
| `balanceCfx` | `string` | `"10000"` | Display balance per account (actual genesis funding is hardcoded in `@xcfx/node`). |
| `miningIntervalMs` | `number` | `500` | Auto-miner tick interval. Set `0` to disable. |
| `mnemonic` | `string` | `random` | BIP-39 mnemonic for account derivation. |
| `dataDir` | `string` | `~/.cfxdevkit/...` | Directory for node data storage. |
| `logging` | `boolean` | `false` | Forward `@xcfx/node` logs to stdout. |

## Core API: `DevNode`

The `DevNode` class is the primary handle for interacting with the local node.

### Lifecycle

*   **`createDevNode(config?: DevNodeConfig)`**: Constructs a new instance. Does not start the server.
*   **`start()`**: Boots the `@xcfx/node` server. Creates the data directory and initializes accounts.
    *   Throws `DevNodeError` with code `devnode/already-running` if called while running.
    *   Throws `DevNodeError` with code `devnode/start-failed` if the server fails to initialize.
*   **`stop()`**: Shuts down the server and stops the auto-miner.
    *   Safe to call when already stopped.
    *   Throws `DevNodeError` with code `devnode/stop-failed` on failure.
*   **`restart()`**: Convenience method calling `stop()` then `start()`.
*   **`getStatus()`**: Returns current `DevNodeStatus` (`'stopped'`, `'starting'`, `'running'`, `'stopping'`, `'error'`).
*   **`isRunning()`**: Boolean check for `'running'` status.

### Mining Control

The module supports both manual and automatic mining.

*   **`mine(blocks?: number)`**: Mines empty Core blocks. Requires the node to be running.
*   **`packMine()`**: Packs pending Core and eSpace transactions into the next block.
*   **`startMining(intervalMs?: number)`**: Enables the auto-miner.
    *   Runs a `setInterval` that calls `testClient.mine({ numTxs: 1 })`.
    *   Includes `autoMineInFlight` protection to prevent concurrent mine calls from locking up slow containers.
    *   Interval must be `>= 100ms`.
*   **`stopMining()`**: Disables the auto-miner.
*   **`getMiningStatus()`**: Returns `MiningStatus` (enabled, interval, ticks, startedAt).

### Accounts & URLs

*   **`urls`**: Getter returning `DevNodeUrls` (core, espace, coreWs, espaceWs).
*   **`accounts`**: Array of `DevNodeAccount` objects derived from the mnemonic.
*   **`faucet`**: Dedicated mining/faucet account (index 0, type 'mining').

## Error Handling

All operations throw `DevNodeError`, which extends `CfxError`. Errors include a specific `code` for programmatic handling.

| Code | Description |
| :--- | :--- |
| `devnode/already-running` | Attempted to start a running node. |
| `devnode/not-running` | Attempted to mine/stop on a stopped node. |
| `devnode/start-failed` | Underlying server failed to initialize. |
| `devnode/stop-failed` | Server shutdown failed. |
| `devnode/mining-already-running` | Auto-miner already active. |
| `devnode/mining-not-running` | Attempted to stop inactive auto-miner. |
| `devnode/invalid-config` | Invalid mnemonic or mining interval. |

## Integration

The module is designed to be consumed by higher-level tooling.

### Testing Fixtures
Used by `@cfxdevkit/testing` via `createDevNodeFixture`. Tests typically call `start()` before execution and `stop()` in teardown.

### DevNode Server
The `devnode-server` module wraps `DevNode` to expose HTTP endpoints.
*   **Controller**: Maps HTTP requests to `DevNode` methods (`start`, `stop`, `mine`, `status`).
*   **Runtime**: Uses `createRuntimeClient` to query `getMiningStatus`.

### Contract Deployment
Helpers like `deployContractCommand` utilize `packMine()` to ensure transactions are included in the chain immediately after signing.

## Implementation Notes

*   **Genesis Funding**: While `balanceCfx` is configurable in `DevNodeConfig`, the actual genesis funding is hardcoded in `@xcfx/node` via `genesisSecrets`. The `balanceCfx` field is primarily for display purposes in the CLI output.
*   **Account Derivation**: Accounts are derived using `deriveDualAccount` from `@cfxdevkit/cdk`, supporting both Core and eSpace addresses from a single mnemonic.
*   **Concurrency**: The auto-miner uses a flag (`autoMineInFlight`) to prevent overlapping `mine` calls, which can cause deadlocks in containerized environments.
*   **Cleanup**: The CLI registers `SIGINT` and `SIGTERM` handlers to ensure `node.stop()` is called gracefully on termination.
