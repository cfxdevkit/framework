# Wallet Abstraction & Hardware

The `@cfxdevkit/wallet` module provides a unified interface for managing cryptographic keys across multiple storage backends and hardware devices. It abstracts away vendor-specific details while enforcing security policies through capability-based access control.

## Core Components

### Error Handling

Two custom error classes handle failures across the module:

| Error Class | Purpose | Common Codes |
|-------------|---------|--------------|
| `SessionKeyError` | Session-key and policy enforcement failures | `wallet/session-key/expired`, `wallet/session-key/revoked`, `wallet/policies/*` |
| `HardwareWalletError` | Hardware adapter failures | `wallet/hardware/ledger/*`, `wallet/hardware/onekey/*`, `wallet/hardware/satochip/*` |

All errors extend `CfxError` from `@cfxdevkit/cdk` and include structured metadata for programmatic handling.

### Local Wallet Management

#### `initLocalWallet`

Creates a new encrypted keystore file with a fresh BIP-39 mnemonic.

```typescript
import { initLocalWallet } from '@cfxdevkit/wallet/init';

const result = await initLocalWallet({
  passphrase: 'strong-passphrase',
  label: 'deployer',
  derivationPath: "m/44'/60'/0'/0/0",
  kdf: { memKiB: 64, iterations: 1 } // Only for tests
});

console.log(result.address);
console.log(result.mnemonic); // Display once, then secure backup
```

**Security Notes:**
- Creates `~/.cfxdevkit/keystore.json` by default (override via `CFXDEVKIT_KEYSTORE` env var)
- Uses Argon2id for key derivation (AES-256-GCM for secret encryption)
- Mnemonic is returned once and never persisted — back it up immediately
- File permissions set to `0600`

#### `openLocalWallet`

Opens an existing keystore for signing operations.

```typescript
import { openLocalWallet } from '@cfxdevkit/wallet/init';

const { signer, ref } = await openLocalWallet({
  passphrase: 'strong-passphrase',
  ref: { service: 'cfxdevkit', account: 'deployer' }
});
```

#### `rotateLocalPassphrase`

Re-encrypts the keystore under a new passphrase without exposing the mnemonic.

```typescript
import { rotateLocalPassphrase } from '@cfxdevkit/wallet/init';

await rotateLocalPassphrase({
  oldPassphrase: 'old-pass',
  newPassphrase: 'new-pass'
});
```

### Hardware Wallet Adapters

All adapters conform to the `HardwareWalletAdapter` interface:

```typescript
interface HardwareWalletAdapter {
  readonly kind: 'ledger' | 'onekey' | 'satochip';
  getSigner(path?: string): Promise<Signer>;
}
```

#### Ledger

```typescript
import { createLedgerHardwareAdapter } from '@cfxdevkit/wallet/hardware/ledger';

const adapter = createLedgerHardwareAdapter({
  eth: ledgerEthApp,
  coreTransport: transport,
  path: "m/44'/60'/0'/0/0"
});

const signer = await adapter.getSigner();
```

**Supported Operations:**
- `signTransaction` (EIP-1559 only)
- `signMessage`
- `signTypedData`

#### OneKey

OneKey requires the SDK to be injected rather than imported to support multiple runtimes.

```typescript
import HardwareSDK from '@onekeyfe/hd-common-sdk';
import { signerFromOneKey } from '@cfxdevkit/wallet/hardware/onekey';

await HardwareSDK.init({ debug: false });

const signer = await signerFromOneKey({
  sdk: HardwareSDK,
  connectId: 'device-connect-id',
  deviceId: 'device-unique-id',
  chainId: 1030,
  expectedAddress: '0x...', // Optional device-swap protection
  showOnDevice: true
});
```

**Security Features:**
- `expectedAddress` parameter prevents device-swap attacks
- All signatures are normalised to canonical 65-byte format
- Only EIP-1559 transactions are supported

#### Satochip

Satochip communicates via a local bridge service (default `http://127.0.0.1:8397`).

```typescript
import { signerFromSatochip } from '@cfxdevkit/wallet/hardware/satochip';

const signer = await signerFromSatochip({
  bridgeUrl: 'http://127.0.0.1:8397',
  pin: '1234',
  keypath: "m/44'/60'/0'/0/0",
  expectedAddress: '0x...'
});
```

**Bridge Endpoints:**
| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/health` | GET | Check card connection status |
| `/init` | POST | Unlock card with PIN |
| `/address` | GET | Derive address from keypath |
| `/sign-message` | POST | Sign arbitrary message |
| `/sign-transaction` | POST | Sign EIP-1559 transaction |

**Security Notes:**
- Bridge must run on loopback only (never expose externally)
- PIN is only sent during `/init` call
- All responses are validated for signature format

### Capability Enforcement

The `withCapability` wrapper enforces transaction policies before signing:

```typescript
import { withCapability } from '@cfxdevkit/wallet/policies';

const scoped = withCapability(rootSigner, {
  chains: [1030],
  contracts: ['0x...'],
  selectors: ['0xa9059cbb'], // ERC-20 transfer
  maxValuePerTx: 1000000000000000000n, // 1 ETH
  notAfter: Date.now() + 60 * 60_000 // 1 hour expiry
});

await scoped.signTransaction(tx); // Throws SessionKeyError if policy violated
```

**Policy Checks:**
| Check | Error Code | Condition |
|-------|------------|-----------|
| Expiry | `wallet/policies/expired` | `Date.now() > capability.notAfter` |
| Chain | `wallet/policies/chain-denied` | `tx.chainId ∉ capability.chains` |
| Contract | `wallet/policies/contract-denied` | `tx.to ∉ capability.contracts` |
| Selector | `wallet/policies/selector-denied` | `tx.data[0..4] ∉ capability.selectors` |
| Value | `wallet/policies/value-exceeded` | `tx.value > capability.maxValuePerTx` |

**Dry-Run Validation:**

```typescript
import { checkCapability } from '@cfxdevkit/wallet/policies';

const error = checkCapability(capability, tx);
if (error) {
  console.log('Policy violation:', error.code);
}
```

### Session Keys

Session keys are ephemeral, in-memory keypairs scoped by capabilities. They enable delegation without exposing the parent's private key.

```typescript
import { createSessionKey } from '@cfxdevkit/wallet/session-key';

const session = await createSessionKey({
  parent: rootSigner,
  capability: {
    chains: [1030],
    contracts: [tokenAddress],
    selectors: ['0xa9059cbb'],
    maxValuePerTx: 0n,
    notAfter: Date.now() + 60 * 60_000
  }
});

// Use session.signer for delegated operations
await session.signer.signTransaction(tx);

// Revoke when done
session.revoke();
```

**Session Key Properties:**
| Property | Description |
|----------|-------------|
| `signer` | Capability-enforcing signer for delegated operations |
| `address` | Session key's public address |
| `parent` | Parent signer's address (attestation signer) |
| `capability` | Bound capability constraints |
| `attestation` | Off-chain proof: `{ message, signature, digest }` |
| `revoke()` | Wipe private key and disable signer |
| `isRevoked` | True after `revoke()` or expiry |

**Attestation Format:**

The attestation is a canonical JSON message signed by the parent:

```json
{
  "v": 1,
  "type": "cfxdevkit.session-key.v1",
  "parent": "0x...",
  "session": "0x...",
  "capability": {
    "chains": [1030],
    "contracts": ["0x..."],
    "selectors": ["0xa9059cbb"],
    "maxValuePerTx": "0",
    "notAfter": 1234567890
  }
}
```

Pair this attestation with an on-chain validator (EIP-7702, 4337 paymaster, or custom registry) for actual privilege delegation.

### Signer Factories

#### `signerFromKeystore`

The recommended entry point for production code:

```typescript
import { signerFromKeystore } from '@cfxdevkit/wallet/signers';

const signer = await signerFromKeystore({
  provider: keystoreProvider,
  ref: { service: 'cfxdevkit', account: 'deployer' },
  capability: { chains: [1030] }
});
```

#### `readonlySigner`

A placeholder for read-only operations:

```typescript
import { readonlySigner } from '@cfxdevkit/wallet/signers';

const signer = readonlySigner('0x...');
// All sign* calls throw SessionKeyError with code 'wallet/signers/readonly'
```

## Integration Patterns

### Hardware + Policy Enforcement

```typescript
import { createLedgerHardwareAdapter } from '@cfxdevkit/wallet/hardware/ledger';
import { withCapability } from '@cfxdevkit/wallet/policies';

const adapter = createLedgerHardwareAdapter({ eth: ledgerApp, transport });
const hardwareSigner = await adapter.getSigner();

const policySigner = withCapability(hardwareSigner, {
  chains: [1030],
  contracts: ['0x...'],
  maxValuePerTx: 1000000000000000000n
});
```

### Session Key + Hardware Parent

```typescript
import { createSessionKey } from '@cfxdevkit/wallet/session-key';
import { createLedgerHardwareAdapter } from '@cfxdevkit/wallet/hardware/ledger';

const adapter = createLedgerHardwareAdapter({ eth: ledgerApp, transport });
const parentSigner = await adapter.getSigner();

const session = await createSessionKey({
  parent: parentSigner,
  capability: {
    chains: [1030],
    contracts: ['0x...'],
    selectors: ['0xa9059cbb'],
    notAfter: Date.now() + 3600_000
  }
});

// session.signer can be handed to a dapp or agent
// session.attestation can be submitted to on-chain validator
```

### Local Wallet + Session Key

```typescript
import { initLocalWallet } from '@cfxdevkit/wallet/init';
import { createSessionKey } from '@cfxdevkit/wallet/session-key';

const { signer: parentSigner } = await initLocalWallet({
  passphrase: 'strong-passphrase'
});

const session = await createSessionKey({
  parent: parentSigner,
  capability: {
    chains: [1030],
    maxValuePerTx: 1000000000000000000n,
    notAfter: Date.now() + 3600_000
  }
});
```

## Signature Normalisation

Hardware devices return `(r, s, v)` tuples. The module normalises these to the canonical 65-byte hex format expected by `viem` and EVM consumers.

```typescript
import { rawSignatureToHex, finaliseEip1559Tx } from '@cfxdevkit/wallet/hardware/types';

// Personal sign (v = 27|28)
const personalSig = rawSignatureToHex({ r, s, v: 27 });

// EIP-1559 transaction (v = 0|1, normalised to yParity)
const txSig = finaliseEip1559Tx(unsignedTx, { r, s, v: 0 });
```

## Error Handling Best Practices

```typescript
import { SessionKeyError, HardwareWalletError } from '@cfxdevkit/wallet/errors';

try {
  await session.signer.signTransaction(tx);
} catch (err) {
  if (err instanceof SessionKeyError) {
    switch (err.code) {
      case 'wallet/policies/expired':
        // Refresh session or create new one
        break;
      case 'wallet/policies/contract-denied':
        // Log policy violation
        break;
      case 'wallet/session-key/revoked':
        // Session already invalidated
        break;
    }
  } else if (err instanceof HardwareWalletError) {
    switch (err.code) {
      case 'wallet/hardware/aborted':
        // User cancelled operation
        break;
      case 'wallet/hardware/address-mismatch':
        // Device-swap detected
        break;
    }
  }
}
```

## Testing Considerations

For tests, reduce KDF parameters to speed up keystore operations:

```typescript
await initLocalWallet({
  passphrase: 'test-pass',
  kdf: { memKiB: 64, iterations: 1 } // ~10ms instead of ~5s
});
```

For session key tests, provide a deterministic private key:

```typescript
const session = await createSessionKey({
  parent: parentSigner,
  capability: { chains: [1030] },
  privateKey: '0x...' // Deterministic for reproducible tests
});
```

## Security Checklist

- [ ] Never log or expose mnemonics in production
- [ ] Use `expectedAddress` for hardware wallets to prevent device-swap
- [ ] Set `notAfter` on session keys to limit exposure window
- [ ] Run Satochip bridge on loopback only (`127.0.0.1`)
- [ ] Validate `capability` is non-empty before creating session keys
- [ ] Call `session.revoke()` when session is no longer needed
- [ ] Use `withCapability` even for backends that natively enforce policies (defence in depth)
