# Wallet Abstraction & Hardware

This module provides core wallet abstraction and hardware wallet integration capabilities for the `@cfxdevkit/wallet` package. It enables developers to work with various wallet backends (local keystore, hardware wallets, session keys) in a consistent, policy-enforced manner.

## Purpose

The module abstracts wallet operations across different backends:
- Local keystore-backed wallets with encrypted storage
- Hardware wallets (Ledger, OneKey, Satochip)
- Session keys with capability-based access control
- Batched transaction execution
- Policy enforcement for secure signing

## Key Components

### 1. Transaction Batching (`batcher`)

The `TransactionBatcher` class enables batched execution of asynchronous tasks with configurable concurrency and error handling.

```ts
import { TransactionBatcher } from '@cfxdevkit/wallet/batcher';

const batcher = new TransactionBatcher({ concurrency: 1 });
batcher.add(async () => await fetch('/api/tx1'));
batcher.add(async () => await fetch('/api/tx2'));
const results = await batcher.execute();
```

### 2. Hardware Wallet Adapters

Hardware wallet support is provided through vendor-specific adapters that implement a common `HardwareWalletAdapter` interface.

#### Ledger
```ts
import { createLedgerHardwareAdapter } from '@cfxdevkit/wallet/hardware/ledger';

const adapter = createLedgerHardwareAdapter({
  coreTransport: transport,
  path: "m/44'/60'/0'/0/0"
});
const signer = await adapter.getSigner();
```

#### OneKey
```ts
import { signerFromOneKey } from '@cfxdevkit/wallet/hardware/onekey';

const signer = await signerFromOneKey({
  sdk: HardwareSDK,
  connectId: 'device-connect-id',
  deviceId: 'device-id',
  chainId: 1030
});
```

#### Satochip
```ts
import { signerFromSatochip } from '@cfxdevkit/wallet/hardware/satochip';

const signer = await signerFromSatochip({
  bridgeUrl: 'http://127.0.0.1:8397',
  pin: '123456'
});
```

### 3. Session Keys

Session keys provide ephemeral, capability-constrained signers that can be revoked or expire.

```ts
import { createSessionKey } from '@cfxdevkit/wallet/session-key';

const session = await createSessionKey({
  parent: hardwareSigner,
  capability: {
    chains: [1030],
    contracts: ['0x...'],
    maxValuePerTx: 0n,
    notAfter: Date.now() + 3600000
  }
});
// Use session.signer for signing with enforced policies
```

### 4. Policy Enforcement

Capability-based policies restrict what operations a signer can perform.

```ts
import { withCapability } from '@cfxdevkit/wallet/policies';

const scopedSigner = withCapability(signer, {
  chains: [1030],
  contracts: ['0x...'],
  selectors: ['0xa9059cbb']
});
```

### 5. Local Wallet Initialization

Turn-key wallet initialization and management for local keystores.

```ts
import { initLocalWallet, openLocalWallet } from '@cfxdevkit/wallet/init';

// First-time setup
const wallet = await initLocalWallet({
  passphrase: 'secure-passphrase',
  label: 'deployer'
});

// Subsequent access
const wallet = await openLocalWallet({
  passphrase: 'secure-passphrase',
  ref: { service: 'cfxdevkit', account: 'deployer' }
});
```

## Error Handling

All wallet operations throw specific error types:
- `HardwareWalletError` for hardware-related failures
- `SessionKeyError` for session key operations
- `KeystoreError` for keystore backend issues

## Integration Points

This module integrates with:
- `@cfxdevkit/cdk` for core signer interfaces
- `@cfxdevkit/services/keystore` for backend storage
- `@cfxdevkit/executor` for batched execution
- `viem` for transaction serialization and EIP-1559 support

The module provides a consistent interface across all wallet types, enabling seamless switching between local and hardware wallets while maintaining security through capability-based policies.
