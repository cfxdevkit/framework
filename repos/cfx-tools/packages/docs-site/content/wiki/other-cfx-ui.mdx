# Other — cfx-ui

# cfx-ui

**Tier 0c — React UI surface.** Carve-out target per ADR-0003.

This workspace provides the frontend infrastructure for Conflux ecosystem applications. It is structured as a monorepo using `pnpm` workspaces, containing four primary libraries that layer functionality from design tokens up to DeFi widgets.

## Architecture

The UI surface is organized into four distinct packages. Dependencies flow from foundational utilities up to high-level components.

## Package Overview

### @cfxdevkit/theme
**Purpose:** Design tokens and CSS layer.
**Version:** 2.0.0

Provides the visual foundation for the UI. It exports React components for theming and raw CSS files for styling.

*   **Exports:**
    *   `.`: Main entry (`ThemeProvider`, tokens).
    *   `./tokens`: Design token definitions.
    *   `./react`: React-specific theme providers.
    *   `./css`: Base CSS (`base.css`, `dark.css`).
*   **Build Configuration:**
    *   Uses `vite-plugin-dts` for TypeScript declarations.
    *   Includes a custom `copyCssPlugin` in `vite.config.ts` that manually copies `src/css/*.css` to `dist/css/` after the JS build completes.
    *   Marks CSS files as `sideEffects` to ensure they are not tree-shaken.

### @cfxdevkit/react
**Purpose:** React hooks over `@cfxdevkit/cdk`.
**Version:** 2.0.0

Wraps core logic into React hooks for state management, account handling, and transaction execution.

*   **Exports:**
    *   `.`: Main entry (re-exports all hooks).
    *   `./account`, `./balance`, `./context`, `./contract`, `./events`, `./tx`, `./keystore`: Feature-specific modules.
*   **Build Configuration:**
    *   Includes a custom `preserveDirectives` plugin in `vite.config.ts`.
    *   **Functionality:** Scans source modules for `"use client"` directives. If a chunk contains client modules, it prepends `"use client"` to the generated bundle. This ensures compatibility with React Server Components boundaries.
    *   Targets `es2023`.

### @cfxdevkit/wallet-connect
**Purpose:** Browser wallet connectors and headless UI.
**Version:** 2.0.0

Handles wallet connection logic, chain switching, and authentication (SIWE).

*   **Exports:**
    *   `.`: Main entry.
    *   `./config`: Chain configurations.
    *   `./hooks`: Wallet connection hooks.
    *   `./siwe`: Sign-In with Ethereum logic.
    *   `./ui`: Headless UI components.
    *   `./auth`: Authentication utilities.
*   **Key Logic:**
    *   **Chain Management:** Exports `CORE_CHAIN_CONFIGS` and `createSupportedEspaceChains` (located in `src/config/chains.ts`).
    *   **State Logic:** Exports `needsESpaceSwitch` (located in `src/lib/walletState.ts`) to determine if a chain switch is required during wallet operations.
    *   **Error Handling:** Exports `errMsg` for standardized error formatting.
*   **Testing:**
    *   Tests verify `CORE_CHAIN_CONFIGS` chain IDs (e.g., `0x405` for chain 1029).
    *   Tests verify `createSupportedEspaceChains` returns expected IDs `[1030, 71, 2030]`.

### @cfxdevkit/defi-react
**Purpose:** Opinionated DeFi widgets (swap, portfolio, picker).
**Version:** 2.0.0

High-level components for interacting with DeFi protocols.

*   **Exports:**
    *   `.`: Main entry.
    *   `./swap`, `./balance`, `./token-picker`, `./tx-status`, `./primitives`, `./service`, `./pool`, `./lp`: Feature-specific widgets.
*   **Dependencies:**
    *   Depends on `@cfxdevkit/react`, `@cfxdevkit/theme`, `@cfxdevkit/abis`, `@cfxdevkit/cdk`.
    *   Peer dependencies include `@cfxdevkit/wallet`.
*   **Build Configuration:**
    *   Configured in `vite.config.ts` with specific entry points for `swap`, `balance`, `token-picker`, `tx-status`, and `primitives`.
    *   **Note:** The `package.json` exports `./lp` pointing to `./src/lp/index.ts` for types and `./src/lp/index.js` for imports, bypassing the standard `dist` build path for this specific entry.

## Development Workflow

### Environment Requirements
*   **Node:** `>=22`
*   **Package Manager:** `pnpm@10.6.0` (enforced via `packageManager` field)

### Scripts
Run commands from the workspace root or specific package directories.

| Command | Description |
| :--- | :--- |
| `pnpm validate:workspace` | Validates workspace structure (`cfx-ui` root). |
| `pnpm validate:deps` | Checks dependency consistency (`cfx-ui` root). |
| `pnpm build` | Builds the package using Vite. |
| `pnpm test` | Runs Vitest suite. |
| `pnpm test:watch` | Runs Vitest in watch mode. |
| `pnpm typecheck` | Runs TypeScript compiler without emitting. |
| `pnpm lint` | Runs Biome linter. |
| `pnpm clean` | Removes `dist`, `.vitest`, and `coverage` directories. |

### Testing Strategy
All packages use **Vitest** with a Node environment.
*   **Test Files:** Located in `src/**/*.test.ts` or `src/**/*.test.tsx`.
*   **Exclusions:** Test files are excluded from TypeScript declaration generation (`emitDeclarationOnly`).
*   **Standard Pattern:** Each package exports a `__packageName` constant which is verified in the root test file to ensure correct module resolution.

```typescript
// Example test pattern found across packages
import { __packageName } from './index.js';

describe('@cfxdevkit/<package>', () => {
  it('exposes its package name', () => {
    expect(__packageName).toBe('@cfxdevkit/<package>');
  });
});
```

### Build Output
*   **Format:** ES Modules (`es`).
*   **Output Directory:** `dist/`.
*   **Type Definitions:** Generated via `vite-plugin-dts` into `dist/`.
*   **Source Maps:** Enabled (`sourcemap: true`).
*   **Minification:** Disabled (`minify: false`) to preserve debugging capabilities.

## Contributing

1.  **Install Dependencies:**
    ```bash
    pnpm install
    ```
2.  **Validate Workspace:**
    Ensure the workspace structure is valid before committing.
    ```bash
    pnpm validate:workspace
    ```
3.  **Add Exports:**
    When adding new modules, update `package.json` `exports` and `vite.config.ts` `build.lib.entry` simultaneously to ensure the build pipeline includes the new entry point.
4.  **React Directives:**
    If adding new components that require client-side interactivity, ensure `"use client"` is at the top of the file. The `preserveDirectives` plugin in `@cfxdevkit/react` will handle bundling, but verify the directive is present in source.
5.  **CSS Assets:**
    For `@cfxdevkit/theme`, new CSS files must be manually added to the `copyCssPlugin` in `vite.config.ts` to ensure they are copied to `dist/css/`.
