# cfx-domain

The `cfx-domain` module serves as the Tier 2 verticals container for game-engine and automation logic. It is designated as a carve-out target per ADR-0003. This monorepo manages domain-specific packages that interact with on-chain state, automation strategies, and hardware interfaces.

## Architecture

The domain is structured as a pnpm workspace containing three primary packages. Each package is configured for TypeScript, ES module output, and Vite-based builds.

## Packages

### @cfxdevkit/automation

The core automation package implements trading and execution strategies including limit orders, DCA, stop-loss, and scheduled actions. It integrates with blockchain protocols via `viem` and manages state using `drizzle-orm`.

**Entry Point:** `src/index.ts`

**Public Exports:**
The package exposes the following modules via its main entry point:

| Export Path | Description |
| :--- | :--- |
| `./conditions/price.js` | Price-based condition logic |
| `./conditions/time.js` | Time-based condition logic |
| `./keeper/index.js` | Keeper orchestration |
| `./keeper.js` | Keeper utilities |
| `./priceSources/geckoTerminal.js` | GeckoTerminal price feed integration |
| `./priceSources/swappi.js` | Swappi price feed integration |
| `./repository/memory.js` | In-memory repository implementation |
| `./repository.js` | Repository abstraction |
| `./retryQueue.js` | Retry queue management |
| `./safety.js` | Safety checks and guards |
| `./strategies/index.js` | Strategy implementations |
| `./swap/index.js` | Swap execution logic |
| `./types.js` | Shared type definitions |

**Database Export:**
A separate entry point is available for database-specific functionality:
*   **Path:** `./db`
*   **Files:** `dist/db/index.js`, `dist/db/index.d.ts`

**Dependencies:**
*   `@cfxdevkit/cdk` (workspace)
*   `@cfxdevkit/executor` (workspace)
*   `@cfxdevkit/protocol` (workspace)
*   `better-sqlite3`
*   `drizzle-orm`
*   `viem`

**Peer Dependencies:**
*   `mysql2` (optional)
*   `postgres` (optional)

**Constants:**
*   `__packageName`: Exports the string `'@cfxdevkit/automation'` for identification.

### @cfxdevkit/game-engine

This package is designed as a reusable on-chain game state engine. As of version 2.0.0, it is in a scaffolding state.

**Entry Point:** `src/index.ts`

**Public Exports:**
*   `__packageName`: Exports the string `'@cfxdevkit/game-engine'`.

**Status:**
Implementation details are pending. Refer to `./API.md` and `./STRUCTURE.md` within the package directory for future specifications.

### @cfxdevkit/hardware-bridge

This package exists within the workspace structure but source files are not accessible in the current context. It is intended to bridge domain logic with external hardware interfaces.

## Build & Configuration

### Build System
All packages use **Vite** for bundling and **vite-plugin-dts** for TypeScript declaration generation.

**Build Target:** `es2023`
**Output Format:** ES Modules (`es`)
**Source Maps:** Enabled

**Vite Configuration (`vite.config.ts`):**
*   **Automation:** Builds two entries: `index` and `db/index`.
*   **Game Engine:** Builds single entry: `index`.
*   **Rollup:** Externalizes all non-relative imports.

### TypeScript Configuration
Packages extend `@cfxdevkit/tsconfig/lib.json`.

**Key Compiler Options:**
*   `rootDir`: `src`
*   `outDir`: `dist`
*   `emitDeclarationOnly`: `true` (TypeScript emits `.d.ts` files; Vite handles JS output)

### Moon Configuration
Projects are defined in `moon.yml` with the following settings:
*   `layer`: `library`
*   `language`: `typescript`

## Development Workflow

### Prerequisites
*   Node.js >= 22
*   pnpm >= 10

### Commands

| Command | Description |
| :--- | :--- |
| `pnpm build` | Compiles TypeScript and bundles with Vite |
| `pnpm test` | Runs Vitest suite |
| `pnpm test:watch` | Runs Vitest in watch mode |
| `pnpm typecheck` | Runs TypeScript compiler without emitting |
| `pnpm lint` | Runs Biome linter on `src` |
| `pnpm clean` | Removes `dist`, `.vitest`, and `coverage` directories |
| `pnpm prepublishOnly` | Runs `clean` before publishing |

### Workspace Validation
The root `cfx-domain` package includes scripts to validate the workspace structure and dependencies:
*   `pnpm validate:workspace`: Executes `scripts/validate-workspace.mjs`
*   `pnpm validate:deps`: Executes `scripts/check-deps.mjs`

## Testing

Tests are configured using **Vitest**.

**Configuration (`vitest.config.ts`):**
*   **Environment:** `node`
*   **Include:** `src/**/*.test.ts`, `src/**/*.test.tsx`
*   **Aliases:** `@cfxdevkit/executor` is aliased to the core package source for testing purposes.

**Example Test Structure:**
Tests verify package identity and module exports.
```typescript
import { describe, expect, it } from 'vitest';
import { __packageName } from './index.js';

describe('@cfxdevkit/automation', () => {
  it('exposes its package name', () => {
    expect(__packageName).toBe('@cfxdevkit/automation');
  });
});
```
