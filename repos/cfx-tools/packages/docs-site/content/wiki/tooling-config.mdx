# Tooling & Config

This module serves as the root configuration and automation layer for the `@cfxdevkit/framework` monorepo. It defines the workspace topology, enforces code quality standards, manages build pipelines via `moon`, and provides utility scripts for scaffolding new packages and wiring internal dependencies.

## Overview

The Tooling & Config module centralizes:
- **Workspace Management:** `pnpm` workspace definitions and dependency overrides.
- **Code Quality:** Linting (`biome`), type checking (`typescript`), and unused dependency detection (`knip`).
- **Build System:** `moon` orchestration for parallel task execution across packages.
- **Automation:** Bash scripts for package scaffolding and dependency graph wiring.
- **Shared Configs:** Reusable configurations for TypeScript, Biome, and Moon located in `tools/`.

## Workspace Topology

The monorepo structure is defined in `pnpm-workspace.yaml`. Packages are organized into three primary categories:

```yaml
packages:
  - "repos/cfx-config/packages/*"
  - "repos/cfx-meta/packages/*"
  - "repos/*/packages/*"
  - "repos/cfx-tools/devtools/*"
  - "repos/cfx-tools/infra/*"
  - "repos/cfx-tools/templates/*"
  - "projects/*/apps/*"
  - "projects/*/packages/*"
  - "projects/*/contracts"
```

### Shared Configuration (`tools/`)

Common configurations are versioned and shared across the workspace to ensure consistency:

| Directory | Purpose |
| :--- | :--- |
| `tools/biome-config` | Shared linting and formatting rules. |
| `tools/moon-config` | Shared build task definitions and project templates. |
| `tools/tsconfig` | Shared TypeScript compiler options (`base.json`, `lib.json`, `app-node.json`, `app-web.json`). |
| `tools/codegen` | Code generation utilities (API types, Wagmi bindings). |

## Configuration Files

### `package.json`
Defines the root scripts and engine requirements.

- **Node Engine:** `>=24.15.0`
- **Package Manager:** `pnpm@10.33.2`
- **Key Dependencies:**
  - `@biomejs/biome`: Linting and formatting.
  - `@moonrepo/cli`: Build orchestration.
  - `knip`: Unused dependency detection.
  - `vite` / `vitest`: Build and testing infrastructure.

### `biome.json`
Extends the shared `biome-config` and enables specific parser features:
```json
{
  "extends": ["./repos/cfx-config/packages/biome-config/biome.json"],
  "css": {
    "parser": {
      "tailwindDirectives": true
    }
  }
}
```

### `tsconfig.base.json`
Extends the shared `tsconfig` and defines path aliases for core packages to resolve imports locally without publishing:
```json
{
  "extends": "./repos/cfx-config/packages/tsconfig/base.json",
  "compilerOptions": {
    "paths": {
      "@cfxdevkit/core": ["./repos/cfx-core/packages/core/src/index.ts"],
      "@cfxdevkit/protocol": ["./repos/cfx-core/packages/protocol/src/index.ts"],
      "@cfxdevkit/executor": ["./repos/cfx-core/packages/executor/src/index.ts"],
      "@cfxdevkit/devnode": ["./repos/cfx-core/packages/devnode/src/index.ts"],
      "@cfxdevkit/testing": ["./repos/cfx-core/packages/testing/src/index.ts"]
    }
  }
}
```

### `pnpm.overrides`
Enforces specific versions of transitive dependencies to ensure security and compatibility:
- `axios`: `^1.15.0`
- `esbuild`: `>=0.25.0`
- `zod`: `4.3.4`
- `viem`: `^2.21.0`
- `wagmi`: `^3.6.9`

## Automation Scripts

### Package Scaffolding
`scripts/scaffold-empty-packages.sh` generates boilerplate for new packages. It is idempotent (skips existing files).

**Usage:**
```bash
./scripts/scaffold-empty-packages.sh
```

**Logic Flow:**
1. Defines `scaffold_lib` helper function.
2. Accepts arguments: `<pkg_dir> <pkg_name> <description> [kind]`.
3. Generates standard files: `package.json`, `tsconfig.json`, `vite.config.ts`, `vitest.config.ts`, `moon.yml`, `src/index.ts`, `src/index.test.ts`.
4. Supports `node` (default) and `react` kinds (adds React peer dependencies and JSX handling).

**Supported Packages:**
- **Framework:** `@cfxdevkit/cdk`, `@cfxdevkit/services`, `@cfxdevkit/wallet`, `@cfxdevkit/compiler`, `@cfxdevkit/devnode`, `@cfxdevkit/contracts`, `@cfxdevkit/protocol`, `@cfxdevkit/executor`, `@cfxdevkit/react`, `@cfxdevkit/wallet-connect`, `@cfxdevkit/defi-react`, `@cfxdevkit/theme`, `@cfxdevkit/testing`.
- **Domains:** `@cfxdevkit/game-engine`, `@cfxdevkit/automation`.
- **Platform:** `@cfxdevkit/mcp-server`, `@cfxdevkit/create`.

### Dependency Wiring
`scripts/wire-package-dependencies.sh` updates `package.json` files to reflect internal workspace dependencies.

**Usage:**
```bash
./scripts/wire-package-dependencies.sh
```

**Logic Flow:**
1. Defines `add_dep` helper function.
2. Runs `pnpm --filter <filter> add --save-prod|--save-peer <deps>`.
3. Ensures `package.json` and `pnpm-lock.yaml` remain coherent.

**Dependency Graph:**
- `@cfxdevkit/services` → `@cfxdevkit/cdk`
- `@cfxdevkit/wallet` → `@cfxdevkit/cdk`, `@cfxdevkit/services`
- `@cfxdevkit/react` → `@cfxdevkit/cdk` (prod), `@cfxdevkit/wallet` (peer)
- `@cfxdevkit/defi-react` → `@cfxdevkit/cdk`, `@cfxdevkit/services`, `@cfxdevkit/react`, `@cfxdevkit/wallet-connect` (prod), `@cfxdevkit/theme` (peer)
- `@cfxdevkit/mcp-server` → `@cfxdevkit/cdk`, `@cfxdevkit/services`, `@cfxdevkit/wallet`, `@cfxdevkit/contracts`, `@cfxdevkit/compiler`, `@cfxdevkit/devnode`

## Lifecycle Commands

### Build & Test
| Command | Description |
| :--- | :--- |
| `pnpm build` | Runs `moon run :build` across the workspace. |
| `pnpm test` | Runs `moon run :test --concurrency 1`. |
| `pnpm test:cover` | Runs coverage across all packages (`pnpm -r --if-present test:coverage`). |
| `pnpm clean` | Runs `moon run :clean`. |

### Quality & Linting
| Command | Description |
| :--- | :--- |
| `pnpm lint` | Runs `moon run :lint`. |
| `pnpm typecheck` | Runs `moon run :typecheck`. |
| `pnpm check` | Runs `moon run :check`. |
| `pnpm format` | Runs `biome format --write .`. |
| `pnpm format:check` | Runs `biome format .`. |
| `pnpm check:unused` | Runs `knip` to detect unused dependencies. |

### Security
| Command | Description |
| :--- | :--- |
| `pnpm security:audit` | Runs `pnpm audit --prod --audit-level moderate`. |
| `pnpm security:check` | Runs audit + secret leak check (`check:secrets`). |
| `pnpm check:secrets` | Runs `moon run arch-check:check-secrets`. |

### Documentation & LLM
| Command | Description |
| :--- | :--- |
| `pnpm docs:sync` | Syncs documentation site (`@cfxdevkit/docs-site`). |
| `pnpm docs:validate-wiki` | Validates wiki structure. |
| `pnpm llm:all` | Runs all LLM tooling actions. |
| `pnpm llm:docs-pipeline` | Runs documentation generation pipeline. |
| `pnpm llm:review` | Runs LLM code review. |

### Repository Management
| Command | Description |
| :--- | :--- |
| `pnpm gitnexus:ensure` | Ensures `gitnexus` script is available. |
| `pnpm gitnexus:list` | Lists repository connections. |
| `pnpm gitnexus:status` | Checks status and lists connections. |

## Contributing

When adding a new package to the monorepo:
1. **Scaffold:** Run `./scripts/scaffold-empty-packages.sh` to generate the base structure.
2. **Wire:** Run `./scripts/wire-package-dependencies.sh` to ensure internal dependencies are declared.
3. **Configure:** Update `tools/tsconfig` or `tools/biome-config` if new global rules are required.
4. **Verify:** Run `pnpm check` and `pnpm security:check` before committing.
