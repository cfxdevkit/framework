# scripts

# scripts/ Module Documentation

## Overview
The `scripts/` module provides automation tooling for the `cfxdevkit` monorepo. It handles repository maintenance, automated refactoring, package publishing, scaffolding, validation, and local development environment management. The module is split between ESM Node.js scripts (`.mjs`) for complex path resolution and publishing logic, and Bash scripts (`.sh`) for idempotent file generation and process management.

---

## Refactoring & Import Management

These two scripts automate the resolution of "kebab-group" violations, where files in a directory share a common prefix but are not grouped into a subdirectory.

### `fix-kebab-groups.mjs`
Moves grouped files into dedicated subdirectories and rewrites all import references across the repository.

* **Input**: `artifacts/llm/reports/kebab-groups.json` (generated by `pnpm run check:kebab-groups`)
* **Execution Flow**:
  1. Parses the report and sorts groups by prefix length (longest first) to handle nested groups correctly.
  2. Builds a `moveMap` tracking `oldRelativePath → newRelativePath`.
  3. Executes `git mv` (or simulates in `--dry-run` mode) to preserve VCS history.
  4. Walks all source files via `walkFiles()` and applies regex-based import replacements using `replaceMap`.
  5. Writes modified files and logs counts of moved files and updated imports.

### `fix-kebab-imports.mjs`
Handles import path depth adjustment after files have been moved deeper into the directory tree.

* **Execution Flow**:
  1. Reuses `kebab-groups.json` to build `moveMap` (resolving exact file paths and extensionless module specifiers) and `depthShiftMap` (tracking how many directory levels deeper each file moved).
  2. Walks all source files and matches imports against `importRe` (`/from\s+|import\s*\(|require\s*\(/`).
  3. For each import, resolves candidate paths (base, `.ts`, `.tsx`, `.js`, `.jsx`, `.mjs`, `/index.*`) against `moveMap`.
  4. If the file itself moved, applies `adjustDepth()` to prepend the correct number of `../` prefixes to relative imports that didn't match the move map.
  5. Writes changes unless `--dry-run` is passed.

## Package Publishing

### `publish-packages.mjs`
Orchestrates publishing packages to npm with proper authentication, deduplication, and SLSA provenance handling.

* **Key Functions**:
  * `collectPackageDirs()`: Scans `repos/*/packages/` for directories containing `package.json`, excluding the root workspace and paths matching `.changeset/config.json`'s `ignore` list.
  * `readJson(path)`: Synchronous JSON parser for `package.json` access.
  * `versionExists(packageName, version)`: Checks npm registry for exact version conflicts using `npm view`.
  * `packageIsRegistered(packageName)`: Determines if a package name is already registered to handle provenance restrictions.
  * `run(command, args, options)`: Cross-platform wrapper around `spawnSync` with error capturing and stdio control.
  * `buildAuthArgs()`: Manages npm authentication. Prioritizes `NPM_TOKEN` env var (writes temporary `.npmrc.publish`), falls back to OIDC (`ACTIONS_ID_TOKEN_REQUEST_TOKEN`), then local `.npmrc`.
  * `cleanupAuthFiles()`: Ensures temporary auth artifacts are removed on `exit`, `SIGINT`, or `SIGTERM`.

* **Publishing Logic**:
  1. Detects auth method and logs it.
  2. Iterates publishable packages. Skips `private: true` or ignored packages.
  3. Skips packages where `versionExists()` returns true.
  4. Runs `pnpm pack` to generate a tarball.
  5. Publishes with `--access public`. Adds `--provenance` only if OIDC is active AND the package already exists on npm (npm blocks provenance on first-time package creation).
  6. Cleans up tarballs and exits with code 1 if any failures occurred.

## Scaffolding & Dependency Wiring

### `scaffold-empty-packages.sh`
Generates a standardized, idempotent package structure for new internal libraries.

* **Helper**: `write_if_missing(path, heredoc)` creates directories and writes files only if they don't exist.
* **`scaffold_lib(dir, name, desc, kind)`**:
  * Generates `package.json` with standardized `exports`, `files`, and scripts. Conditionally adds React peer dependencies (`@types/react`, `react`, `react-dom`) if `kind=react`.
  * Generates `tsconfig.json` extending `@cfxdevkit/tsconfig/lib.json`.
  * Generates `vite.config.ts` with library build config. Sets `peerExternals` to include React packages if `kind=react`.
  * Generates `vitest.config.ts` with environment and reporter defaults.
  * Creates `src/index.ts` and `src/index.test.ts` with a basic smoke test.
  * Creates `moon.yml` for moonrepo project detection.
* **Execution**: Calls `scaffold_lib` across `framework/`, `domains/`, and `platform/` directories with predefined package names and descriptions.

### `wire-package-dependencies.sh`
Wires internal workspace dependencies using pnpm's `--save-prod` and `--save-peer` flags.

* **Pattern**: `add_dep <package_filter> <kind> <dep1> [dep2...]`
* Idempotent: pnpm gracefully handles existing dependency upgrades in-place.
* Organized by domain (`framework`, `domains`, `platform`) to match the monorepo's architectural layers.

## Repository Validation

### `validate-repos.mjs`
Sequentially executes per-repo validation scripts across all `cfx-*` repositories.

* **Execution**:
  1. Defines hardcoded `REPOS` list (`cfx-config`, `cfx-core`, etc.).
  2. Iterates each repo and runs `repos/<repo>/scripts/validate-workspace.mjs` and `check-deps.mjs`.
  3. Catches non-zero exit statuses and aggregates failures.
  4. Exits with code 1 and prints all failures if any occurred, otherwise confirms success.

## Development Showcase Management

### `showcase-dev.sh` & `stop-showcase.sh`
Manages concurrent development servers for `@cfxdevkit/example-showcase-local` (port 3011) and `@cfxdevkit/example-showcase-public` (port 3010).

* `showcase-dev.sh`:
  * Uses `ss -ltnp` to detect existing listeners on target ports.
  * Validates processes via `is_showcase_process()` by walking parent PIDs and checking command paths.
  * Prevents duplicate starts and port conflicts.
  * Runs `pnpm --parallel --filter ... dev`.
* `stop-showcase.sh`:
  * Queries ports 3010/3011 for PIDs.
  * Issues `kill` and exits.

## DevContainer & Local CA Trust

### `setup-trust-local-ca.sh`
Installs a locally generated `mkcert` root CA into the host machine's trust store, enabling HTTPS for `*.dev.cfxdevkit.org` in development.

* **Execution Flow**:
  1. `find_container_engine()`: Detects `docker` or `podman`.
  2. `detect_container_caroot()`: Locates `rootCA.pem` by either:
     * Inspecting running devcontainer IDs via `docker exec`/`podman exec`
     * Falling back to `$HOME/.local/share/mkcert/ca` or `.devcontainer/mkcert/ca`
  3. Installs `mkcert` (and `libnss3-tools`/`nss-tools`) if missing.
  4. Runs `mkcert -install` pointing at the discovered CA root.
  5. `import_ca_to_nss_db()` & `import_ca_to_firefox_profiles()`: Uses `certutil` to inject the CA into system NSS databases and local Firefox profile directories.
  6. Cleans up temporary CA directories via `trap cleanup EXIT`.

## Implementation Notes & Patterns

* **Dry Run Support**: Refactoring scripts (`fix-kebab-groups.mjs`, `fix-kebab-imports.mjs`) consistently check `process.argv.includes('--dry-run')` to skip filesystem mutations while logging planned actions.
* **Path Resolution**: Both `fix-kebab-*.mjs` scripts rely on `node:path` methods (`resolve`, `relative`, `normalize`, `dirname`) and handle extension/index resolution explicitly to match Node.js module resolution semantics.
* **Authentication Safety**: `publish-packages.mjs` writes temporary npm auth configs only in memory/disk during execution and guarantees cleanup via `process.on('exit')` and signal handlers to prevent credential leakage in CI artifacts.
* **Idempotency**: Bash scripts (`scaffold-empty-packages.sh`, `wire-package-dependencies.sh`, `stop-showcase.sh`) are designed to run safely multiple times without overwriting existing state or crashing on pre-existing processes.
