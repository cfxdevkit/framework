## @cfxdevkit/devnode

Local Conflux node lifecycle management. Wraps the native `@xcfx/node` binary to spawn a dual-space (Core + eSpace) chain with deterministic ports and pre-funded genesis accounts.

### Purpose

Bring up a local Conflux node for development and testing with sensible defaults that match `@cfxdevkit/cdk/chains`'s `core-local` (chainId 2029, port 12537) and `espace-local` (chainId 2030, port 8545).

### Key APIs

```typescript
import { createDevNode, DevNode } from '@cfxdevkit/devnode';

const node = createDevNode({
  mnemonic: 'test test test test test test test test test test test junk',
  accounts: 5,
  balanceCfx: '10000',
  coreRpcPort: 12537,
  evmRpcPort: 8545,
});

await node.start();
const status = node.getStatus(); // 'running' | 'stopped' | 'error'
await node.mine(1);
await node.stop();
```

### Configuration

| Option | Default | Description |
|--------|---------|-------------|
| `mnemonic` | Required | BIP-39 mnemonic for account derivation |
| `accounts` | 10 | Number of pre-funded dual-address accounts |
| `balanceCfx` | `'10000'` | Initial balance per account (in CFX) |
| `coreRpcPort` | `12537` | Core Space RPC port |
| `evmRpcPort` | `8545` | eSpace RPC port |
| `miningIntervalMs` | `1000` | Mining interval (0 for no mining) |
| `logging` | `false` | Enable node logging |

### Account Derivation

Accounts use separate BIP-44 coin types:
- **eSpace**: `m/44'/60'/0'/0/{index}` (coin type 60)
- **Core**: `m/44'/503'/0'/0/{index}` (coin type 503)

The faucet/mining account uses a separate derivation path (`m/44'/60'/1'/0/0`) to stay clear of user account indices.

### CLI

```bash
cfxdevkit-devnode --core-port 3000 --espace-port 4000 --accounts 5 --balance 500
```

| Flag | Description |
|------|-------------|
| `--core-port` | Core Space RPC port |
| `--espace-port` | eSpace RPC port |
| `--accounts` | Number of pre-funded accounts |
| `--balance` | Initial balance per account |
| `--no-mining` | Disable mining (interval = 0) |
| `--logging` | Enable node logging |
| `--help` | Show help |

### URLs

```typescript
node.urls.core      // http://127.0.0.1:12537
node.urls.espace    // http://127.0.0.1:8545
node.urls.coreWs    // ws://127.0.0.1:12536
```

### Errors

```typescript
import { DevNodeError, DevNodeErrorCode } from '@cfxdevkit/devnode';

try {
  await node.start();
} catch (error) {
  if (error instanceof DevNodeError) {
    console.error(error.code); // 'ALREADY_RUNNING' | 'NOT_RUNNING' | 'INVALID_CONFIG'
  }
}
```

## @cfxdevkit/executor

Generic background job runner with queues, schedulers, and concurrency control.

### Purpose

Execute tasks with retry policies, timeouts, abort signals, and optional locking. Useful for automation, polling, and batch processing.

### Key APIs

#### `execute`

Run a single task with retry, timeout, and abort support.

```typescript
import { execute } from '@cfxdevkit/executor';

const result = await execute(
  async (ctx) => {
    // ctx.attempt, ctx.signal, ctx.metadata
    return await doWork();
  },
  {
    attempts: 3,
    timeoutMs: 5000,
    delayMs: (attempt, error) => attempt * 1000,
    shouldRetry: (error, attempt) => attempt < 3,
    signal: abortController.signal,
  },
);

if (result.ok) {
  console.log(result.value, result.attempts, result.durationMs);
} else {
  console.error(result.error, result.attempts);
}
```

#### `executeBatch`

Run multiple tasks with concurrency control.

```typescript
import { executeBatch } from '@cfxdevkit/executor';

const results = await executeBatch(
  [
    () => task1(),
    () => task2(),
    () => task3(),
  ],
  { concurrency: 2, stopOnError: true },
);
```

#### `createTaskQueue`

Queue tasks for batch execution.

```typescript
import { createTaskQueue } from '@cfxdevkit/executor';

const queue = createTaskQueue({
  onResult: (result, index) => {
    if (!result.ok) console.error(result.error);
  },
});

queue.add(() => work1());
queue.add(() => work2());
await queue.run();
```

#### `createPoller`

Create a polling loop.

```typescript
import { createPoller } from '@cfxdevkit/executor';

const poller = createPoller(
  async ({ signal }) => {
    if (signal.aborted) return;
    await checkStatus();
  },
  1000, // intervalMs
);

poller.start();
await poller.stop();
```

#### `withLock`

Serialize tasks sharing the same key.

```typescript
import { withLock } from '@cfxdevkit/executor';

await Promise.all([
  withLock('job:1', async () => {
    await workA();
  }),
  withLock('job:1', async () => {
    await workB(); // Waits for workA to complete
  }),
]);
```

### Execution Result

```typescript
type ExecutionResult<T> =
  | { ok: true; value: T; attempts: number; durationMs: number }
  | { ok: false; error: unknown; attempts: number; durationMs: number };
```

## @cfxdevkit/protocol

Conflux protocol helpers, precompile ABIs, and reusable DevKit contract metadata.

### Purpose

Provide common utilities for interacting with Core Space and eSpace, including transaction receipt handling, gas estimation, and chain progress tracking.

### Key APIs

#### `waitForTransactionReceipt`

Wait for a transaction to be confirmed.

```typescript
import { waitForTransactionReceipt } from '@cfxdevkit/protocol';

const receipt = await waitForTransactionReceipt(client, hash, {
  intervalMs: 1000,
  timeoutMs: 60000,
  signal: abortController.signal,
});
```

#### `getChainProgress`

Get current chain height (epoch for Core, block for eSpace).

```typescript
import { getChainProgress } from '@cfxdevkit/protocol';

const progress = await getChainProgress(client);
// { family: 'core' | 'espace', chainId: number, height: bigint }
```

#### `estimateTransaction`

Estimate gas and collateral.

```typescript
import { estimateTransaction } from '@cfxdevkit/protocol';

const estimate = await estimateTransaction(client, {
  from: 'cfx:...',
  to: 'cfx:...',
  value: 1000000000000000000n,
});
// { gas: bigint, storageCollateral?: bigint }
```

#### `collectLogs`

Collect logs (Core Space only).

```typescript
import { collectLogs } from '@cfxdevkit/protocol';

const logs = await collectLogs(client, {
  fromEpoch: 100n,
  toEpoch: 200n,
  address: ['cfx:contract'],
});
```

#### `assertSuccessfulReceipt` / `isSuccessfulReceipt`

Check transaction status.

```typescript
import { assertSuccessfulReceipt, isSuccessfulReceipt } from '@cfxdevkit/protocol';

const receipt = await client.getTransactionReceipt(hash);
if (!isSuccessfulReceipt(receipt)) {
  throw new Error('Transaction reverted');
}
assertSuccessfulReceipt(receipt, hash); // Throws if failed
```

## @cfxdevkit/testing

Shared test fixtures and matchers for Conflux development.

### Purpose

Provide mock clients, DevNode fixtures, and utility functions for testing.

### Key APIs

#### `createMockClient`

Create a mock client for Core or eSpace.

```typescript
import { createMockClient } from '@cfxdevkit/testing';

const client = createMockClient({
  family: 'core',
  chainId: 2029,
  receipts: new Map([['0x1', receipt]]),
  logs: [{ address: 'cfx:test' }],
  request: (method, params) => ({ method, params }),
});

await client.getEpochNumber();
await client.getLogs({});
await client.request({ method: 'cfx_getStatus' });
```

#### `createDevNodeFixture`

Create a DevNode instance for testing.

```typescript
import { createDevNodeFixture } from '@cfxdevkit/testing';

const node = await createDevNodeFixture({
  mnemonic: 'test...',
  accounts: 5,
  autoStart: true, // Start node automatically
});

// node.isRunning(), node.start(), node.stop(), etc.
```

#### `createDeferred`

Create a deferred promise.

```typescript
import { createDeferred } from '@cfxdevkit/testing';

const deferred = createDeferred<string>();
deferred.resolve('ok');
await deferred.promise;
```

#### `waitFor`

Wait for a condition to be true.

```typescript
import { waitFor } from '@cfxdevkit/testing';

await waitFor(() => someCondition(), {
  timeoutMs: 5000,
  intervalMs: 100,
});
```

## Notes

- **Node.js Only**: `devnode` requires Node.js (native binary dependency)
- **Dual-Space**: Core (chainId 2029) and eSpace (chainId 2030) run simultaneously
- **Deterministic**: Default ports and chain IDs match `@cfxdevkit/cdk/chains`
- **Test-Only**: `devnode` and `testing` packages are for development/testing, not production
