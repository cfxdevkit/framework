# repos — cfx-core

# @cfxdevkit/cfx-core Module

## Overview & Package Layout

The `cfx-core` module provides the foundational primitives and development tooling for the Conflux ecosystem. It is organized as a monorepo with two primary packages:

| Package | Purpose | Key Entry Points |
|---------|---------|------------------|
| `@cfxdevkit/cdk` | Core library: address codecs, chain catalog, dual-family RPC clients, typed errors, unit math, and wallet/signing utilities | `@cfxdevkit/cdk/client`, `chains`, `address`, `wallet`, `errors`, `units`, `types` |
| `@cfxdevkit/devnode` | CLI & programmatic API for managing a local `@xcfx/node` instance during development/testing | `createDevNode`, `DevNode`, `parseArgs` |

Both packages target Node.js ≥22, use ES modules exclusively, and are built with Vite + `vite-plugin-dts` for tree-shakable ESM outputs and declaration files.

## Core Architecture & Client Dispatch

The RPC client layer abstracts over two distinct underlying libraries: `viem` for eSpace (EVM-compatible) and `cive` for Core Space (Conflux-native). `createClient({ chain, transport })` inspects `chain.family` and delegates to either `createEspaceClient` or `createCoreClient`.

Both client implementations normalize all underlying library calls through `wrapRpc`, which catches rejections and promotes non-`CfxError` instances to structured `RpcError` objects. For "not found" scenarios, `nullWhenNotFound` converts specific error names (`TransactionReceiptNotFoundError`, `TransactionNotFoundError`) into `null` instead of throwing.

### Transport Layer
The transport abstraction (`http()`, `ws()`, `fallback()`) returns an opaque `Transport` object containing both `_viem` and `_cive` transport instances. Options are forwarded to the underlying libraries with a unified interface:
- `HttpTransportOptions`: `url`, `headers`, `timeoutMs`, `retries`
- `WsTransportOptions`: `url`, `reconnect`, `timeoutMs`

`fallback()` validates that at least one transport is provided, then composes the underlying viem/cive fallback transports.

## Sub-Module Reference

### Address Codec (`@cfxdevkit/cdk/address`)
Delegates to `cive/utils` for base32 ↔ hex conversion. Handles:
- `hexToBase32(hex, networkId, opts?)`: Prefixes with `cfx:`, `cfxtest:`, or `net<id>:` based on `networkId`. Supports `{ verbose: true }` for uppercase verbose forms.
- `base32ToHex(address, opts?)`: Converts back to `0x`-prefixed hex. `{ strict: false }` relaxes case validation.
- `isBase32Address(address)`: Returns `boolean` for Conflux base32 validation.
- `getCoreAddress(address)`: Canonicalizes prefix and body casing; throws on malformed input.

### Chain Catalog (`@cfxdevkit/cdk/chains`)
Pure data exports for six pre-configured chains (`coreSpaceMainnet`, `coreSpaceTestnet`, `coreSpaceLocal`, `espaceMainnet`, `espaceTestnet`, `espaceLocal`). 
- `getChain(idOrName)`: O(1) lookup via internal `Map`s. Throws `CfxError` with code `core/chains/unknown` if not found.
- `listChains(filter?)`: Filters by `family` and/or `network`.
- `defineChain(input)`: Identity validator for user-supplied configs. Enforces `id > 0` and `rpc.http.length > 0`.

### Types (`@cfxdevkit/cdk/types`)
Re-exports EVM primitives from `viem` (`Address`, `Hash`, `TxReceipt`, `BlockTag`) and defines Conflux-specific types:
- `EpochTag`: `'earliest' | 'latest_checkpoint' | 'latest_finalized' | 'latest_mined' | 'latest_state'`
- `CoreLogFilter`: Supports epoch range, block range, or `blockHashes` windows.
- `SponsorInfo`, `NodeStatus`, `CoreLog`: Conflux-native RPC response shapes.
- All monetary amounts use `bigint` (`Wei`).

### Units (`@cfxdevkit/cdk/units`)
BigInt-only decimal math. Wraps `viem`'s `parseUnits`/`formatUnits` with Conflux-specific aliases:
- `parseCFX` / `formatCFX`: 18 decimals
- `parseDrip` / `formatDrip`: 18 decimals (semantic alias)
- `parseGDrip` / `formatGDrip`: 9 decimals (gas-price unit)
- `stringifyBigInt`: JSON replacer that serializes `bigint` as decimal strings to avoid `TypeError` during `JSON.stringify`.

## Error Handling Strategy

The framework enforces a strict `CfxError` hierarchy. All framework-generated errors extend `CfxError`, which carries:
- `code`: Stable, namespaced identifier (e.g., `core/rpc/request`, `core/wallet/derivation`, `core/chains/unknown`)
- `message`: Human-readable description
- `cause`: Original underlying error
- `meta`: Optional structured context (sanitized via `toJSON()`)

Subclasses include `RpcError`, `ContractError`, `WalletError`, and `KeystoreError`. The `isCfxError(value)` type guard is the standard boundary check. RPC wrappers (`wrapRpc`) automatically promote non-`CfxError` rejections to `RpcError`. Execution flows like `signMessage` or `signTypedData` catch underlying library errors and re-throw them as `WalletError` via `signRejected`.

## Wallet & Derivation System

Wallet operations center on deterministic BIP-39/BIP-32 derivation and dual-family signing:

- **Mnemonic**: `generateMnemonic(strength?)` and `validateMnemonic(mnemonic)` use `@scure/bip39`.
- **Derivation**: `deriveAccount({ mnemonic, path?, passphrase? })` defaults to `m/44'/503'/0'/0/0` (Core). `deriveDualAccount` synchronizes eSpace (`m/44'/60'/...`) and Core (`m/44'/503'/...`) addresses for a given `index`. Supports `accountType: 'mining'` (branch `1'`) and configurable `coreNetworkId`.
- **Signing**: `signerFromPrivateKey(privateKey, coreNetworkId?)` returns a `Signer` interface:
  - `signTransaction(tx)`: Routes to `viem` for EVM or `signCoreTransaction` for Core Space.
  - `signMessage(message)` / `signTypedData(typedData)`: Delegates to `viem`, catches rejections, and throws `WalletError` via `signRejected`.
- **Core Signing Validation**: `signCoreTransaction(privateKey, tx)` enforces required fields (`epochHeight`, `nonce`, `gas`, `storageLimit`). Type-specific validation ensures `gasPrice` is present for `legacy`/`cip2930`, while `cip1559` requires `maxFeePerGas` + `maxPriorityFeePerGas`.

## Local Node Lifecycle (`@cfxdevkit/devnode`)

`createDevNode(config)` wraps the native `@xcfx/node` binary for deterministic test/dev environments:

1. **Config Resolution**: Applies defaults matching `coreSpaceLocal`/`espaceLocal` (`coreRpcPort: 12537`, `evmRpcPort: 8545`, `accounts: 10`, `miningIntervalMs: 500`). Validates mnemonic via BIP-39.
2. **Account Derivation**: Calls `createAccounts` → `deriveDualAccount` to generate genesis accounts and a dedicated faucet/mining account (branch `1'`).
3. **Lifecycle Management**: 
   - `start()`: Spawns `@xcfx/node`, creates data directory, funds accounts, and optionally starts auto-mining.
   - `stop()`: Gracefully terminates the server.
   - `mine(n)`: Triggers a single manual mine cycle.
   - `startMining(intervalMs)`: Sets up an interval timer with `autoMineInFlight` guard to prevent concurrent mine calls.
   - `isRunning()`, `getStatus()`, `getMiningStatus()`: State introspection.
4. **CLI**: `cfxdevkit-devnode` parses `--core-port`, `--accounts`, `--no-mining`, `--mnemonic`, etc., and prints connection URLs, mnemonic, and funded accounts. Handles `SIGINT`/`SIGTERM` for clean shutdown.

## Cross-Module Dependencies & Integration

`@cfxdevkit/cdk` acts as the canonical primitive layer for the broader framework. Key integration points observed in call graphs:
- **Address Encoding**: `signerFromOneKey`, `crossSpaceCallAddress`, and hardware wallet backends call `hexToBase32`, `base32ToHex`, and `isBase32Address` to resolve Conflux addresses from EVM payloads.
- **Error Propagation**: Keystore backends (`keystore/ledger`, `keystore/file`, `keystore/memory`) and protocol layers catch `CfxError` subclasses (`KeystoreError`, `RpcError`) to surface structured failures to UI/executor layers.
- **RPC Delegation**: Protocol and keeper clients (`#sendContractTransaction`, `handleBlockchainTool`) consume `getBlock`, `estimateGas`, and `getBlockNumber` through the unified client interface, relying on `wrapRpc` for consistent error handling.

## Build & Test Workflow

- **Build**: `pnpm run build` runs Vite in library mode. Entry points map to sub-paths. Output is ESM-only with sourcemaps. TypeScript declaration generation is handled by `vite-plugin-dts`.
- **Testing**: `pnpm run test` runs Vitest in Node environment. Coverage uses V8 provider. Network smoke tests are gated behind `RUN_NETWORK_TESTS=1`.
- **Validation**: Workspace-level scripts (`validate:workspace`, `validate:deps`) enforce monorepo consistency. Biome linting (`biome check src`) and strict TypeScript checks (`tsc --noEmit`) are mandatory for contributions.
