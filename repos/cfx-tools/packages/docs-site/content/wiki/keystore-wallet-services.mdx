# Keystore & Wallet Services

This module provides a pluggable, secure keystore and wallet management system for handling private keys, mnemonics, and signing operations across multiple backends.

## Purpose

The keystore system abstracts private key storage and signing operations into a unified interface (`KeystoreProvider`) that supports multiple backends:
- File-based encrypted storage
- In-memory (for testing)
- Hardware wallets (Ledger)
- OS keyring
- Cloud KMS/Vault
- Forwarded to remote services

It also provides wallet management for embedded wallets, which are created and managed via the keystore.

## Key Components

### 1. Keystore Provider Interface

The `KeystoreProvider` interface defines the contract for all keystore backends:

```ts
interface KeystoreProvider {
  readonly id: string;
  readonly capabilities: KeystoreCapabilities;
  list(opts?: KeystoreListOptions): Promise<StoredSecret[]>;
  has(ref: SecretRef, opts?: KeystoreCallOptions): Promise<boolean>;
  getSigner(ref: SecretRef, capability?: Capability, opts?: KeystoreCallOptions): Promise<Signer>;
  put?(input: KeystorePutInput, opts?: KeystoreCallOptions): Promise<void>;
  updateMeta?(ref: SecretRef, meta: Record<string, string>, opts?: KeystoreCallOptions): Promise<void>;
  remove?(ref: SecretRef, opts?: KeystoreCallOptions): Promise<void>;
  rotate?(ref: SecretRef, opts?: KeystoreCallOptions): Promise<{ ref: SecretRef }>;
}
```

### 2. Authentication Services

#### Nonce Management
The `MemoryNonceStore` provides in-memory storage for nonces used in authentication flows:

```ts
class MemoryNonceStore {
  issue(address: string): string;
  peek(nonce: string): NonceEntry | null;
  consume(nonce: string, address: string): boolean;
  sweepExpired(now?: number): void;
}
```

#### Session Tokens
Session tokens are used for secure authentication:

```ts
function signSessionToken(address: string, options: SessionTokenOptions): string;
function verifySessionToken(token: string, options: VerifySessionTokenOptions): SessionToken | null;
function readBearerToken(header: string | null | undefined): string | null;
```

### 3. Cryptographic Utilities

#### AES-256-GCM Encryption
Secure encryption/decryption using AES-256-GCM:

```ts
async function encryptAesGcm(input: EncryptInput): Promise<EncryptOutput>;
async function decryptAesGcm(input: DecryptInput): Promise<Uint8Array>;
```

#### Key Derivation Functions
- Argon2id for password-based key derivation
- HKDF for key derivation from existing keys

```ts
async function deriveKeyArgon2id(input: Argon2idInput): Promise<AesGcmKey>;
async function deriveKeyHkdf(input: HkdfInput): Promise<Uint8Array>;
```

#### Random Number Generation
Cryptographically secure random bytes:

```ts
function randomBytes(n: number): Uint8Array;
```

### 4. Embedded Wallet Management

The `KeystoreEmbeddedWalletManager` provides a high-level interface for managing embedded wallets:

```ts
class KeystoreEmbeddedWalletManager {
  createWallet(userId: string, password?: string): Promise<EmbeddedWallet>;
  hasWallet(userId: string): Promise<boolean>;
  getWallet(userId: string): Promise<EmbeddedWallet>;
  signerFor(userId: string, password?: string): Promise<Signer>;
  deleteWallet(userId: string, password?: string): Promise<void>;
}
```

## Backends

### File Keystore
`@cfxdevkit/services/keystore-file` — Encrypted file-based storage using AES-256-GCM and Argon2id.

### Memory Keystore
`@cfxdevkit/services/keystore-memory` — In-memory storage for testing only.

### Ledger Keystore
`@cfxdevkit/services/keystore-ledger` — Hardware wallet integration for Ledger devices.

## Usage Examples

### Creating a File Keystore

```ts
import { createFileKeystore } from '@cfxdevkit/services/keystore/file';

const keystore = createFileKeystore({
  path: '/path/to/keystore.json',
  unlock: async () => ({ passphrase: 'secret-passphrase' }),
});
```

### Creating an Embedded Wallet

```ts
import { createEmbeddedWalletManager } from '@cfxdevkit/services/embedded-wallet';

const manager = createEmbeddedWalletManager({
  provider: fileKeystore,
});

const wallet = await manager.createWallet('user123');
```

### Signing a Transaction

```ts
const signer = await keystore.getSigner({ service: 'wallet', account: 'user123' });
const signature = await signer.signTransaction(transaction);
```

## Security Considerations

- Private keys are never returned across the keystore boundary
- All encryption uses industry-standard algorithms (AES-256-GCM, Argon2id)
- Session tokens are time-bound and use HMAC for integrity
- Nonces are used to prevent replay attacks
- All operations support cancellation via `AbortSignal`
