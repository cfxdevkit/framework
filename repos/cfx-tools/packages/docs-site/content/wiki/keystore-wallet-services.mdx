# Keystore & Wallet Services

**Package**: `@cfxdevkit/services`

The Keystore & Wallet Services module provides a unified interface for secure secret storage, cryptographic operations, and wallet management. It supports multiple backend implementations (file-based encryption, memory for testing, Ledger hardware wallets) while maintaining a consistent API for key operations.

## Overview

This module implements a pluggable keystore architecture where:

1. **Private key material never crosses the provider boundary** — backends only return `Signer` objects
2. **Multiple storage backends** can be used interchangeably (file, memory, Ledger, OS keyring, KMS)
3. **Capability enforcement** allows constraining what operations a signer can perform
4. **Audit logging** provides tamper-evident records of keystore operations

## Keystore Provider Interface

All backend implementations must implement the `KeystoreProvider` interface defined in `src/keystore/index.ts`:

```typescript
export interface KeystoreProvider {
  readonly id: string;
  readonly capabilities: KeystoreCapabilities;
  
  list(opts?: KeystoreListOptions): Promise<StoredSecret[]>;
  has(ref: SecretRef, opts?: KeystoreCallOptions): Promise<boolean>;
  getSigner(ref: SecretRef, capability?: Capability, opts?: KeystoreCallOptions): Promise<Signer>;
  
  put?(input: KeystorePutInput, opts?: KeystoreCallOptions): Promise<void>;
  updateMeta?(ref: SecretRef, meta: Record<string, string>, opts?: KeystoreCallOptions): Promise<void>;
  remove?(ref: SecretRef, opts?: KeystoreCallOptions): Promise<void>;
  rotate?(ref: SecretRef, opts?: KeystoreCallOptions): Promise<{ ref: SecretRef }>;
}
```

### Key Concepts

| Concept | Description |
|---------|-------------|
| `SecretRef` | `{ service: string, account: string }` uniquely identifies a secret |
| `StoredSecret` | Public metadata only — never contains private key material |
| `Signer` | Bound to a secret, holds key material internally |
| `Capability` | Constraints on signer operations (chains, contracts, selectors, value limits, expiry) |

### Secret Reference Format

Secrets are identified by a composite key: `${ref.service}\0${ref.account}`

```typescript
const ref: SecretRef = { service: 'cfxdevkit', account: 'deployer' };
```

## Backend Implementations

### File-Based Keystore

**Location**: `src/keystore/file/index.ts`

Encrypts secrets to a JSON file using AES-256-GCM. Passphrase is derived via Argon2id.

**Initialization**:
```typescript
import { initFileKeystore } from '@cfxdevkit/services/keystore/file';

await initFileKeystore({
  path: './keystore.json',
  passphrase: 'strong-passphrase',
  kdf: { memKiB: 64 * 1024, iterations: 3, parallelism: 1 }
});
```

**Usage**:
```typescript
import { createFileKeystore } from '@cfxdevkit/services/keystore/file';

const provider = createFileKeystore({
  path: './keystore.json',
  unlock: async () => ({ passphrase: getPassphrase() }),
  audit: createAppendOnlyAuditLogger({ path: './audit.log' })
});
```

**Features**:
- AES-256-GCM encryption with separate IV and tag
- Argon2id key derivation (64 MiB default memory cost)
- Atomic writes via temp file + rename
- Append-only audit logging support

**Envelope Structure**:
```json
{
  "version": "cfx-v1",
  "kdf": {
    "name": "argon2id",
    "salt": "base64url-encoded-salt",
    "memKiB": 65536,
    "iterations": 3,
    "parallelism": 1
  },
  "secrets": {
    "service\0account": {
      "kind": "private-key",
      "createdAt": 1234567890,
      "meta": {},
      "iv": "base64url",
      "ct": "base64url",
      "tag": "base64url"
    }
  }
}
```

### Memory Keystore (Test Only)

**Location**: `src/keystore/memory/index.ts`

In-memory keystore for testing. **Never import in production** — throws if `NODE_ENV === 'production'`.

```typescript
import { createMemoryKeystore } from '@cfxdevkit/services/keystore/memory';

const provider = createMemoryKeystore({
  seed: [{
    ref: { service: 'test', account: 'key1' },
    privateKey: '0x...'
  }]
});
```

### Ledger Hardware Wallet

**Location**: `src/keystore/ledger/index.ts`

Supports both Conflux Core Space and Ethereum (eSpace) via Ledger devices.

**Setup**:
```typescript
import { createLedgerKeystore } from '@cfxdevkit/services/keystore/ledger';
import { createNodeHidLedgerTransport, createLedgerEthApp } from '@cfxdevkit/services/keystore/ledger';

const transport = await createNodeHidLedgerTransport();
const ethApp = await createLedgerEthApp(transport);

const provider = createLedgerKeystore({
  eth: ethApp,
  accounts: [{
    ref: { service: 'ledger', account: 'espace-0' },
    family: 'espace',
    path: "m/44'/60'/0'/0/0"
  }]
});
```

**Supported Operations**:
- Conflux Core: Transaction signing (CIP-2930, CIP-1559, legacy), message signing (v2.3.0+)
- Ethereum: Transaction signing (EIP-1559), personal message signing, EIP-712 typed data

## Cryptographic Utilities

### AES-GCM Encryption/Decryption

**Location**: `src/crypto/aead.ts`

```typescript
import { encryptAesGcm, decryptAesGcm } from '@cfxdevkit/services/crypto';

// Encrypt
const { ciphertext, iv, tag } = await encryptAesGcm({
  key: aesKey,
  plaintext: data,
  aad: associatedData,  // Optional, bound to tag
  iv: customIv          // Optional, defaults to random 12 bytes
});

// Decrypt
const plaintext = await decryptAesGcm({
  key: aesKey,
  ciphertext,
  iv,
  tag,
  aad: associatedData
});
```

**Constants**:
- `GCM_IV_LEN = 12` bytes
- `GCM_TAG_LEN = 16` bytes
- `AES_KEY_LEN = 32` bytes (256-bit)

### Key Derivation

**Location**: `src/crypto/kdf.ts`

```typescript
import { deriveKeyArgon2id, deriveKeyHkdf } from '@cfxdevkit/services/crypto';

// Argon2id for passphrase-to-key
const key = await deriveKeyArgon2id({
  passphrase: 'user-passphrase',
  salt: randomBytes(16),
  memKiB: 64 * 1024,
  iterations: 3,
  parallelism: 1
});

// HKDF for key expansion
const expanded = await deriveKeyHkdf({
  ikm: masterKey,
  salt: contextSalt,
  info: 'domain-separation-context',
  length: 32
});
```

### Encoding Utilities

**Location**: `src/crypto/encoding.ts`

## Authentication & Session Tokens

**Location**: `src/auth/`

### Session Tokens

HMAC-based session tokens with configurable TTL.

```typescript
import { signSessionToken, verifySessionToken, readBearerToken } from '@cfxdevkit/services/auth';

// Sign
const token = signSessionToken('user-address', {
  secret: 'shared-secret',
  ttlMs: 24 * 60 * 60 * 1000,
  claims: { role: 'admin' }
});

// Verify
const payload = verifySessionToken(token, {
  secret: 'shared-secret',
  now: Date.now
});

// Extract from Authorization header
const token = readBearerToken('Bearer eyJhbGc...');
```

**Token Structure**: `base64url(payload).base64url(hmac)`

### Nonce Store

Prevents replay attacks with one-time nonces.

```typescript
import { createMemoryNonceStore } from '@cfxdevkit/services/auth';

const store = createMemoryNonceStore({ ttlMs: 5 * 60 * 1000 });

// Issue nonce
const nonce = store.issue('user-address');

// Consume (validates address, marks used, checks expiry)
const valid = store.consume(nonce, 'user-address');
```

## Embedded Wallet Management

**Location**: `src/embedded-wallet/`

Manages wallets stored in the keystore, providing higher-level wallet operations.

```typescript
import { createEmbeddedWalletManager } from '@cfxdevkit/services/embedded-wallet';

const manager = createEmbeddedWalletManager({
  provider: keystoreProvider,
  service: 'cfxdevkit-embedded-wallet',
  coreNetworkId: 1029
});

// Create wallet
const wallet = await manager.createWallet('user-id-123');

// Check existence
const exists = await manager.hasWallet('user-id-123');

// Get signer
const signer = await manager.signerFor('user-id-123');

// Delete wallet
await manager.deleteWallet('user-id-123');
```

**Wallet Metadata**:
- `espaceAddress`: EVM-compatible address
- `coreAddress`: Conflux Core space address
- `createdAt`: Unix timestamp
- `ref`: Secret reference for keystore lookup

## Ledger Hardware Wallet Support

### Core Space (Conflux)

Supports multiple transaction types:

| Type | Description |
|------|-------------|
| CIP-1559 | EIP-1559 style transactions |
| CIP-2930 | Access list transactions |
| Legacy | Classic format |

### APDU Communication

**Location**: `src/keystore/ledger/core-framing.ts`

Handles chunked APDU communication for large payloads:

```typescript
// Standard exchange (payload < 255 bytes)
const response = await exchange(transport, ins, p1, p2, data);

// Chunked exchange (payload >= 255 bytes)
const response = await exchangeChunks(transport, ins, prefix, data);
```

### Common Status Codes

| Status | Meaning |
|--------|---------|
| `0x9000` | Success |
| `0x6a86` | Wrong parameters (e.g., INS=0x04 not exposed) |
| `0x6985` | Operation rejected on device |
| `0x6b00` | Wrong parameters (payload length) |

## Audit Logging

**Location**: `src/keystore/audit.ts`

Append-only audit logger with SHA-256 chaining for tamper detection.

```typescript
import { createAppendOnlyAuditLogger } from '@cfxdevkit/services/keystore';

const audit = createAppendOnlyAuditLogger({
  path: './audit.log',
  onError: (error) => console.error('Audit write failed', error)
});

// Pass to keystore provider
const provider = createFileKeystore({
  path: './keystore.json',
  unlock: () => ({ passphrase }),
  audit
});
```

**Entry Format**:
```json
{
  "at": 1234567890,
  "provider": "file",
  "action": "put",
  "ref": { "service": "cfxdevkit", "account": "deployer" },
  "ok": true,
  "sequence": 42,
  "previousHash": "sha256-of-previous-entry",
  "entryHash": "sha256-of-current-entry"
}
```

## Capability Enforcement

**Location**: `src/keystore/memory/capability.ts`

Apply constraints to signers at runtime:

```typescript
import { applyCapability } from '@cfxdevkit/services/keystore/memory';

const limitedSigner = applyCapability(baseSigner, {
  chains: [1029],
  contracts: ['0x...'],
  selectors: ['0xa9059cbb'],  // transfer function
  maxValuePerTx: 1000000n,
  notAfter: Date.now() + 24 * 60 * 60 * 1000
});
```

**Enforcement Points**:
- Chain ID restrictions
- Contract address allowlists
- Function selector allowlists
- Maximum transaction value
- Expiration timestamps

## Error Handling

All errors extend `KeystoreError` from `@cfxdevkit/cdk`:

```typescript
import { KeystoreError } from '@cfxdevkit/cdk';

try {
  await provider.getSigner(ref);
} catch (error) {
  if (error instanceof KeystoreError) {
    console.error(error.code, error.message);
  }
}
```

**Common Error Codes**:
- `services/keystore/not-found`: Secret doesn't exist
- `services/keystore/bad-passphrase`: Decryption failed
- `services/keystore/unsupported`: Operation not supported by backend
- `services/keystore/ledger/core-apdu-error`: Ledger communication failed

## Security Considerations

1. **Private keys never leave the provider** — only `Signer` objects are returned
2. **File keystore uses atomic writes** — prevents corruption on crash
3. **Argon2id with high memory cost** — resists brute-force attacks
4. **Aad binding** — decryption fails if reference changes
5. **Audit logging** — tamper-evident operation records
6. **Capability enforcement** — runtime constraints on signer behavior
