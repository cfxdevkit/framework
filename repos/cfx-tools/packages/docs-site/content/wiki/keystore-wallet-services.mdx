# Keystore & Wallet Services

The `@cfxdevkit/services` module provides authentication primitives, cryptographic utilities, and a pluggable keystore system for managing cryptographic secrets and wallet operations. It is designed to support secure key storage across multiple backends while maintaining a consistent interface for signing operations.

## Core Concepts

### Keystore Provider

The `KeystoreProvider` interface is the central abstraction for secret management. It ensures private key material never crosses the provider boundary—only `Signer` objects are returned for cryptographic operations.

```typescript
export interface KeystoreProvider {
  readonly id: string;
  readonly capabilities: KeystoreCapabilities;
  
  list(opts?: KeystoreListOptions): Promise<StoredSecret[]>;
  has(ref: SecretRef, opts?: KeystoreCallOptions): Promise<boolean>;
  getSigner(ref: SecretRef, capability?: Capability, opts?: KeystoreCallOptions): Promise<Signer>;
  put?(input: KeystorePutInput, opts?: KeystoreCallOptions): Promise<void>;
  remove?(ref: SecretRef, opts?: KeystoreCallOptions): Promise<void>;
  updateMeta?(ref: SecretRef, meta: Record<string, string>, opts?: KeystoreCallOptions): Promise<void>;
}
```

### Secret Reference

Each secret is identified by a `SecretRef` consisting of a service namespace and account name:

```typescript
export interface SecretRef {
  service: string;  // e.g., "cfxdevkit", "ledger"
  account: string;  // e.g., "deployer", "espace-0"
}
```

### Capabilities

Signers can be wrapped with capability constraints that enforce restrictions at sign time:

```typescript
export interface Capability {
  chains?: ChainId[];           // Permitted chain IDs
  contracts?: Address[];        // Permitted contract targets
  selectors?: Hex[];            // Permitted function selectors
  maxValuePerTx?: bigint;       // Maximum transaction value
  notAfter?: Timestamp;         // Expiration timestamp
}
```

## Authentication

### Session Tokens

Session tokens are signed, time-limited credentials for authentication. They use HMAC-SHA256 for integrity verification.

```typescript
// Sign a session token for an address
const token = signSessionToken(address, {
  secret: process.env.SESSION_SECRET!,
  ttlMs: 24 * 60 * 60 * 1000,  // 24 hours default
  claims: { role: 'admin' }
});

// Verify a session token
const payload = verifySessionToken(token, {
  secret: process.env.SESSION_SECRET!
});

if (payload) {
  console.log(payload.address, payload.expiresAt);
}
```

### Nonce Management

The `MemoryNonceStore` provides one-time-use nonces for preventing replay attacks. Nonces are automatically expired based on TTL.

```typescript
import { createMemoryNonceStore } from '@cfxdevkit/services';

const store = createMemoryNonceStore({ ttlMs: 5 * 60 * 1000 });

// Issue a new nonce for an address
const nonce = store.issue('0x123...');

// Consume the nonce (validates address, marks as used)
const valid = store.consume(nonce, '0x123...');
```

## Cryptographic Utilities

### AES-GCM Encryption

The module provides authenticated encryption using AES-256-GCM:

```typescript
import { encryptAesGcm, decryptAesGcm, generateAesGcmKey } from '@cfxdevkit/services';

// Generate a key
const key = generateAesGcmKey();

// Encrypt
const { ciphertext, iv, tag } = await encryptAesGcm({
  key,
  plaintext: new TextEncoder().encode('secret data'),
  aad: new TextEncoder().encode('associated data')
});

// Decrypt
const plaintext = await decryptAesGcm({
  key,
  ciphertext,
  iv,
  tag,
  aad: new TextEncoder().encode('associated data')
});
```

### Key Derivation

Two key derivation methods are available:

```typescript
import { deriveKeyArgon2id, deriveKeyHkdf } from '@cfxdevkit/services';

// Argon2id for passphrase-based derivation (recommended for user passwords)
const aesKey = await deriveKeyArgon2id({
  passphrase: 'user-password',
  salt: randomBytes(16),
  memKiB: 64 * 1024,    // 64 MiB memory cost
  iterations: 3,
  parallelism: 1
});

// HKDF-SHA256 for key expansion
const derived = await deriveKeyHkdf({
  ikm: aesKey,
  salt: randomBytes(16),
  info: new TextEncoder().encode('context'),
  length: 32
});
```

### Encoding Utilities

```typescript
import { toHex, fromHex, toBase64Url, fromBase64Url } from '@cfxdevkit/services';

const bytes = new Uint8Array([1, 2, 3]);
const hex = toHex(bytes);           // '0x010203'
const back = fromHex(hex);

const base64 = toBase64Url(bytes);  // URL-safe base64
const back2 = fromBase64Url(base64);
```

## Keystore Backends

### File Keystore

The file backend stores encrypted secrets in a JSON envelope protected by Argon2id-derived keys.

```typescript
import { createFileKeystore, initFileKeystore } from '@cfxdevkit/services';

// Initialize a new keystore
await initFileKeystore({
  path: './keystore.json',
  passphrase: 'master-password'
});

// Create a provider instance
const provider = createFileKeystore({
  path: './keystore.json',
  unlock: async () => ({ passphrase: 'master-password' }),
  audit: createAppendOnlyAuditLogger({ path: './audit.log' })
});

// Store a private key
await provider.put({
  ref: { service: 'cfxdevkit', account: 'deployer' },
  kind: 'private-key',
  secret: '0x...'
});

// Get a signer for signing operations
const signer = await provider.getSigner({ service: 'cfxdevkit', account: 'deployer' });
```

**Features:**
- AES-GCM encryption with per-secret AAD binding
- Atomic writes via temp file + rename
- Optional audit logging
- Passphrase rotation via `changeFilePassphrase`

### Ledger Keystore

The Ledger backend supports hardware wallet signing for both Conflux eSpace and Core Space.

```typescript
import { createLedgerKeystore, createNodeHidLedgerTransport, createLedgerEthApp } from '@cfxdevkit/services';

// Set up transport and app
const transport = await createNodeHidLedgerTransport();
const ethApp = await createLedgerEthApp(transport);

// Create provider with configured accounts
const provider = createLedgerKeystore({
  eth: ethApp,
  accounts: [
    {
      ref: { service: 'ledger', account: 'espace-0' },
      family: 'espace',
      path: "m/44'/60'/0'/0/0",
      showAddressOnDevice: true
    }
  ]
});

// Get signer (requires Ledger device interaction)
const signer = await provider.getSigner({ service: 'ledger', account: 'espace-0' });
```

**Supported Operations:**
- eSpace: EIP-1559 transactions, personal messages, EIP-712 typed data
- Core Space: Core transactions (message signing requires app v2.3.0+)

### Memory Keystore

The memory backend stores secrets in RAM only. **Production guard:** throws if `NODE_ENV === 'production'`.

```typescript
import { createMemoryKeystore } from '@cfxdevkit/services';

const provider = createMemoryKeystore({
  seed: [
    {
      ref: { service: 'test', account: 'user1' },
      privateKey: '0x...'
    }
  ]
});
```

## Embedded Wallet Manager

The embedded wallet layer simplifies wallet lifecycle management on top of any keystore provider.

```typescript
import { createEmbeddedWalletManager } from '@cfxdevkit/services';

const manager = createEmbeddedWalletManager({
  provider,  // Any KeystoreProvider implementation
  service: 'cfxdevkit-embedded-wallet',
  coreNetworkId: 1029
});

// Create a new wallet (generates keypair, stores in keystore)
const wallet = await manager.createWallet('user-123');

// Check if wallet exists
const exists = await manager.hasWallet('user-123');

// Get wallet info (without exposing private key)
const walletInfo = await manager.getWallet('user-123');

// Get signer for operations
const signer = await manager.signerFor('user-123');

// Delete wallet
await manager.deleteWallet('user-123');
```

**Wallet Metadata:**
- `userId`: User identifier
- `ref`: Secret reference for keystore lookup
- `espaceAddress`: eSpace address (0x-prefixed)
- `coreAddress`: Core address
- `createdAt`: Unix timestamp

## Audit Logging

The audit system provides append-only logging of keystore operations:

```typescript
import { createAppendOnlyAuditLogger } from '@cfxdevkit/services';

const audit = createAppendOnlyAuditLogger({
  path: './audit.log',
  onError: (error) => console.error('Audit error:', error)
});

const provider = createFileKeystore({
  path: './keystore.json',
  unlock: async () => ({ passphrase: '...' }),
  audit
});
```

**Entry Format:**
```json
{
  "at": 1234567890123,
  "provider": "file",
  "action": "getSigner",
  "ref": { "service": "cfxdevkit", "account": "deployer" },
  "ok": true,
  "entryHash": "...",
  "previousHash": "...",
  "sequence": 1
}
```

## Error Handling

All operations throw `KeystoreError` or `CryptoError` with structured error codes:

```typescript
import { KeystoreError, CryptoError } from '@cfxdevkit/services';

try {
  await provider.getSigner({ service: 'cfxdevkit', account: 'unknown' });
} catch (error) {
  if (error instanceof KeystoreError) {
    console.log(error.code);  // e.g., 'services/keystore/not-found'
    console.log(error.message);
    console.log(error.meta);   // Additional context
  }
}
```

**Common Error Codes:**
- `services/keystore/not-found`: Secret does not exist
- `services/keystore/bad-passphrase`: Decryption failed
- `services/keystore/unsupported`: Feature not supported by backend
- `services/keystore/backend-unavailable`: I/O or file access error
- `services/crypto/bad-key`: Invalid key material
- `services/crypto/decrypt-failed`: Decryption/authentication failed

## Best Practices

1. **Never expose private keys**: All backends return `Signer` objects, not raw key material.
2. **Use capability constraints**: Wrap signers with `Capability` objects for least-privilege access.
3. **Enable audit logging**: Production deployments should use `createAppendOnlyAuditLogger`.
4. **Rotate passphrases**: Use `changeFilePassphrase` to rotate master passwords without re-encrypting all secrets.
5. **Handle cancellation**: All async methods honor `AbortSignal` via `KeystoreCallOptions`.
6. **Test with memory keystore**: Use `createMemoryKeystore` in tests (never in production).
7. **Secure file permissions**: File keystore uses `0o600` mode for keystore files.
