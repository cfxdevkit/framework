# Keystore & Wallet Services

## Overview

The Keystore & Wallet Services module provides a pluggable backend system for secure secret storage and cryptographic signing operations. It implements a unified `KeystoreProvider` interface that abstracts over multiple storage backends (file, memory, Ledger hardware wallet, OS keyring, KMS/Vault, forwarded) while ensuring private key material never crosses the provider boundary.

### Design Principles

- **Secret Isolation**: Private keys are never returned from keystore operations. Only `Signer` objects from `@cfxdevkit/cdk/wallet` are exposed.
- **Pluggable Backends**: All storage mechanisms implement the same `KeystoreProvider` contract.
- **Capability Enforcement**: Signers can be wrapped with capability constraints (chain restrictions, contract whitelists, value limits, expiration).
- **Audit Trail**: Optional append-only audit logging for all keystore operations.
- **Cancellation Support**: All operations honor `AbortSignal` for cancellation.

## Core Interfaces

### KeystoreProvider

The central contract that all keystore backends implement:

```typescript
export interface KeystoreProvider {
  readonly id: string;
  readonly capabilities: KeystoreCapabilities;
  
  list(opts?: KeystoreListOptions): Promise<StoredSecret[]>;
  has(ref: SecretRef, opts?: KeystoreCallOptions): Promise<boolean>;
  getSigner(ref: SecretRef, capability?: Capability, opts?: KeystoreCallOptions): Promise<Signer>;
  
  // Optional methods (omit on read-only backends)
  put?(input: KeystorePutInput, opts?: KeystoreCallOptions): Promise<void>;
  updateMeta?(ref: SecretRef, meta: Record<string, string>, opts?: KeystoreCallOptions): Promise<void>;
  remove?(ref: SecretRef, opts?: KeystoreCallOptions): Promise<void>;
  rotate?(ref: SecretRef, opts?: KeystoreCallOptions): Promise<{ ref: SecretRef }>;
}
```

### SecretRef

Stable identifier for stored secrets:

```typescript
export interface SecretRef {
  service: string;  // Logical namespace, e.g. "cfxdevkit"
  account: string;  // Secret name within service, e.g. "deployer"
}
```

### Capability

Optional constraints applied when building a `Signer`:

```typescript
export interface Capability {
  chains?: ChainId[];           // Allowed chain IDs
  contracts?: Address[];        // Allowed contract addresses
  selectors?: Hex[];            // Allowed 4-byte function selectors
  maxValuePerTx?: bigint;       // Maximum transaction value
  notAfter?: Timestamp;         // Expiration timestamp
}
```

## Crypto Layer

### AES-256-GCM Encryption

Located in `src/crypto/aead.ts`. Provides authenticated encryption with separate IV and tag storage:

```typescript
export async function encryptAesGcm(input: EncryptInput): Promise<EncryptOutput>
export async function decryptAesGcm(input: DecryptInput): Promise<Uint8Array>
```

**Key Points:**
- IV is 12 bytes (generated randomly if not provided)
- Tag is 16 bytes (returned separately from ciphertext)
- AAD (Additional Authenticated Data) binds the secret reference to the encryption
- Throws `CryptoError` on tag mismatch or corrupt input

### Key Derivation

Located in `src/crypto/kdf.ts`. Supports two KDFs:

```typescript
// Argon2id for passphrase-based derivation
export async function deriveKeyArgon2id(input: Argon2idInput): Promise<AesGcmKey>

// HKDF-SHA256 for key expansion
export async function deriveKeyHkdf(input: HkdfInput): Promise<Uint8Array>
```

**Argon2id Defaults:**
- Memory: 64 MiB
- Iterations: 3
- Parallelism: 1
- Salt: Minimum 8 bytes

### Key Branding

Located in `src/crypto/keys.ts`. Uses TypeScript branded types to prevent key misuse:

```typescript
export type AesGcmKey = Uint8Array & { readonly __brand: 'AesGcmKey' };

export function brandKey(bytes: Uint8Array): AesGcmKey;
export function generateAesGcmKey(): AesGcmKey;
```

### Encoding Utilities

Located in `src/crypto/encoding.ts`:

| Function | Description |
|----------|-------------|
| `toHex(bytes)` | Bytes → 0x-prefixed hex |
| `fromHex(hex)` | 0x-prefixed hex → bytes (throws on invalid) |
| `toBase64Url(bytes)` | Bytes → URL-safe base64 (no padding) |
| `fromBase64Url(text)` | URL-safe base64 → bytes |

## File Keystore

### Overview

Located in `src/keystore/file/`. Stores encrypted secrets in a JSON envelope file protected by a passphrase-derived key.

### Envelope Format

```typescript
export interface Envelope {
  version: 'cfx-v1';
  kdf: {
    name: 'argon2id';
    salt: string;           // Base64Url-encoded
    memKiB: number;
    iterations: number;
    parallelism: number;
  };
  secrets: Record<string, EncryptedSecret>;
}

export interface EncryptedSecret {
  kind: 'private-key' | 'mnemonic';
  createdAt: number;
  meta?: Record<string, string>;
  iv: string;               // Base64Url-encoded
  ct: string;               // Base64Url-encoded ciphertext
  tag: string;              // Base64Url-encoded tag
}
```

### Key Operations

```typescript
// Initialize new keystore
export async function initFileKeystore(input: {
  path: string;
  passphrase: string;
  kdf?: KdfParams;
}): Promise<void>;

// Create keystore provider
export function createFileKeystore(opts: FileKeystoreOptions): KeystoreProvider;

// Read mnemonic (for wallet recovery)
export async function readFileKeystoreMnemonic(input: {
  path: string;
  passphrase: string;
  ref: SecretRef;
}): Promise<string>;

// Change passphrase (re-encrypts all secrets)
export async function changeFilePassphrase(input: {
  path: string;
  oldPassphrase: string;
  newPassphrase: string;
}): Promise<void>;
```

### File Operations

| Method | Description |
|--------|-------------|
| `list()` | Returns metadata for all stored secrets |
| `has(ref)` | Checks if a secret exists |
| `getSigner(ref, capability)` | Returns a `Signer` for the secret |
| `put(input)` | Stores a new secret (private-key or mnemonic) |
| `updateMeta(ref, meta)` | Updates secret metadata |
| `remove(ref)` | Deletes a secret |

### Security Notes

- File permissions set to `0o600` (owner read/write only)
- Atomic writes via temp file + rename
- Passphrase validation via "unlock and probe" (decrypts first secret to verify)
- Plaintext buffers are zeroed after use

## Ledger Keystore

### Overview

Located in `src/keystore/ledger/`. Supports both Ethereum (eSpace) and Conflux Core Space hardware wallet signing.

### Supported Families

| Family | App | Path Default |
|--------|-----|--------------|
| `espace` | Ethereum | `m/44'/60'/0'/0/0` |
| `core` | Conflux Core | `m/44'/523'/0'/0/0` |

### Key Operations

```typescript

export function createLedgerKeystore(opts: LedgerKeystoreOptions): KeystoreProvider;

// Create transport (Node.js)
export async function createNodeHidLedgerTransport(): Promise<LedgerTransportLike>;

// Create Ethereum app wrapper
```

### Transaction Signing

The Ledger backend handles three Core Space transaction types:

```typescript
// Transaction serialization
export function serializeCoreTransaction(tx: SignableTx): Hex;
export function finaliseCoreTransaction(tx: SignableTx, signature: LedgerSignature): Hex;

// Supported types
- cip1559 (EIP-1559)
- cip2930 (EIP-2930)
- legacy
```

### APDU Communication

Located in `src/keystore/ledger/core-apdu.ts` and `core-framing.ts`:

| Instruction | Purpose |
|-------------|---------|
| `0x01` | Get app info |
| `0x02` | Get public key/address |
| `0x03` | Sign transaction |
| `0x04` | Sign personal message (Core Space v2.3.0+) |

**Chunking**: Large payloads are split into 255-byte chunks via `exchangeChunks()`.

### Error Handling

Status codes are mapped to user-friendly messages in `STATUS_HINTS`:

```typescript
// Example: Locked device
0x5515: 'Ledger is locked or not ready. Unlock the device, open the Conflux Core app, then retry.'

// Example: Wrong app
0x6a85: 'Wrong Ledger app for the selected chain. Open Conflux Core for Core Space.'
```

## Audit Logging

### Overview

Located in `src/keystore/audit.ts`. Provides append-only audit logging with hash chaining.

```typescript
export function createAppendOnlyAuditLogger(
  opts: AppendOnlyAuditLoggerOptions
): AuditLogger;
```

### Audit Entry Format

```typescript
export interface AuditEntry {
  at: Timestamp;
  provider: string;
  action: string;
  ref?: SecretRef;
  ok: boolean;
  meta?: Record<string, unknown>;
}
```

### Hash Chain

Each entry includes:
- `sequence`: Incrementing counter
- `previousHash`: SHA-256 hash of previous entry
- `entryHash`: SHA-256 hash of current entry (including sequence and previousHash)

This creates an immutable audit trail where tampering with any entry breaks the chain.

## Capability Enforcement

### Overview

Located in `src/keystore/memory/capability.ts`. Wraps a `Signer` to enforce transaction constraints.

```typescript
export function applyCapability(base: Signer, capability: Capability): Signer;
```

### Enforcement Rules

| Constraint | Check |
|------------|-------|
| `chains` | `tx.chainId` must be in allowed list |
| `contracts` | `tx.to` must be in allowed list |
| `selectors` | First 4 bytes of `tx.data` must be in allowed list |
| `maxValuePerTx` | `tx.value` must not exceed limit |
| `notAfter` | Current time must be before expiration |

**Note**: `signMessage` and `signTypedData` are delegated unchanged — capabilities only constrain transactions.

Located in `@cfxdevkit/cdk`. Common keystore error codes:

```typescript

'services/keystore/not-found'           // Secret doesn't exist
'services/keystore/backend-unavailable' // File read/write error

'services/keystore/ledger/address-mismatch' // Expected vs actual address
```

## Integration Patterns

### Creating a Keystore Provider

```typescript
import { createFileKeystore } from '@cfxdevkit/services/keystore-file';

const keystore = createFileKeystore({
  path: './keystore.json',
  unlock: async () => {
    const passphrase = await promptForPassphrase();
    return { passphrase };
  },
  audit: createAppendOnlyAuditLogger({ path: './audit.log' })
});
```

### Using a Signer with Capabilities

```typescript
const signer = await keystore.getSigner(
  { service: 'cfxdevkit', account: 'deployer' },
  {
    chains: [1029],
    contracts: ['0x...'],
    maxValuePerTx: 1000000000000000000n,
    notAfter: Date.now() + 86400000
  }
);

// This will throw if constraints are violated
await signer.signTransaction(tx);
```

### Reading a Mnemonic

```typescript
const mnemonic = await readFileKeystoreMnemonic({
  path: './keystore.json',
  passphrase: '...',
  ref: { service: 'cfxdevkit', account: 'recovery' }
});
```

### Changing Passphrase

```typescript
await changeFilePassphrase({
  path: './keystore.json',
  oldPassphrase: 'old',
  newPassphrase: 'new'
});
```

## Security Considerations

1. **Passphrase Storage**: Never store passphrases in code or config files. Use environment variables or secure input.
2. **File Permissions**: File keystore sets `0o600` permissions, but ensure parent directories are secure.
3. **Memory Cleanup**: Plaintext buffers are zeroed after use in file keystore operations.
4. **Audit Logging**: Enable audit logging in production for compliance and forensics.
5. **Capability Limits**: Always apply capabilities when sharing signer access with third-party code.
6. **Ledger App Updates**: Keep Ledger apps updated to support latest signing features.

## Migration Notes

### From v1 to v2

- Envelope format version is `cfx-v1` (stable)
- Argon2id parameters are now configurable per-keystore
- Capability enforcement is now opt-in via `getSigner()`

### Breaking Changes

- `put()` now requires explicit `kind` field
- `remove()` throws if secret not found (previously silent)
- Ledger Core message signing requires app v2.3.0+
