# CLI & Developer Tools

This module provides the command-line interface (CLI), project scaffolding utilities, and Visual Studio Code extension for the Conflux DevKit. It serves as the primary interaction layer for developers to manage wallets, derive accounts, check network status, scaffold projects, and interact with smart contracts.

The module is split into three distinct packages:
1.  **`packages/cli`**: The core command-line tool (`cfx`).
2.  **`packages/scaffold-cli`**: Project generation and template management.
3.  **`packages/vscode-extension`**: The IDE integration for wallet and contract management.

## Architecture Overview

The CLI uses a minimal, dependency-free argument parser to keep the install footprint low. The VSCode extension relies on a runtime class (`ExtensionRuntime`) to manage state, node lifecycle, and command registration.

## CLI Module (`packages/cli`)

The CLI entry point is `run` in `src/run.ts`. It parses arguments, routes to specific command handlers, and manages output streams.

### Argument Parsing

The module uses a custom parser in `src/args.ts` to avoid heavy dependencies like `commander` or `yargs`.

*   **`parseArgs(argv: readonly string[])`**: Parses `argv` into `command`, `positionals`, and `flags`.
    *   Supports `--flag` (boolean), `--key value`, and `--key=value`.
    *   The first non-flag positional argument is treated as the `command`.
*   **`getString(flags, key)`**: Retrieves a string value from flags.
*   **`getNumber(flags, key)`**: Retrieves and validates a numeric value.
*   **`getBool(flags, key)`**: Checks if a flag is `true` or `'true'`.

### Commands

Commands are implemented as modules in `src/commands/`. Each command exports a `*FromFlags` function that accepts parsed flags and an output stream.

#### `derive`
Derives Conflux accounts from a mnemonic or generates a new one.
*   **`runDerive(opts: RunDeriveOptions)`**: Core logic. Validates mnemonic, derives accounts using `@cfxdevkit/cdk`.
*   **`deriveFromFlags(flags, out)`**: CLI entry point. Handles JSON output (`--json`) and private key exposure (`--show-private-keys`).
*   **Key Options**:
    *   `--mnemonic <phrase>`: Input mnemonic.
    *   `--generate`: Generate a new mnemonic (default strength 128).
    *   `--count <N>`: Number of accounts to derive.
    *   `--type <standard|mining>`: Account type.
    *   `--core-network-id <id>`: Network ID (e.g., 1029 for mainnet).

#### `generate`
Generates a fresh BIP-39 mnemonic.
*   **`runGenerate(opts: RunGenerateOptions)`**: Generates mnemonic and validates it.
*   **`generateFromFlags(flags, out)`**: Outputs to stdout or JSON.
*   **Key Options**:
    *   `--strength <128|256>`: Entropy strength.

#### `status`
Pings Conflux chains to verify connectivity and retrieve head block/epoch numbers.
*   **`runStatus(opts: RunStatusOptions)`**: Iterates over chains (or a specific `--chain`) and pings RPC endpoints.
*   **`statusFromFlags(flags, out)`**: Formats output. Returns exit code `0` if all chains are OK, `1` otherwise.
*   **Key Options**:
    *   `--chain <id|name>`: Target specific chain.
    *   `--rpc <url>`: Override RPC endpoint.
    *   `--timeout-ms <ms>`: Request timeout.

#### `contracts`
*   **`contracts extract`**: Extracts ABI and bytecode from Hardhat artifacts.
*   **`contractsExtractFromFlags(flags, opts)`**: Handles artifact directory input and output path configuration.

### Entry Point

*   **`run(argv, opts)`**: The main function exported from `src/run.ts`.
    *   Handles `--help` and unknown commands.
    *   Wraps command execution in a `try/catch` block to report errors to `stderr`.
    *   Returns exit codes: `0` (success), `1` (error), `2` (usage error).

## Scaffold Module (`packages/scaffold-cli`)

This package handles project initialization using predefined templates.

### Project Scaffolding

*   **`scaffoldProject(projectDir, templateName, options)`**:
    1.  Retrieves the template definition via `getTemplate`.
    2.  Validates the target directory (checks `force` flag if exists).
    3.  Renders files using `renderFile` (replaces `{{variable}}` placeholders).
    4.  Executes `npm install` via `spawnSync` unless `skipInstall` is true.

### Templates

Templates are defined in `src/templates.ts`.
*   **`listTemplates()`**: Returns available template definitions.
*   **`getTemplate(name)`**: Retrieves a specific template by name (e.g., `minimal-dapp`, `wallet-probe`, `project-example`).
*   **`renderFile(content, values)`**: Simple string replacement for template variables.
*   **`getTemplateFiles(template, target)`**: Resolves base files plus target-specific files (e.g., `default` vs `react`).

### Validation

*   **`validateName(name)`**: Ensures project names follow npm conventions (no spaces, valid characters, no leading numbers). Returns `{ valid, error }`.

## VSCode Extension (`packages/vscode-extension`)

The extension integrates Conflux development tools directly into the IDE. It manages a local node runtime, wallet state, and contract interactions.

### Lifecycle

*   **`activate(context)`**: Instantiates `ExtensionRuntime`, registers commands, and triggers `refreshAll`. If `autoStartNode` is enabled and network is `local`, it starts the node.
*   **`deactivate()`**: Awaits `stopNodeForDeactivation` to ensure ports are released before the extension host restarts.
*   **`ExtensionRuntime`**: The central state manager.
    *   Manages `node` (local runtime), `unlockedPassphrase`, `fileProvider`, and `cachedSigner`.
    *   Wraps helper functions (e.g., `workspaceRoot`, `config`, `selectedNetwork`) as class methods for context access.

### Runtime Management

The extension can manage a local Conflux node:
*   **`startNode()` / `stopNode()`**: Controls the local node process.
*   **`restartNode()` / `wipeNode()`**: Resets the node state.
*   **`mineBlocks()` / `startMining()`**: Development tools for block generation.

### Command Registration

Commands are registered via `registerCommands` (imported from helpers). Common commands include:
*   **Wallet**: `addWallet`, `selectWallet`, `unlockKeystore`, `lockKeystore`.
*   **Network**: `selectNetwork`, `pickChainTarget`.
*   **Contracts**: `deployContractCommand`, `importContractCommand`, `abiCallRead`, `abiCallWrite`.
*   **View**: `showAccountsQuickPick`, `showContractsQuickPick`.

### Helper Pattern

Most logic is delegated to helper functions imported from `./helpers/index.js`. The `ExtensionRuntime` class acts as a bridge, binding these helpers to the extension context (`vscode.ExtensionContext`).

*   **State Helpers**: `selectedNetwork`, `selectedSpace`, `selectedBackend`.
*   **Wallet Helpers**: `ensureUnlockedWallet`, `walletSignerFor`, `deriveRunningNodeAccounts`.
*   **Contract Helpers**: `readDeployments`, `writeDeployments`, `pickWorkspaceArtifact`.

## Integration & Dependencies

*   **`@cfxdevkit/cdk`**: The core dependency for cryptographic operations (mnemonic generation, derivation), chain configuration, and RPC clients.
    *   Used heavily in `cli/commands/derive.ts` and `cli/commands/status.ts`.
    *   Used in VSCode for `createClient`, `getChain`, and `listChains`.
*   **Hardware Wallets**: The extension integrates with hardware wallets (e.g., Satochip) via helper calls (e.g., `selectedNetwork` -> `hardware/satochip/index.ts`).
*   **File System**: `scaffold-cli` uses `node:fs` for template writing; VSCode uses `vscode.workspace` for file operations.

## Contributing

### Adding a CLI Command
1.  Create a new file in `packages/cli/src/commands/`.
2.  Export `run*` (logic) and `*FromFlags` (CLI handler).
3.  Import the handler in `packages/cli/src/run.ts`.
4.  Add the case to the `switch (parsed.command)` block in `run`.
5.  Update the `HELP` constant in `run.ts` with usage examples.

### Adding a Scaffold Template
1.  Define the template structure in `packages/scaffold-cli/src/templates/`.
2.  Register the template in `packages/scaffold-cli/src/templates.ts`.
3.  Ensure `renderFile` placeholders (e.g., `{{name}}`) match the `ScaffoldOptions` keys.

### Adding VSCode Commands
1.  Implement the command logic in `packages/vscode-extension/src/helpers/`.
2.  Register the command in `packages/vscode-extension/src/extension-command-registration.ts`.
3.  Ensure the command is added to `package.json` `contributes.commands` (not shown in source but required for VSCode).
