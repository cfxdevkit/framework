# projects — cas

# `@cfxdevkit/cas` Module

## Overview
The `cas` module implements a non-custodial automation system for Conflux eSpace, enabling users to create, monitor, and execute on-chain strategies (limit orders, DCA, TWAP, swaps). It consists of a Node.js/Express backend for state management and execution orchestration, and a Next.js frontend for wallet authentication, strategy creation, and system monitoring. The backend uses SQLite for local persistence during development, while the keeper worker bridges the application layer to smart contracts via the Automation Manager.

## Backend (`apps/backend`)

### Configuration & State
- `resolveCasBackendConfig()` parses environment variables (`CAS_BACKEND_PORT`, `NETWORK`, `SIGNER_PRIVATE_KEY`, etc.) and applies sensible defaults. Network selection maps to correct chain IDs (71 for testnet, 1030 for mainnet) and resolves protocol contract addresses via `@cfxdevkit/protocol`.
- `createCasBackendState()` initializes the application state, returning a `CasBackendState` object that couples `CasBackendConfig` with a `CasSqliteRuntime`.
- The worker is conditionally initialized in `src/index.ts`. If `KEEPER_ENABLED` is true, `SIGNER_PRIVATE_KEY` must be provided; otherwise, the backend runs in read-only/proxy mode.

### Database Layer
`src/db/sqlite.ts` wraps a raw SQLite connection into a structured runtime:
- `CasSqliteRuntime`: Aggregates Drizzle repositories (`DrizzleJobRepository`, `DrizzleExecutionRepository`) with custom stores.
- `SqliteNonceStore`: Manages SIWE nonces with TTL-based expiration. `issue()` generates and stores nonces; `consume()` validates address match, checks expiration, and marks used.
- `SqliteSettingsStore`: Persists safety parameters (`paused`, `maxSwapUsd`, `slippageBps`, `maxRetries`). Reads fall back to defaults (`DEFAULT_SAFETY_SLIPPAGE_BPS = 0`, `DEFAULT_SAFETY_MAX_RETRIES = 3`).
- `SqliteWorkerHeartbeatStore`: Tracks the last seen timestamp and PID of the keeper worker for liveness monitoring.
- Schema initialization (`initializeCasSqliteSchema`) creates `auth_nonces`, `worker_heartbeat`, and relies on Drizzle for `jobs`/`executions`.

### Authentication & Authorization
- `createAuthRouter()` handles the SIWE flow:
  - `GET /auth/nonce`: Validates EVM address format, delegates to `state.db.nonces.issue()`.
  - `POST /auth/verify`: Parses SIWE message, verifies signature via `verifySiweMessage()` against the expected chain ID, consumes the nonce, and issues a JWT session token via `signSessionToken()`.
  - `GET /auth/me`: Validates the bearer token, returns session payload with resolved `isAdmin` status.
- `src/routes/session.ts` provides middleware guards:
  - `requireSession()`: Validates JWT, returns `401` if missing/invalid.
  - `requireAdmin()`: Extends `requireSession()`, checks `session.claims.isAdmin` or fallbacks to `state.config.adminAddresses`. Returns `403` otherwise.

### Route Handlers
| Route | Method | Guard | Purpose |
|-------|--------|-------|---------|
| `/jobs` | GET | `requireSession` | Lists jobs by owner, supports `?status=` & `?type=` filters |
| `/jobs` | POST | `requireSession` | Creates job via `parseCreateJobInput()`, publishes SSE event |
| `/jobs/:id` | GET/DELETE | `requireSession` | Fetches or deletes job (admins can delete owned by others) |
| `/jobs/:id/cancel` | POST | `requireSession` | Transitions to `cancelled`, prevents double-cancel |
| `/jobs/:id/executions` | GET | `requireSession` | Returns execution history for a job |
| `/admin/*` | GET/POST/PATCH | `requireAdmin` | Global pause, safety config patching, admin job inspection |
| `/pools` | GET | None | Returns cached token/pool list (GeckoTerminal mainnet, Swappi fallback testnet) |
| `/system/status` | GET | None | Aggregates DB counts, RPC latency, contract bytecode checks, worker liveness |
| `/sse/jobs` | GET | `readSession` | Opens Server-Sent Events stream for real-time job updates |

### Keeper Worker & Safety Engine
- `createCasWorker()` initializes the `Keeper` from `@cfxdevkit/automation`.
- `DatabaseSafetyGuard` extends `SafetyGuard` and overrides `check()` to dynamically reload safety config via `readSafetyConfig()` before each cycle. This ensures admin patches to `/admin/safety` take effect immediately without restart.
- Price sourcing is configurable via `priceSource` env var (`swappi` or `gecko_terminal`). On testnet, `EspaceSwappiQuoteReader` proxies `getAmountsOut` calls via `eth_call`.
- Heartbeat updates are persisted on each keeper tick via `state.db.heartbeat.update()`.

## Frontend (`apps/frontend`)

### Authentication Context
`src/app/auth-context.tsx` provides `AuthProvider` and `useAuthContext()`:
- Auto-login triggers on wallet connection if no local JWT exists.
- Handles network mismatch checks before SIWE signing.
- Catches specific wallet errors (`UserRejectedRequestError`) and API failures (`CasApiError`).
- Validates session on mount via `client.me()`, clearing local state on mismatch.
- Stores `cas.apiBase`, `cas.token`, and `cas.isAdmin` in `localStorage`.

### Pool & Balance Management
`src/app/pools-context.tsx` aggregates token metadata and on-chain balances:
- `loadPools()` fetches from backend, builds a normalized list including native CFX and WCFX.
- Balances are fetched per-account via `publicClient.readContract()` (ERC20 `balanceOf`) or `getBalance()` for native tokens.
- Debounces redundant balance fetches using `balanceAccount.current` ref.
- Exposes `refresh()` to invalidate pool cache and re-fetch balances.

### System & Safety UI
- `src/app/safety/page.tsx`: Admin-only dashboard. Displays `systemStatus()` and `adminStatus()`. Provides pause/resume controls and safety config display.
- `src/app/status/page.tsx`: Public/Authenticated status board. Shows backend uptime, DB job counts, RPC latency, and contract health (bytecode presence).

### API Proxy & Next.js Configuration
- `src/app/api/[...path]/route.ts`: Next.js API route that proxies all requests to `NEXT_PUBLIC_CAS_API_URL`. Copies `authorization`, `content-type`, and `accept` headers. Redirects `/sse/*` paths via 307 to avoid Next.js body buffering limitations.
- `next.config.ts`: Critical workspace resolution logic. Forces `wagmi`, `viem`, `@tanstack/react-query`, and `@cfxdevkit/ui` to resolve to single instances via Webpack aliases and Turbopack `resolveAlias`. Prevents duplicate React context trees (`WagmiProviderNotFoundError`).

## Key Execution Flows

### SIWE Authentication
1. Client calls `client.nonce(address)` → `GET /auth/nonce`
2. `SqliteNonceStore.issue()` stores nonce with TTL
3. Frontend constructs SIWE message via `createSiweMessage()` → calls wallet `signMessageAsync()`
4. `POST /auth/verify` → `parseSiweMessage()` + `verifySiweMessage()`
5. Chain ID and address match validated. Nonce consumed.
6. JWT issued via `signSessionToken()` with `isAdmin` claim
7. Session persisted, `AuthProvider` updates state

### Job Creation & Execution
1. `POST /jobs` → `parseCreateJobInput()` validates fields, maps to `NewJobInput`
2. `state.db.jobs.create()` persists job, emits `job_update` via SSE
3. Keeper polls `state.db.jobs.list({ status: 'pending' })`
4. `DatabaseSafetyGuard.check()` reads current safety config
5. `KeeperClientImpl` submits transaction to `AutomationManager`
6. On-chain callback updates job status → `state.db.jobs.update()`
7. SSE pushes update to connected frontend clients

### Global Pause & Safety Controls
1. Admin `PATCH /admin/safety` → `parseSafetyPatch()` validates ranges (`slippageBps` 0–10000, `maxRetries` 0–100)
2. `SqliteSettingsStore.set()` persists changes
3. Keeper's `DatabaseSafetyGuard.check()` calls `readSafetyConfig()` before execution cycle
4. If `globalPause === true`, `SafetyGuard.check()` halts execution immediately
5. `GET /admin/status` reflects `state.db.settings.isPaused()`

### Real-time Job Updates (SSE)
1. Client connects to `GET /sse/jobs`
2. `handleJobsSse()` validates session, registers client in `SseHub`
3. Initial snapshot fetched via `state.db.jobs.list()`
4. Heartbeat interval writes `: ping\n\n` every 30s
5. On job mutation, `publishJobUpdate()` iterates `hub.clients`, matches owner, calls `writeEvent()`
6. Fallback poller checks `state.db.jobs.getUpdatedSince()` every 15s for missed updates
7. Client disconnect triggers `hub.clients.delete()` and interval cleanup

## Configuration & Environment Variables

| Variable | Default | Purpose |
|----------|---------|---------|
| `CAS_BACKEND_PORT` | `3011` | Server listen port |
| `NETWORK` | `testnet` | `testnet` or `mainnet` |
| `SIGNER_PRIVATE_KEY` | `undefined` | Keeper wallet private key (required if `KEEPER_ENABLED`) |
| `KEEPER_ENABLED` | `false` | Enable on-chain execution worker |
| `KEEPER_INTERVAL_MS` | `15000` | Keeper polling interval |
| `CAS_AUTH_SECRET` | `cas-local-dev-secret` | JWT signing secret |
| `CAS_SESSION_TTL_MS` | `86400000` | Session token expiry |
| `CAS_NONCE_TTL_MS` | `300000` | SIWE nonce expiry |
| `ADMIN_ADDRESSES` | `undefined` | Comma-separated admin wallet addresses |
| `PRICE_SOURCE` | `swappi` | `swappi` or `gecko_terminal` |
| `NEXT_PUBLIC_CAS_API_URL` | `http://127.0.0.1:3011` | Frontend backend proxy target |

## Integration Points
- **`@cfxdevkit/cas-shared`**: DTOs, API client, type definitions (`CasJobDto`, `CasPoolsResponse`, `CasAdminStatusResponse`)
- **`@cfxdevkit/automation`**: `Keeper`, `SafetyGuard`, `PriceChecker`, `KeeperClientImpl`, Drizzle repositories
- **`@cfxdevkit/services/auth`**: `signSessionToken()`, `verifySessionToken()`, `readBearerToken()`
- **`@cfxdevkit/wallet-connect/siwe`**: `createSiweMessage()`, `parseSiweMessage()`, `verifySiweMessage()`, `generateSiweNonce()`
- **`@cfxdevkit/protocol`**: Contract addresses (`AUTOMATION_MANAGER_ADDRESSES`, `WCFX_ADDRESSES`, `swappiPriceAdapterAddress`)
- **`@cfxdevkit/cdk`**: `createClient()`, `signerFromPrivateKey()`, `EspaceSwappiQuoteReader`, `ZERO_ADDRESS`, chain configs
