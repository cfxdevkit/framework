# Other — examples

# Examples Module

## Overview

The Examples module (`projects/examples`) contains demonstration applications and shared libraries designed to showcase Conflux SDK capabilities. It is structured as a monorepo containing multiple consumer applications and a shared UI library (`showcase-ui`) that standardizes the presentation layer across demos.

This module serves two primary purposes:
1.  **Demonstration:** Providing working examples of hardware wallet integration, backend services, browser interactions, and gateway configurations.
2.  **Reusability:** Offering a shared component library (`@cfxdevkit/example-showcase-ui`) to reduce duplication across example applications.

## Architecture

The module follows a library-first architecture where UI primitives are abstracted into a dedicated package, consumed by various standalone applications.

## Shared UI Library (`showcase-ui`)

Located at `projects/examples/packages/showcase-ui`, this package provides the core UI primitives and utilities used across the example applications. It is built on top of `@cfxdevkit/ui` and `@cfxdevkit/wallet-connect`.

### Package Configuration

*   **Name:** `@cfxdevkit/example-showcase-ui`
*   **Type:** Module (ESM)
*   **Entry Point:** `src/index.ts`
*   **Build Tool:** Vite
*   **Testing:** Vitest

### Dependencies

| Dependency | Version | Purpose |
| :--- | :--- | :--- |
| `@cfxdevkit/ui` | `workspace:*` | Base UI components |
| `@cfxdevkit/wallet-connect` | `workspace:*` | Wallet connection logic and types |
| `react` | `>=18.0.0` | Peer dependency |

### Component API

The library exports components categorized by functionality. All components are re-exported from `src/index.ts`.

#### Wallet & Network Components
These components handle wallet connection states and network switching logic.

*   **`ConnectButton`**: Re-exported from `@cfxdevkit/wallet-connect/ui`.
*   **`WalletButton`**: UI for wallet interaction.
*   **`WalletPickerModal`**: Modal for selecting wallet providers.
*   **`WalletProviderCard`**: Displays provider status and details.
    *   *Props:* `WalletProviderCardProps`, `WalletProviderCardStatus`
*   **`WalletStatusChip`**: Compact status indicator.
    *   *Props:* `WalletStatusChipProps`
*   **`NetworkSwitchNotice`**: Notification for network changes.
    *   *Props:* `NetworkSwitchNoticeProps`

#### Data & Form Components
Components for displaying metrics, tokens, and handling user input.

*   **`AssetConversionPanel`**: Handles asset conversion UI.
    *   *Props:* `AssetConversionPanelProps`
*   **`TokenAmountField`**: Input field for token amounts.
    *   *Props:* `TokenAmountFieldProps`
*   **`TokenPairSelector`**: Selector for token pairs.
    *   *Props:* `TokenPairSelectorProps`
*   **`TokenSelect`**: Dropdown for token selection.
    *   *Props:* `TokenSelectProps`, `TokenSelectOption`
*   **`Field`**: Generic form field wrapper.
    *   *Props:* `FieldProps`
*   **`Metric`**: Displays numerical metrics.
    *   *Props:* `MetricProps`
*   **`StatusGrid`**: Grid layout for status items.
    *   *Props:* `StatusGridProps`
*   **`SegmentedControl`**: Segmented button control.
    *   *Props:* `SegmentedControlProps`, `SegmentedControlOption`

#### Layout & Shell Components
Structural components for the application shell and navigation.

*   **`Shell`**: Main application shell container.
    *   *Props:* `ShellProps`
    *   *Hooks:* `useActivePanelState`
*   **`Sidebar`**: Navigation sidebar.
    *   *Props:* `SidebarProps`, `SidebarGroup`, `SidebarItem`
*   **`PanelSidebar`**: Sidebar variant for panels.
*   **`ShowcaseNav`**: Navigation component specific to showcase apps.
*   **`ShowcaseOpsPanel`**: Operations panel for showcase apps.
*   **`DemoCard`**: Card component for demo listings.
    *   *Props:* `DemoCardProps`
*   **`CodeSnippet`**: Displays code blocks.
    *   *Props:* `CodeSnippetProps`
*   **`LogBox`**: Displays log output.
*   **`CopyButton`**: Utility button for copying text.
*   **`StatusBadge`**: Status indicator badge.
    *   *Props:* `StatusBadgeProps`, `BadgeStatus`
*   **`ConnectWall`**: Component for connection walls/obstacles.
*   **`SharedDevNodePill`**: Indicator for development nodes.

#### Notification Components
*   **`Notice`**: General notice component.
    *   *Props:* `NoticeProps`, `NoticeTone`

### Wallet State Utilities

The library re-exports core wallet state management utilities from `@cfxdevkit/wallet-connect`.

*   **`useCoreWallet`**: Hook for accessing core wallet state.
*   **`deriveCoreState`**: Function to derive core state.
*   **`getFluentCoreProvider`**: Function to retrieve the fluent core provider.
*   **`coreChainLabel`**: Function to format chain labels.
*   **`CORE_CHAIN_CONFIGS`**: Configuration object for core chains.
*   **Types:** `CoreChainConfig`, `CoreWalletStatus`

### Logging Utilities

Located in `src/lib/log`, these utilities manage application logging.

*   **`makeEntry`**: Creates a log entry.
*   **`useLogList`**: Hook for managing a list of logs.
*   **`copy`**: Utility function for copying content.
*   **`errMsg`**: Utility for formatting error messages.
*   **Types:** `LogEntry`, `LogLevel`

### Configuration

#### TypeScript (`tsconfig.json`)
Extends `@cfxdevkit/tsconfig/lib.json`. Configured for React JSX and ESNext modules.
*   **Root Dir:** `../../../../`
*   **Out Dir:** `dist`
*   **Paths:** Maps `@cfxdevkit/ui` and `@cfxdevkit/ui-core` to local repo paths.

#### Vite (`vite.config.ts`)
Configured for library builds.
*   **Format:** ES Module (`es`)
*   **Target:** `es2023`
*   **Plugins:** `vite-plugin-dts` for TypeScript declaration generation.
*   **External:** All non-relative imports are treated as external dependencies.

#### Testing (`vitest.config.ts`)
*   **Environment:** `node`
*   **Include:** `src/**/*.test.ts`

## Consumer Applications

The `projects/examples/apps` directory contains standalone applications that utilize the `showcase-ui` library. While specific implementation details vary per app, they share a common configuration structure.

| Application | Path | Description |
| :--- | :--- | :--- |
| **hardware-wallet-showcase** | `apps/hardware-wallet-showcase` | Demonstrates hardware wallet integration. |
| **showcase-backend** | `apps/showcase-backend` | Backend service example. |
| **showcase-browser** | `apps/showcase-browser` | Browser-based interaction demo. |
| **showcase-gateway** | `apps/showcase-gateway` | Gateway integration example. |
| **showcase-stack** | `apps/showcase-stack` | Full stack demonstration. |
| **showcase** | `apps/showcase` | General showcase application. |

### Common Structure
Each application typically includes:
*   `index.html`: Entry HTML file.
*   `main.tsx`: React entry point.
*   `vite.config.ts`: Build configuration.
*   `tsconfig.json`: TypeScript configuration.
*   `package.json`: Dependencies and scripts.
*   `moon.yml`: Moon workspace configuration.
*   `src/styles.css`: Global styles (often split into parts).
*   `src/*.test.ts`: Unit tests.

## Development Workflow

### Building the UI Library
To typecheck and build the shared UI library:

```bash
# Typecheck
npm run typecheck

# Build (generates dist/)
npm run build

# Lint
npm run lint

# Test
npm run test
```

### Running Consumer Applications
Each application in `apps/` follows standard Vite/React development workflows. Refer to the specific `package.json` within each app directory for start scripts (typically `dev` or `start`).

### Adding New Components
1.  Create the component file in `src/components/`.
2.  Export the component and its props type in `src/index.ts`.
3.  Ensure the component is compatible with `@cfxdevkit/ui` and `@cfxdevkit/wallet-connect` patterns.
4.  Run `npm run lint` and `npm run test` to verify compliance.
