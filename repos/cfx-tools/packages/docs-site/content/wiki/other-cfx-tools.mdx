# cfx-tools

**Tier 1 Developer Experience Tools**

The `cfx-tools` module provides the primary developer-facing interfaces for the Conflux DevKit. It is a monorepo containing four distinct packages that handle project scaffolding, command-line operations, AI agent integration, and IDE support.

## Architecture

The module is structured as a pnpm workspace. All packages share a common build toolchain (`vite`, `vitest`, `biome`, `moon`) and depend on core DevKit libraries (`@cfxdevkit/cdk`, `@cfxdevkit/client`, etc.).

## Package: @cfxdevkit/cli

A command-line interface for network status checks, HD key derivation, and mnemonic generation.

### Entry Point
*   **Binary:** `cfx` (defined in `package.json` bin)
*   **Source:** `src/bin.ts` calls `run(argv)` from `src/run.js`.

### Commands
1.  **`status`**: Pings known chains and prints head block/epoch and latency.
    *   **Function:** `runStatus`
    *   **Flags:** `--chain <id|name>`
2.  **`derive`**: Derives dual-space accounts (EVM `0x...` + Core `cfx:...`).
    *   **Function:** `runDerive`
    *   **Flags:** `--mnemonic`, `--generate`, `--count`, `--start`, `--type` (standard|mining), `--core-network-id`, `--json`, `--show-private-keys`.
    *   **Derivation Paths:**
        *   EVM: `m/44'/60'/0'/0/0` (standard) or `m/44'/60'/1'/0/0` (mining).
        *   Core: `m/44'/503'/0'/0/0`.
3.  **`generate`**: Emits a fresh BIP-39 mnemonic.
    *   **Function:** `runGenerate`
    *   **Flags:** `--strength` (128|256).

### Argument Parsing
Arguments are parsed via `parseArgs` in `src/args.js`. Helper functions validate types:
*   `getString`: Returns string value or `undefined`.
*   `getNumber`: Parses numeric strings, rejects `NaN`/`Infinity`.
*   `getBool`: Returns `true` for boolean `true` or string `"true"`.

### Output
*   **Default:** Human-readable text.
*   **JSON:** Pass `--json` for machine-readable output (e.g., `DeriveReport`).

## Package: @cfxdevkit/mcp-server

Exposes DevKit tools to AI agents via the Model Context Protocol (MCP).

### Server Setup
*   **Function:** `createMcpServer`
*   **Binary:** `cfxdevkit-mcp`
*   **Entry:** `src/index.ts` exports `__packageName`, `createMcpServer`, and tool registry.

### Tool Registry
*   **Definition:** `MCP_TOOL_DEFINITIONS` (37 tools).
*   **Access:** `listMcpTools(category)`, `getMcpTool(name)`.
*   **Safety:** Write and admin tools are marked `requiresConfirmation: true`.
*   **Categories:** Includes `blockchain-write`, `compiler`, `node`, etc.

### Operation Ledger
Tracks tool execution lifecycle for auditing and debugging.
*   **Class:** `OperationLedger`
*   **Methods:**
    *   `startOperation(tool, params)`: Returns operation ID.
    *   `addOperationStep(id, message)`: Logs intermediate steps.
    *   `finishOperation(id, status)`: Finalizes with duration and status.

## Package: @cfxdevkit/create

Project scaffolder invoked via `npm create @cfxdevkit`.

### Scaffolding Flow
1.  **Parse:** `parseArgs` handles positional arguments (project name) and flags (`--template`, `--force`).
2.  **Validate:** `validateName` checks project name rules (no spaces, no consecutive hyphens, no leading numbers).
3.  **Template:** `getTemplate(name)` retrieves definition. `getTemplateFiles(template, target)` resolves files (supports `default` and `devcontainer` targets).
4.  **Render:** `renderFile(content, context)` replaces `{{placeholder}}` tokens.
5.  **Execute:** `scaffoldProject(path, template, options)` creates directory, writes files, and runs `npm install` (unless `skipInstall` is true).

### Templates
*   **Available:** `minimal-dapp`, `wallet-probe`, `project-example`.
*   **Aliases:** `basic`, `react`, `solidity` map to active templates.
*   **Export:** `listTemplates()` returns all available definitions.

## Package: cfxdevkit-vscode-extension

VS Code integration for node lifecycle, keystore management, and contract deployment.

### Activation
*   **Event:** `onStartupFinished`
*   **Kind:** `workspace`

### Commands
*   **Node:** `nodeStart`, `nodeStop`, `nodeRestart`, `nodeWipe`, `mineBlocks`, `mining.start`, `mining.stop`.
*   **Wallet:** `initializeSetup`, `addWallet`, `selectWallet`, `removeWallet`, `unlockKeystore`, `lockKeystore`, `rotateKeystorePassphrase`.
*   **Contracts:** `deployContract`, `importContract`, `listContracts`, `abiCallRead`, `abiCallWrite`.
*   **UI:** `selectNetwork`, `viewAccounts`, `copyAddress`.

### Configuration
Settings are stored in `package.json` under `contributes.configuration`:
*   `cfxdevkit.autoStartNode`: Boolean (default `false`).
*   `cfxdevkit.keystorePath`: Workspace-relative path (default `.cfxdevkit/keystore.json`).
*   `cfxdevkit.nodeDataDir`: Workspace-relative path (default `.cfxdevkit/devnode`).
*   `cfxdevkit.nodeAccounts`: Number of pre-funded accounts (default `10`).

### Views
*   **Sidebar:** `cfxdevkit-sidebar` with `cfxdevkit.mainView`.
*   **Context Menus:** Inline actions for wallets (lock/unlock/remove), nodes (start/stop/wipe), and contracts (copy address).

## Development Workflow

### Workspace Validation
Run validation scripts from the root `cfx-tools` directory:
```bash
pnpm validate:workspace
pnpm validate:deps
```

### Build & Test
Each package uses `vite` for bundling and `vitest` for testing.

| Package | Build Command | Test Command | Typecheck |
| :--- | :--- | :--- | :--- |
| `cli` | `pnpm build` | `pnpm test` | `pnpm typecheck` |
| `mcp-server` | `pnpm build` | `pnpm test` | `pnpm typecheck` |
| `scaffold-cli` | `pnpm build` | `pnpm test` | `pnpm typecheck` |
| `vscode-extension` | `pnpm build` | `pnpm smoke:workflows` | `pnpm typecheck` |

### Moon Tasks
The `moon` build system manages dependencies and caching.
*   **CLI:** `moon run cli:build`
*   **Scaffold:** `moon run scaffold-cli:smoketest`
*   **Extension:** `moon run vscode-extension:build`

### Linting
All packages use `biome` for linting and formatting.
```bash
pnpm lint
```

## Integration Points

*   **CDK:** `@cfxdevkit/cli` and `@cfxdevkit/mcp-server` rely on `@cfxdevkit/cdk` for core blockchain logic.
*   **Client:** `@cfxdevkit/client` is used by MCP and VS Code for RPC interactions.
*   **Compiler:** `@cfxdevkit/compiler` is integrated into MCP and VS Code for Solidity compilation.
*   **Devnode:** `@cfxdevkit/devnode-server` is managed by VS Code and MCP for local node lifecycle.
*   **Wallet:** `@cfxdevkit/wallet` handles keystore encryption/decryption across CLI, MCP, and VS Code.
