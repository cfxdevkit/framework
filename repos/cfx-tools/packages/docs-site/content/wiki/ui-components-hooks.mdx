# UI Components & Hooks

# UI Components & Hooks Module

This module provides the React integration layer for Conflux dApps. It is split into four primary packages that handle blockchain interaction, DeFi primitives, wallet connectivity, and design theming.

- **`@cfxdevkit/react`**: Core hooks for accounts, balances, contracts, and transactions.
- **`@cfxdevkit/defi-react`**: DeFi-specific hooks for swaps, portfolios, and liquidity pools.
- **`@cfxdevkit/wallet-connect`**: Wallet connection management (Fluent Core, Wagmi eSpace) and SIWE authentication.
- **`@cfxdevkit/theme`**: Design system tokens and CSS variables.

## Architecture

The module relies on `@cfxdevkit/cdk` for low-level RPC and type definitions. React hooks consume a `Client` and `Signer` provided by the `CfxProvider` context.

## Core React Hooks (`@cfxdevkit/react`)

These hooks manage state and side effects for blockchain data. They require the `CfxProvider` to be mounted in the app tree.

### Account & Balance

```typescript
import { useAccount, useNativeBalance, useTokenBalance } from '@cfxdevkit/react';

// Check connection status
const { address, isConnected } = useAccount();

// Fetch native CFX balance
const { data: balance, isLoading } = useNativeBalance({ address });

// Fetch ERC-20 balance
const { data: tokenBalance } = useTokenBalance({
  token: '0xTokenAddress',
  owner: address,
});
```

### Contract Interaction

Read operations use `useQuery` internally, caching results automatically. Write operations use `useMutation`.

```typescript
import { useReadContract, useWriteContract } from '@cfxdevkit/react';

// Read
const { data } = useReadContract({
  address: '0xContract',
  abi: MyABI,
  functionName: 'balance',
  args: [userAddress],
});

// Write
const { writeAsync, isPending } = useWriteContract();
await writeAsync({
  address: '0xContract',
  abi: MyABI,
  functionName: 'transfer',
  args: [recipient, amount],
});
```

### Transaction Lifecycle

For raw transactions, use `useSendTransaction` combined with `useWaitForTransaction` to monitor receipt.

```typescript
import { useSendTransaction, useWaitForTransaction } from '@cfxdevkit/react';

const { sendAsync } = useSendTransaction();
const { data: receipt } = useWaitForTransaction({ hash });

const hash = await sendAsync({ to: '0x...', value: 100n });
```

### Event Watching

`useWatchEvent` polls `eth_getLogs` on an interval to detect contract events.

```typescript
import { useWatchEvent } from '@cfxdevkit/react';

useWatchEvent({
  address: '0xContract',
  abi: EventABI,
  eventName: 'Transfer',
  onLogs: (logs) => console.log('New event', logs),
  pollIntervalMs: 4000,
});
```

## DeFi Integration (`@cfxdevkit/defi-react`)

This package builds on core hooks to provide higher-level financial primitives using the `SwapService` adapter pattern.

### Portfolio Management

`usePortfolio` aggregates native and token balances for a user.

```typescript
import { usePortfolio } from '@cfxdevkit/defi-react';

const { rows, isLoading } = usePortfolio({
  address: userAddress,
  tokens: [/* TokenInfo[] */],
});
// rows contains { token, balance, formatted } sorted by value
```

### Token Swapping

`useSwap` handles the full swap lifecycle: fetching a quote, checking/approving allowances, and executing the transaction.

```typescript
import { useSwap, createSwappiAdapter } from '@cfxdevkit/defi-react';

const adapter = createSwappiAdapter({ chainId: 71, client });
const {
  quote,
  swapAsync,
  needsApproval,
  approveAsync,
  isSwapping,
} = useSwap({
  adapter,
  tokenIn: '0xTokenA',
  tokenOut: '0xTokenB',
  amountIn: 1000000000000000000n, // 1 token (18 decimals)
});

// Execution flow
if (needsApproval) {
  await approveAsync();
}
await swapAsync();
```

**Swap Flow:**
1.  **Quote:** Calls `adapter.getQuote` via `SwapService`.
2.  **Allowance:** Calls `useReadContract` to check `allowance(token, spender)`.
3.  **Approval:** Calls `useWriteContract` to `approve(spender, MAX_UINT256)`.
4.  **Swap:** Calls `adapter.buildCalldata` then `useSendTransaction`.

### Liquidity Pools

```typescript
import { usePools, usePoolTokens } from '@cfxdevkit/defi-react';

// Single pair
const { pool } = usePoolTokens({ service, pairAddress: '0xPair' });

// Multiple pairs
const { pools } = usePools({ service, pairAddresses: ['0xPair1', '0xPair2'] });
```

## Wallet Connection (`@cfxdevkit/wallet-connect`)

Handles connection state for both Conflux Core (Fluent) and eSpace (Wagmi).

### Core Wallet (Fluent)

`useCoreWallet` manages the Fluent provider state specifically for Conflux Core space.

```typescript
import { useCoreWallet } from '@cfxdevkit/wallet-connect';

const {
  status, // 'detecting' | 'active' | 'not-installed'
  address,
  connect,
  switchChain,
} = useCoreWallet();
```

### eSpace Connectors (Wagmi)

`useEspaceConnectors` wraps Wagmi connectors for eSpace compatibility.

```typescript
import { useEspaceConnectors } from '@cfxdevkit/wallet-connect';

const { connectors, connect } = useEspaceConnectors();
connectors.forEach(c => console.log(c.name));
```

### Chain Switching

Use `switchEspaceChain` to ensure the wallet is on the correct network.

```typescript
import { switchEspaceChain } from '@cfxdevkit/wallet-connect';
import { espaceMainnet } from '@cfxdevkit/wallet-connect/config';

await switchEspaceChain(provider, espaceMainnet);
```

### SIWE Authentication

Utilities for Sign-In with Ethereum.

```typescript
import { createSiweMessage, verifySiweMessage, generateSiweNonce } from '@cfxdevkit/wallet-connect';

// 1. Generate Nonce
const nonce = generateSiweNonce({ length: 17 });

// 2. Create Message
const message = createSiweMessage({
  domain: 'dapp.example.com',
  address: userAddress,
  uri: 'https://dapp.example.com',
  chainId: 1030,
  nonce,
});

// 3. Verify Signature (on backend or client)
const result = await verifySiweMessage({
  message,
  signature: signedSig,
  expectedAddress: userAddress,
});
```

## Keystore Management (`@cfxdevkit/keystore`)

Provides hooks for managing local wallet keystore state (locked/unlocked, wallet lists).

```typescript
import { KeystoreProvider, useKeystoreWallets, useIsKeystoreActive } from '@cfxdevkit/keystore';

// Wrap app
<KeystoreProvider service={myKeystoreService}>
  <App />
</KeystoreProvider>

// Consume
const wallets = useKeystoreWallets();
const isActive = useIsKeystoreActive();
```

## Design System (`@cfxdevkit/theme`)

Exports CSS variable references and JS constants for styling components consistently.

```typescript
import { colors, spacing, typography } from '@cfxdevkit/theme';

const buttonStyle = {
  background: colors.brand.primary,
  padding: spacing[4],
  fontSize: typography.sizes.base,
};
```

## Error Handling

The module defines specific error types for better debugging:

- **`ContractError`**: Wraps contract call failures (includes `cause`).
- **`DexError`**: Wraps DEX adapter failures (e.g., `NO_ROUTE`, `SAME_TOKEN`).
- **`DexError`**: Used within `SwapService` and `useSwap` to signal protocol-specific issues.

## Dependencies

- **`viem`**: ABI encoding/decoding and chain definitions.
- **`@tanstack/react-query`**: Data fetching and caching.
- **`@cfxdevkit/cdk`**: Core types and RPC clients.
- **`wagmi`**: Wallet connector management (eSpace).
