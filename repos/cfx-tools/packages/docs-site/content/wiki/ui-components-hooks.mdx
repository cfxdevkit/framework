# UI Components & Hooks Module

## Overview

This module provides the React layer for Conflux eSpace applications, including wallet connection, contract interaction, DeFi primitives, and design system tokens. It is organized into four packages:

| Package | Purpose |
|---------|---------|
| `@cfxdevkit/react` | Core hooks for accounts, balances, contracts, transactions, and events |
| `@cfxdevkit/defi-react` | DeFi-specific hooks for swaps, pools, portfolios, and token metadata |
| `@cfxdevkit/wallet-connect` | Wallet detection, connection, chain switching, and SIWE authentication |
| `@cfxdevkit/theme` | Design tokens and CSS custom properties |

All packages depend on `@cfxdevkit/cdk` for type definitions and client abstractions.

## Core React Hooks (`@cfxdevkit/react`)

### Account & Signer

**`useAccount()`**

Returns the current signer's address and connection state.

```typescript
import { useAccount } from '@cfxdevkit/react/account';

const { address, isConnected } = useAccount();
```

- `address`: `Address | null` — The signer's address, or `null` if no signer is connected
- `isConnected`: `boolean` — Whether a signer is available

**Implementation Notes:**
- Reads from the `SignerContext` provided by `<CfxProvider>`
- Returns `null` address and `isConnected: false` in read-only sessions

### Balance Queries

**`useNativeBalance(input)`**

Fetches the native CFX balance for an address.

```typescript
import { useNativeBalance } from '@cfxdevkit/react/balance';

const { data, error, isLoading, refetch } = useNativeBalance({
  address: '0x...',
  enabled: true,
  staleTimeMs: 15_000,
});
```

| Input | Description |
|-------|-------------|
| `address` | Target address (required for query) |
| `enabled` | Whether to run the query (default: `true`) |
| `staleTimeMs` | Cache staleness threshold |

**`useTokenBalance(input)`**

Fetches ERC-20 balance for a token.

```typescript
import { useTokenBalance } from '@cfxdevkit/react/balance';

const { data } = useTokenBalance({
  token: '0xTOKEN_ADDRESS',
  owner: '0xOWNER_ADDRESS',
});
```

**`useTokenMetadata(input)`**

Fetches token name, symbol, and decimals. Cached indefinitely since metadata is immutable.

```typescript
import { useTokenMetadata } from '@cfxdevkit/react/balance';

const { data } = useTokenMetadata({ token: '0xTOKEN_ADDRESS' });
// data: { name: string, symbol: string, decimals: number }
```

### Contract Interaction

**`useReadContract(input)`**

Reads from a contract via `eth_call`.

```typescript
import { useReadContract } from '@cfxdevkit/react/contract';

const { data, error, isLoading, refetch } = useReadContract({
  address: '0xCONTRACT',
  abi: MY_ABI,
  functionName: 'balanceOf',
  args: ['0xOWNER'],
  enabled: true,
  staleTimeMs: 15_000,
});
```

**`useReadContracts(input)`**

Batch read multiple contract calls in parallel. Each call reports its own success/failure status.

```typescript
const { data } = useReadContracts({
  calls: [
    { address: '0xA', abi: ABI_A, functionName: 'balanceOf', args: ['0xOWNER'] },
    { address: '0xB', abi: ABI_B, functionName: 'symbol' },
  ],
});
// data: Array<{ status: 'success' | 'failure', result: unknown }>
```

**`useWriteContract()`**

Submits signed transactions. Returns a `writeAsync` function that handles gas estimation, nonce, and signing.

```typescript
import { useWriteContract } from '@cfxdevkit/react/contract';

const { writeAsync, isPending, error, reset } = useWriteContract();

const hash = await writeAsync({
  address: '0xCONTRACT',
  abi: MY_ABI,
  functionName: 'transfer',
  args: ['0xRECIPIENT', 100n],
  value: 0n,
});
```

**`useSimulateContract(input)`**

Simulates a contract call from the current signer's address. Useful for previewing results before submitting.

```typescript
const { data } = useSimulateContract({
  address: '0xCONTRACT',
  abi: MY_ABI,
  functionName: 'transfer',
  args: ['0xRECIPIENT', 100n],
});
// data: { result: T, request: WriteInput }
```

### Transaction Lifecycle

**`useSendTransaction()`**

Submits a raw transaction (no contract ABI required).

```typescript
import { useSendTransaction } from '@cfxdevkit/react/tx';

const { sendAsync, isPending, error } = useSendTransaction();

const { hash } = await sendAsync({
  to: '0xRECIPIENT',
  value: 100n,
  data: '0x...',
});
```

**`useWaitForTransaction(input)`**

Polls for a transaction receipt until it's available.

```typescript
import { useWaitForTransaction } from '@cfxdevkit/react/tx';

const { data, isLoading, error } = useWaitForTransaction({
  hash: '0xTX_HASH',
  pollIntervalMs: 2_000,
});
// data: TxReceipt | undefined
```

### Event Watching

**`useWatchEvent(input)`**

Polls `eth_getLogs` on an interval and decodes event logs.

```typescript
import { useWatchEvent } from '@cfxdevkit/react/events';

useWatchEvent({
  address: '0xCONTRACT',
  abi: MY_ABI,
  eventName: 'Transfer',
  args: { from: '0xSENDER' },
  pollIntervalMs: 4_000,
  enabled: true,
  onLogs: (logs) => {
    logs.forEach((log) => {
      console.log(log.eventName, log.args);
    });
  },
  onError: (err) => console.error(err),
});
```

**Important:**
- `onLogs` and `onError` are not in the dependency array — wrap them in `useCallback` to avoid restart loops
- The hook tracks `fromBlock` across polls to only fetch new logs
- Automatically cleans up on unmount

### Keystore Hooks

The keystore system manages dual-chain (eSpace + Core) wallet identities. All hooks must be used inside `<KeystoreProvider>`.

**`useKeystoreLifecycle()`**

Returns the keystore phase and lifecycle actions.

```typescript
import { useKeystoreLifecycle } from '@cfxdevkit/react/keystore';

const {
  phase,          // 'blank' | 'locked' | 'unlocked' | 'active-wallet'
  isLocked,
  isBusy,
  setup,
  unlock,
  lock,
  refresh,
} = useKeystoreLifecycle();
```

**`useKeystoreWallets()`**

Returns the list of wallets and mutation actions.

```typescript
import { useKeystoreWallets } from '@cfxdevkit/react/keystore';

const {
  wallets,
  activeWalletId,
  activateWallet,
  addWallet,
  deleteWallet,
  renameWallet,
} = useKeystoreWallets();
```

**`useKeystoreAccounts()`**

Returns accounts for the active wallet and account switching.

```typescript
import { useKeystoreAccounts } from '@cfxdevkit/react/keystore';

const {
  accounts,
  activeAccountIndex,
  activateAccount,
} = useKeystoreAccounts();
```

**`useKeystoreIdentity()`**

Returns the normalized dual-chain identity (both eSpace and Core addresses).

```typescript
import { useKeystoreIdentity } from '@cfxdevkit/react/keystore';

const { identity, hasIdentity } = useKeystoreIdentity();
// identity: { walletId, walletName, accountIndex, espaceAddress, coreAddress, ... }
```

**Phase Predicates:**

```typescript
import {
  useIsKeystoreBlank,
  useIsKeystoredLocked,
  useIsKeystoreReady,
  useIsKeystoreActive,
} from '@cfxdevkit/react/keystore';

const isLocked = useIsKeystoredLocked();
const isReady = useIsKeystoreReady();
```

## DeFi Hooks (`@cfxdevkit/defi-react`)

### Portfolio

**`usePortfolio(input)`**

Fetches balances for native CFX and a list of ERC-20 tokens. Returns rows sorted by balance descending.

```typescript
import { usePortfolio } from '@cfxdevkit/defi-react/balance';

const { rows, isLoading, error } = usePortfolio({
  address: '0xUSER_ADDRESS',
  tokens: [
    { address: '0xTOKEN1', symbol: 'USDT', name: 'Tether', decimals: 6, chainId: 1030 },
    { address: '0xTOKEN2', symbol: 'WCFX', name: 'Wrapped CFX', decimals: 18, chainId: 1030 },
  ],
  refreshMs: 15_000,
});

// rows: Array<{ token: TokenInfo, balance: Wei, formatted: string }>
```

**Implementation Details:**
- Merges wrapped-native (WCFX) balances into the native CFX row
- Uses parallel `useQueries` for ERC-20 balance fetching
- Formats balances using `formatBigInt` with decimal handling

### Pool & Price Queries

**`usePoolTokens(input)`**

Fetches token pair info and reserves from a Swappi V2 pair contract.

```typescript
import { usePoolTokens } from '@cfxdevkit/defi-react/pool';

const { pool, isLoading } = usePoolTokens({
  service: swapService,
  pairAddress: '0xPAIR_ADDRESS',
  refreshMs: 15_000,
});
// pool: { token0, token1, reserve0, reserve1 }
```

**`usePools(input)`**

Bulk fetch pool data for multiple pair addresses.

```typescript
const { pools, isLoading, errors } = usePools({
  service: swapService,
  pairAddresses: ['0xPAIR1', '0xPAIR2'],
});
// pools: Map<Address, PoolData>
```

**`useTokenPrice(input)`**

Fetches the price of a token denominated in a quote token via Swappi V2.

```typescript
const { price } = useTokenPrice({
  service: swapService,
  tokenAddress: '0xTOKEN',
  quoteToken: '0xUSDT',
  refreshMs: 30_000,
});
// price: bigint (raw output for 1e18 units of tokenAddress)
```

### Swap

**`useSwap(input)`**

Fetches a swap quote and provides approval + swap execution functions.

```typescript

import { createSwappiAdapter } from '@cfxdevkit/defi-react/swap';

const adapter = createSwappiAdapter({ chainId: 71, client });

const {
  quote,
  isQuoting,
  quoteError,
  approveAsync,
  needsApproval,
  isApproving,
  approvalError,
  swapAsync,
  isSwapping,
  swapError,
  reset,
} = useSwap({
  adapter,
  tokenIn: '0xTOKEN_IN',
  tokenOut: '0xTOKEN_OUT',
  amountIn: 100n,
  slippageBps: 50,
  deadlineMs: 60_000,
});
```

**`SwapService`**

The Swappi V2 DEX adapter. Implements `DexAdapter` interface.

```typescript
import { createSwapService } from '@cfxdevkit/defi-react/service';

const service = createSwapService({ chainId: 1030, client });

// Get quote
const quote = await service.getQuote({
  tokenIn: '0xTOKEN_IN',
  tokenOut: '0xTOKEN_OUT',
  amountIn: 100n,
  slippageBps: 50,
});

// Build calldata
const calldata = await service.buildCalldata(quote, { recipient: '0xRECIPIENT' });
// calldata: { to: Address, data: Hex, value?: Wei }

// Pool helpers
const pool = await service.getPoolTokens('0xPAIR_ADDRESS');
const price = await service.getTokenPrice('0xTOKEN', '0xQUOTE');
const pair = await service.getPair('0xTOKEN_A', '0xTOKEN_B');
```

function TransferButton({ to, amount }) {

        abi: ERC20_ABI,

      console.error('Transfer failed:', err);

    <button onClick={handleClick} disabled={isPending}>
      {isPending ? 'Sending...' : 'Transfer'}
    </button>
  );

## Best Practices

1. **Always wrap callbacks**: When using `useWatchEvent`, wrap `onLogs` and `onError` in `useCallback` to prevent poll loop restarts.

2. **Handle loading states**: All query hooks return `isLoading` — always show appropriate loading UI.

3. **Check connection before writes**: Use `useAccount()` to verify `isConnected` before calling `writeAsync`.

4. **Use `enabled` flags**: Disable queries when required data (e.g., address) is not available to avoid unnecessary RPC calls.

5. **Set appropriate `staleTimeMs`**: Balance freshness vs. RPC cost. Use 15-30 seconds for balances, `Infinity` for immutable metadata.

6. **Catch DexError specifically**: Swappi errors (no route, same token) are `DexError` instances with a `code` property for programmatic handling.

7. **Optimistic updates**: For keystore mutations (`addWallet`, `deleteWallet`, `renameWallet`), the hooks support optimistic UI updates with automatic rollback on error.

8. **Chain switching**: Always check `showSwitch` in `deriveESpaceState` or `deriveCoreState` before rendering chain switch UI.

9. **SIWE validation**: Always verify `expirationTime` and `notBefore` fields when validating SIWE messages.

10. **Gas estimation**: `useWriteContract` handles gas estimation automatically. For native CFX transfers, use `useSendTransaction` instead.
