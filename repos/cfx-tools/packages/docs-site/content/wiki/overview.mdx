# Conflux DevKit — Framework Overview

Welcome to the **`@cfxdevkit/framework`** monorepo. This is the canonical starting point for understanding how the codebase is organized, how the pieces relate, and where to go next.

---

## What This Is

Conflux DevKit is a pnpm monorepo for building applications on the [Conflux Network](https://confluxnetwork.org/). It covers the full developer journey: on-chain interaction, key management, smart contract tooling, local devnet, React UI primitives, a VS Code extension, a scaffold CLI, and an LLM-assisted automation pipeline for code quality and documentation.

Everything is TypeScript-first and governed by a **strict 5-tier dependency hierarchy** enforced at commit time by [`@cfxdevkit/arch-check`](architecture-check-and-repo-validation.md).

## The Foundation

Everything builds on [Core Protocol & Chain](core-protocol-and-chain.md) (`@cfxdevkit/cdk` and `@cfxdevkit/protocol`). These packages provide chain configuration, RPC clients, address encoding, unit math, wallet primitives, and contract ABIs. If a package does anything on-chain, it imports from here.

[DevNode & Runtime](devnode-and-runtime.md) wraps the native Conflux binary to spin up a local dual-space chain (Core + eSpace) with pre-funded accounts and auto-mining — wired directly to the chain configs in `@cfxdevkit/cdk` so everything works with zero extra setup.

---

## Key Management

The cryptographic stack is split across two layers. [Keystore & Wallet Services](keystore-and-wallet-services.md) (`@cfxdevkit/services`) handles crypto primitives, pluggable keystore backends, and session authentication. Private key material never escapes the keystore boundary — callers receive `Signer` objects, not raw keys.

[Wallet Abstraction & Hardware](wallet-abstraction-and-hardware.md) (`@cfxdevkit/wallet`) sits on top and adds hardware device signing (Ledger), session key delegation, and transaction batching. The VS Code extension's keystore panel drives this entire stack through 53 direct call edges into the wallet layer.

---

## Contract & Application Layer

[Solidity Compiler & Contracts](solidity-compiler-and-contracts.md) (`@cfxdevkit/contracts`) provides a unified API for reading, writing, and deploying contracts that transparently routes to the correct RPC methods depending on whether you are targeting eSpace (EVM) or Core Space.

The [UI Components & Hooks](ui-components-and-hooks.md) packages (`@cfxdevkit/react`, `@cfxdevkit/defi-react`, `@cfxdevkit/wallet-connect`) layer React hooks for accounts, balances, events, DeFi primitives, and wallet connection on top of all of the above.

---

## Developer Tooling

The [CLI & Developer Tools](cli-and-developer-tools.md) packages (`@cfxdevkit/cli` and `@cfxdevkit/create`) give you a `cfx` command for chain health checks, HD wallet derivation, and contract artifact extraction, plus `npm create @cfxdevkit` scaffolding. Both expose their logic as plain TypeScript functions so they can be called programmatically.

The [VS Code Extension](vs-code-extension.md) brings the full stack into the editor: a sidebar tree view manages a local devnode, an encrypted keystore, contract deployments, and live ABI interaction across local, testnet, and mainnet.

The [PI Agent & Provider Integration](pi-agent-and-provider-integration.md) module bridges the toolchain to the PI coding agent shell, exposing LLM-backed repo commands (`/repo-*`) alongside deterministic CDK wrappers (`/cdk`).

---

## LLM Automation Pipeline

The repo includes a first-class LLM layer used for commit quality gates, documentation generation, and code review. The [LLM Completion Engine](llm-completion-engine.md) handles provider routing, context assembly, and structured output parsing. The [LLM Agent Workflows](llm-agent-workflows.md) module builds on it to run validation pipelines, generate reports, and drive commit workflows. The [Docs Pipeline](docs-pipeline.md) (`@cfxdevkit/docs-pipeline`) uses this machinery to keep MDX content, wiki pages, and API docs in sync with the source.

Architecture enforcement, secret scanning, hotspot analysis, and documentation completeness checks all live in [Architecture Check & Repo Validation](architecture-check-and-repo-validation.md) (`@cfxdevkit/arch-check`) and run on every commit.

## Getting Started

```bash
# Install dependencies
pnpm install

# Build all packages
pnpm build

# Run the full check suite (arch, secrets, docs, CI)
pnpm check

# Start the local devnode
pnpm devnode

# Run tests with coverage
pnpm test:cover
```
