# Conflux DevKit Framework (`@cfxdevkit/framework`)

Welcome to the **Conflux DevKit** monorepo. This is the canonical reference for building on the [Conflux Network](https://confluxnetwork.org/). Whether you are integrating blockchain primitives, automating workflows with LLMs, or developing dApps, this repository provides the shared infrastructure, tooling, and SDKs required to build securely and efficiently.

This monorepo is structured as a **pnpm workspace** with a strict 5-tier dependency hierarchy. All architectural rules are enforced at commit time by the [`@cfxdevkit/arch-rules`](architecture-check-repo-validation.md) system.

## Architecture Overview

The system is designed around a central **Core Protocol** layer that abstracts the dual-chain nature of Conflux (Core Space and eSpace). Higher-level modules build on this foundation to provide wallets, smart contract interactions, local development environments, and automated agent workflows.

## Key Modules

### Blockchain Primitives
At the heart of the SDK is the [`Core Protocol & Chain`](core-protocol-chain.md) module. It provides a unified TypeScript client for both **Core Space** (native Conflux) and **eSpace** (EVM-compatible), handling address encoding, chain registry, and RPC abstraction.

Building on this, the [`Solidity Compiler & Contracts`](solidity-compiler-contracts.md) package manages ABI encoding and transaction construction across both chains, while the [`Wallet Abstraction & Hardware`](wallet-abstraction-hardware.md) and [`Keystore & Wallet Services`](keystore-wallet-services.md) modules handle secure key management, signing policies, and hardware wallet integration. Note that all wallet operations flow through the core protocol, ensuring consistent error handling (e.g., `CryptoError` propagation during keystore operations).

### Local Development & Tooling
For local testing, the [`DevNode & Runtime`](devnode-runtime.md) module allows you to spawn deterministic Conflux nodes with pre-funded accounts and auto-mining enabled. This is essential for CI pipelines and local integration testing.

Developer productivity is supported by the [`CLI & Developer Tools`](cli-developer-tools.md) for scaffolding new projects, and the [`VS Code Extension`](vs-code-extension.md) which integrates node management and contract deployment directly into your editor.

### Automation & Intelligence
The DevKit includes a robust automation layer powered by the [`LLM Completion Engine`](llm-completion-engine.md). This engine standardizes provider connectivity and prompt execution, serving as the backbone for the [`LLM Agent Workflows`](llm-agent-workflows.md). These agents handle repository analysis, commit quality gates, and documentation enrichment.

When running agent checks (e.g., `RunAgentCheck`), the system resolves execution contexts and validates configuration paths before proceeding, ensuring that automation steps are deterministic and reproducible.

### Quality & Validation
All code contributions are validated by the [`Architecture Check & Repo Validation`](architecture-check-repo-validation.md) suite. This includes architecture rule enforcement, secret scanning, and documentation alignment checks. The [`Docs Pipeline`](docs-pipeline.md) module orchestrates the generation of API references and READMEs, ensuring documentation stays in sync with the codebase.

## Getting Started

1.  **Install Dependencies:**
    Ensure you have `pnpm` installed, then run:
    ```bash
    pnpm install
    ```

2.  **Local Development:**
    Start a local Conflux node using the DevNode CLI:
    ```bash
    pnpm devnode
    ```
    This will spin up Core and eSpace endpoints on `127.0.0.1:12537` and `127.0.0.1:8545` respectively.

3.  **Validation:**
    Before committing, run the full validation suite to ensure architectural compliance:
    ```bash
    pnpm repo:check
    ```

4.  **Documentation:**
    To sync and enrich documentation across the monorepo:
    ```bash
    pnpm docs:sync
    ```
