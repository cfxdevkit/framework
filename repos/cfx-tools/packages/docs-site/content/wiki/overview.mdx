# @cfxdevkit/framework

Welcome to the **Conflux DevKit** monorepo. This repository contains the complete ecosystem for building on the Conflux Network, including the shared TypeScript framework, developer platform tooling, LLM automation pipelines, and the Contract Automation Service (CAS).

The monorepo is structured around a **strict 5-tier dependency hierarchy** enforced by `@cfxdevkit/arch-rules`. Dependencies are strictly unidirectional, and the architecture is validated automatically at commit time to maintain system integrity.

## Key Components

### Root Module
The [Root](module-root.md) module (`@cfxdevkit/framework`) serves as the central orchestration layer. It defines workspace boundaries, `pnpm` configuration, and provides the suite of scripts used for building, linting, testing, and analyzing the ecosystem. All package development flows are routed through this root context.

### Framework Libraries
The [Repos](module-repos.md) module houses the core framework packages that power the entire ecosystem:
- **`cfx-core`**: The foundational layer providing RPC clients, type definitions, and chain configurations. Most other framework modules depend on this.
- **`cfx-keys`**: Implements cryptographic trust boundaries, including keystore management and key derivation using Argon2id. It has a heavy reliance on `cfx-core` for chain primitives.
- **`cfx-domain`**: Contains business logic for on-chain strategies and wallet interactions. It sits above `cfx-core` and is consumed by higher-level projects.
- **`cfx-ui`**: Shared React components and hooks for building Conflux-facing applications.
- **`cfx-tools`**: Utility helpers and path resolution logic used across the monorepo.

### Projects and Applications
The [Projects](module-projects.md) module encapsulates full-stack applications built on the framework:
- **`cas`**: The Contract Automation Service, a non-custodial system for executing on-chain strategies (DCA, TWAP, limit orders). It relies heavily on `cfx-domain` and `cfx-ui`.
- **`examples`**: Showcase applications demonstrating framework usage, including the local devnode environment. These projects integrate `cfx-ui`, `cfx-tools`, and `cfx-keys`.

### Automation and Tooling
The [Scripts](module-scripts.md) module provides the automation glue for the monorepo. It handles repository maintenance, automated refactoring (including "kebab-group" import resolution), package publishing, and local development environment management.

## End-to-End Flows

Several critical flows span multiple modules, demonstrating how the system behaves in practice:

- **Crypto and Keystore Operations**: When storing a key (`put` in `keystore/file/index.ts`), the system ensures the Key Encryption Key (KEK) exists, unlocks and probes the keystore, derives the KEK from an envelope using Argon2id, and brands the key. Errors are propagated via the `CryptoError` class.
- **Agent Configuration Resolution**: The `RunConfigCli` flow resolves agent configurations by traversing the workspace. It calls `FindRepoRoot` to locate the monorepo root, then resolves unit configurations through `findMonorepoUnit` and `resolveAgentConfigPath`.
- **Wallet and UI Context**: Pages like `CorePage` and `OneKeyPage` consume the `useCoreWallet` hook, which syncs state and normalizes the chain ID via `normalizeCoreChainId` to ensure consistent network context across the application.
- **Agent Health Checks**: The `RunAgentCheck` worker resolves the execution context and loads base configurations. It uses `isErrnoException` guards to safely handle filesystem access errors during configuration loading.

## Getting Started

1. **Install Dependencies**:
   ```bash
   pnpm install
   ```

2. **Verify Architecture**:
   Ensure the dependency graph is valid before committing:
   ```bash
   pnpm arch:check
   ```

3. **Local Development**:
   Use the provided scripts to spin up a local development node and run the showcase applications:
   ```bash
   pnpm devnode
   pnpm showcase:public
   ```

4. **LLM and Automation**:
   The repository includes an extensive LLM pipeline for code quality gates, doc generation, and automated refactoring. Run the agent configuration helper to get started:
   ```bash
   pnpm agent:config
   ```
