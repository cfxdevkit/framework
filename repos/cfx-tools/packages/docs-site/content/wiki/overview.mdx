# Conflux DevKit Framework Overview

Welcome to the **Conflux DevKit** (`@cfxdevkit/framework`). This monorepo serves as the canonical reference for building on the Conflux Network, encompassing shared TypeScript frameworks, developer platform tooling, LLM automation pipelines, and example applications.

The repository is structured as a **pnpm monorepo** with a strict 5-tier dependency hierarchy enforced by `@cfxdevkit/arch-rules`. This ensures that core protocol logic remains isolated from tooling and application layers, maintaining stability across the ecosystem.

## High-Level Architecture

The system is organized around a central **Core Protocol** layer, with specialized modules branching out for runtime, security, development tools, and automation. The diagram below illustrates the primary dependencies and execution flow between the top 10 modules.

## Key Capabilities

### Development & Runtime
To build locally, the **DevNode & Runtime** module provides a dual-space (Core + eSpace) Conflux node environment with pre-funded accounts. It wraps the native node binary to ensure deterministic ports for testing. Developers interact with this environment primarily through the **CLI & Developer Tools**, which offers commands for wallet management, contract extraction, and project scaffolding (`create-conflux`).

### Security & Signing
All cryptographic operations converge through the **Keystore & Wallet Services** module. It provides a unified interface for secret storage, supporting file-based encryption, memory backends, and hardware wallets (Ledger, OneKey). The **Wallet Abstraction & Hardware** layer sits on top, managing signing sessions and transaction batching. This security layer is heavily utilized by the **VS Code Extension**, which integrates wallet connection and transaction signing directly into the IDE.

### Smart Contracts
The **Solidity Compiler & Contracts** package abstracts the differences between Conflux Core Space and EVM-compatible eSpace. It handles address formatting and RPC dispatch automatically, allowing developers to interact with contracts using a single TypeScript interface built on `viem`.

### Automation & Governance
The repository is self-governing. The **Architecture Check & Repo Validation** module enforces dependency rules, security policies, and documentation standards at commit time. These checks feed into the **LLM Agent Workflows**, which automate code review, commit quality gates, and documentation upkeep. The **Docs Pipeline** then synchronizes all package metadata, READMEs, and API docs into a consistent Nextra-based documentation site.

## Key Workflows

### Secure Keystore Unlock
When a user interacts with a wallet via the UI or Extension, the system follows a deterministic decryption path:
1.  **KeystorePanel** initiates the unlock request.
2.  The request flows through the **Keystore Service** runtime to read the stored mnemonic.
3.  **File Keystore** operations decrypt the envelope using Argon2id key derivation.
4.  The derived key brands the private key, which is then exposed to the signing interface.

### Repository Governance
CI/CD pipelines rely on the **Architecture Check** module to validate repository health:
1.  **Agent Workflows** trigger validation checks before commits.
2.  **Architecture Check** scans for dependency violations, secrets, and documentation staleness.
3.  If changes are detected, the **Docs Pipeline** regenerates affected markdown artifacts (API.md, README.md) with embedded content hashes to prevent unnecessary diffs.

## Getting Started

This repository uses `pnpm` for package management. Ensure you have Node.js v18+ installed.

1.  **Install Dependencies**
    ```bash
    pnpm install
    ```

2.  **Build the Workspace**
    ```bash
    pnpm build
    ```

3.  **Run Tests**
    ```bash
    pnpm test
    ```

4.  **Start Local DevNode**
    ```bash
    pnpm devnode:start
    ```

For detailed workflow conventions, see [CONTRIBUTING.md](CONTRIBUTING.md). For design rationale, see [ARCHITECTURE.md](ARCHITECTURE.md).
