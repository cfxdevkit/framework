# PI Agent & Provider Integration Module

## Overview

The PI Agent & Provider Integration module provides repository agent commands, tools, and UI integration for the PI coding agent system. It enables developers to execute repository workflows, validate code changes, manage commits, and interact with LLM providers through a unified command interface.

## Key Components

### Commands (`commands.ts`)

Repository agent commands are registered through `registerPiRepoCommands()`. Each command provides a specific workflow capability:

| Command | Description | Handler |
|---------|-------------|---------|
| `repo-status` | Show current repo agent scope, provider, and model context | `createPiRuntimeUiState()` |
| `repo-actions` | List available shared repo workflows | `renderPiActionCatalogLines()` |
| `repo-run` | Run a shared repo action with optional prompt | `executePiRepoAction()` |
| `repo-check` | Run full repo validation → OpenSpec change planning | `executePiAgentCheck()` |
| `repo-commit` | Run non-exiting repository commit workflow | `executePiCommitSession()` |

**Command Arguments:**
```typescript
// repo-run: /repo-run <action> [--quick] [--model <id>] [prompt]
// repo-check: /repo-check [--dry-run] [--create-branch] [--quick]
// repo-commit: /repo-commit [--quick] [--model <id>] [prompt]
```

### Tools (`tools.ts`)

Tools are registered via `registerPiRepoTools()` and expose repository functionality to the agent:

| Tool | Name | Purpose |
|------|------|---------|
| `repo_agent_check` | Validates repository and creates OpenSpec changes |
| `repo_action_catalog` | Lists available repository workflows |
| `repo_run_action` | Executes a specific repository workflow |
| `repo_commit_workflow` | Manages interactive commit workflow |

```typescript
// Example tool registration
export function registerPiRepoTools(pi: ExtensionAPI): void {
  pi.registerTool(repoAgentCheckTool);
  pi.registerTool(repoActionCatalogTool);
  pi.registerTool(repoRunActionTool);
  pi.registerTool(repoCommitWorkflowTool);
}
```

### Extension Lifecycle (`extension.ts`)

The extension is registered through `registerPiAgentProjectExtension()`:

```typescript
export async function registerPiAgentProjectExtension(
  pi: ExtensionAPI
): Promise<void> {
  // 1. Initialize provider bridge
  registerPiProviderBridge(pi, await createPiProviderBridge(resolvePiScopeFromEnv()));
  
  // 2. Register commands
  registerPiRepoCommands(pi);
  registerPiCdkCommands(pi);
  
  // 3. Register tools
  registerPiRepoTools(pi);

  // 4. Session lifecycle hooks
  pi.on('session_start', async (_event, ctx) => {
    await refreshPiRuntimeUi(ctx);
  });

  pi.on('session_shutdown', async (_event, ctx) => {
    ctx.ui.setStatus('repo-agent', undefined);
    clearPiOperatorWidgets(ctx);
    clearPiWorkflowProgress(ctx);
  });
}
```

## Execution Flow

### Command Execution

```typescript
// 1. User invokes command
// /repo-run my-action --quick --model gpt-4

// 2. Command handler parses arguments
const parsed = parseRepoRunArgs(args);
// { action: 'my-action', quick: true, model: 'gpt-4', prompt: undefined }

// 3. Set workflow progress
setPiWorkflowProgress(ctx, `Running repo action: my-action`);

// 4. Execute action
const result = await executePiRepoAction({
  action: parsed.action,
  quick: parsed.quick,
  model: parsed.model,
});

// 5. Create UI state
const uiState = createPiRepoActionUiState(result);

// 6. Emit operator message
emitPiOperatorMessage(pi, ctx, uiState, { tone: 'success' });

// 7. Clear progress
clearPiWorkflowProgress(ctx);
```

### Tool Execution

```typescript
// 1. Tool is invoked by agent
const result = await executePiRepoAction({
  action: 'my-action',
  prompt: 'custom prompt',
  quick: false,
  model: undefined,
});

// 2. Tool returns structured response
return {
  content: [{ type: 'text', text: result.response.content }],
  details: {
    action: result.action,
    title: result.definition.title,
    mode: result.definition.mode,
    executionContext: result.executionContext,
  },
};
```

## LLM Agents Integration (`llm-agents-runtime.ts`)

The module integrates with `@cfxdevkit/llm-agents` for action execution:

```typescript
// Load LLM agents module dynamically
async function loadLlmAgentsModule(): Promise<LlmAgentsModule> {
  if (existsSync(llmAgentsDistEntry)) {
    return (await import(llmAgentsPackageName)) as LlmAgentsModule;
  }
  return (await import(llmAgentsSourceModulePath)) as LlmAgentsModule;
}

// Execute repository action
export async function executePiAction(
  args: string[]
): Promise<PiRepoActionExecutionResult> {
  return await (await loadLlmAgentsModule()).executeAction(args);
}
```

## Error Handling

Commands wrap execution in try-catch blocks with UI notification:

```typescript
try {
  const result = await executePiRepoAction({ ... });
  emitPiOperatorMessage(pi, ctx, createPiRepoActionUiState(result), { tone: 'success' });
} catch (error) {
  if (ctx.hasUI) {
    ctx.ui.notify(error instanceof Error ? error.message : String(error), 'error');
  }
} finally {
  clearPiWorkflowProgress(ctx);
}
```

## Contributing

When adding new commands:
1. Register in `registerPiRepoCommands()`
2. Add handler with argument parsing
3. Create corresponding UI state function in `ui.ts`
4. Add tool definition in `tools.ts` if agent-accessible
5. Update tests in corresponding test files

When adding new tools:
1. Define tool with `defineTool()`
2. Register in `registerPiRepoTools()`
3. Ensure proper error handling and UI state updates
4. Add TypeBox schema for parameters
