# Repository Layout — cfx-solidity

# cfx-solidity

**Solidity pipeline.** An independent carve-out providing standard ABIs, contract bindings, and the Solidity compilation pipeline. This module is usable on its own without pulling the rest of the devkit.

## Architecture & Layering

The `cfx-solidity` repository is organized into four distinct packages. Dependencies flow strictly from leaf to root to prevent circular dependencies and ensure stability.

<Mermaid chart={`graph TD
    A[viem] --> B[abis]
    B --> C[contracts]
    C --> D[compiler]
    E[cdk] --> C
    E --> D
    F[solc] --> D
    G[contracts-extract] -.->|Standalone CLI| H[Hardhat Artifacts]
`} />

| Package | npm Scope | Surface | Dependencies |
| :--- | :--- | :--- | :--- |
| `abis` | `@cfxdevkit/abis` | Standard ABI shapes (ERC-20/721/1155, Multicall3) | `viem` only |
| `contracts` | `@cfxdevkit/contracts` | Read/write/deploy + ERC-20 helpers + Conflux bridge | `abis`, `@cfxdevkit/cdk`, `viem` |
| `compiler` | `@cfxdevkit/compiler` | Solidity compile pipeline (solc loader, resolvers, templates) | `@cfxdevkit/cdk`, `solc`, `viem` |
| `contracts-extract` | `@cfxdevkit/contracts-extract` | Hardhat artifact → TS module codegen | Standalone CLI |

## Package: `@cfxdevkit/abis`

Standard EVM ABI shapes used across the devkit. Definitions are sourced from [`viem`](https://viem.sh) and re-exported under framework-stable aliases.

### Key Characteristics
*   **Zero cfxdevkit dependencies**: Only depends on `viem`.
*   **Tiny surface**: Constants only. No runtime logic.
*   **Stable**: Insulates downstream code from upstream re-export churn.

### Installation
```bash
npm install @cfxdevkit/abis
```

### Public API

#### Core Standards
Import standard ABI constants and addresses from the root path:

```ts
import {
  ERC20_ABI,
  ERC20_EXTENDED_ABI,
  ERC2612_ABI,
  ERC4626_ABI,
  ERC721_ABI,
  ERC721_EXTENDED_ABI,
  ERC1155_ABI,
  ERC165_ABI,
  ERC721_ENUMERABLE_ABI,
  ERC2981_ABI,
  MULTICALL3_ABI,
  MULTICALL3_ADDRESS,
} from '@cfxdevkit/abis';
```

#### Swappi Protocol
Import DEX-specific ABIs and addresses from the sub-path:

```ts
import {
  SWAPPI_FACTORY_ABI,
  SWAPPI_PAIR_ABI,
  SWAPPI_ROUTER_ABI,
  SWAPPI_FACTORY_ADDRESS,
  SWAPPI_ROUTER_ADDRESS,
  WCFX_ADDRESS,
} from '@cfxdevkit/abis/swappi';
```

### Directory Structure
```
abis/
├── src/
│   ├── index.ts                    # Stable ABI constants and compatibility aliases
│   └── index.test.ts               # ABI shape and signature assertions
```

## Package: `@cfxdevkit/contracts`

Standard contract bindings and a thin, framework-native `read` / `write` / `deploy` surface for `@cfxdevkit/cdk`. This package handles interaction logic, error handling, and specific protocol helpers (ERC-20, Bridge).

### Installation
```bash
npm install @cfxdevkit/contracts @cfxdevkit/cdk
```

### Interaction Patterns

#### Read Calls
Use `readContract` for view/pure functions. Requires a `client` from `@cfxdevkit/cdk`.

```ts
import { readContract } from '@cfxdevkit/contracts';
import { erc20 } from '@cfxdevkit/contracts/erc20';

const balance = await readContract({
  client,
  abi: erc20.abi,
  address: '0xToken…',
  functionName: 'balanceOf',
  args: ['0xUser…'],
});
```

#### Write Calls
Use `prepareWrite` to construct a transaction, then `sendWrite` to submit it.

```ts
import { prepareWrite, sendWrite } from '@cfxdevkit/contracts';
import { erc20 } from '@cfxdevkit/contracts/erc20';

const tx = await prepareWrite({
  client,
  abi: erc20.abi,
  address: '0xToken…',
  functionName: 'transfer',
  args: ['0xUser…', 1000n],
});

const receipt = await sendWrite({ client, tx });
```

#### Deploy
Deploy contracts using `deployContract`.

```ts
import { deployContract } from '@cfxdevkit/contracts';

const result = await deployContract({
  client,
  abi: myContractAbi,
  bytecode: '0x...',
  args: ['arg1', 42n],
});
```

### Sub-Paths

| Sub-path | Purpose | Key Exports |
| :--- | :--- | :--- |
| `.` | Barrel export | `readContract`, `prepareWrite`, `sendWrite`, `deployContract`, `ContractsError` |
| `./abis` | ABI constants | `ERC20_ABI`, `MULTICALL3_ADDRESS`, etc. |
| `./read` | Read operations | `readContract`, `ReadContractInput` |
| `./write` | Write operations | `prepareWrite`, `sendWrite`, `waitForReceipt` |
| `./deploy` | Deployment | `deployContract`, `DeployContractInput` |
| `./erc20` | ERC-20 helpers | `erc20` (typed helpers for symbol, decimals, etc.) |
| `./bridge` | Conflux Bridge | `mappedEspaceAddress`, `transferToEspace`, `callEspace` |
| `./errors` | Error types | `ContractsError`, `ContractsErrorCode` |

#### Bridge Helpers
The `./bridge` sub-path provides utilities for cross-space transfers between Core Space and eSpace.

```ts
import { mappedEspaceAddress, transferToEspace } from '@cfxdevkit/contracts/bridge';

// Convert core space address to mapped eSpace address
const espaceAddr = mappedEspaceAddress(coreHexAddress);

// Transfer to eSpace
await transferToEspace({ client, to: espaceAddr, amount: 100n });
```

#### Error Handling
All errors extend `ContractsError` and are typed via `ContractsErrorCode`.

```ts
import { ContractsError } from '@cfxdevkit/contracts';

try {
  await readContract(...);
} catch (err) {
  if (err instanceof ContractsError) {
    console.error(err.code); // e.g., 'contracts/reverted', 'contracts/decode-failure'
  }
}
```

### Directory Structure
```
contracts/
├── src/
│   ├── index.ts                    # Barrel (re-exports every sub-path)
│   ├── abis/                       # ERC20/721/1155 + Multicall3 ABIs
│   ├── errors/                     # ContractsError + ContractsErrorCode
│   ├── read/                       # readContract()
│   ├── write/                      # prepareWrite() / sendWrite() / waitForReceipt()
│   ├── deploy/                     # deployContract()
│   ├── erc20/                      # Typed ERC-20 helpers
│   └── test/
│       └── mocks.ts                # In-memory Client + recording Signer for tests
```

### Dependencies & Build
*   **Runtime**: `viem` (used purely as encoder/decoder), `@cfxdevkit/cdk`.
*   **Build**: `vite-plugin-dts` emits one `.d.ts` per entry point.
*   **Testing**: 21 unit tests (vitest, node env) using in-memory mocks. No live RPC required.

## Package: `@cfxdevkit/compiler`

Solidity compilation pipeline. Handles solc loading, resolvers, templates, and artifact generation.

*   **Dependencies**: `@cfxdevkit/cdk`, `solc`, `viem`.
*   **Usage**: DevDep on `contracts/devnode` for tests.
*   **Functionality**: Compiles Solidity sources into artifacts compatible with the devkit ecosystem.

---

## Package: `@cfxdevkit/contracts-extract`

Standalone CLI tool for code generation.

*   **Functionality**: Converts Hardhat artifacts into TypeScript module code.
*   **Usage**: Used to generate type-safe bindings from existing contract artifacts.

## Development Guidelines

### Adding New ABIs
1.  Update `@cfxdevkit/abis` if the ABI is standard (ERC, Multicall) or widely used (Swappi).
2.  Ensure the ABI is sourced from `viem` where possible to maintain stability.
3.  Add tests in `abis/src/index.test.ts` to assert shape and signature.

### Adding Contract Helpers
1.  Implement logic in `@cfxdevkit/contracts` under the appropriate sub-path (`read`, `write`, `erc20`, `bridge`).
2.  Use `@cfxdevkit/cdk` `Client` for transport and signing.
3.  Wrap errors in `ContractsError` with specific `ContractsErrorCode` values.
4.  Add unit tests in `contracts/src/<sub-path>/<sub-path>.test.ts`.

### Testing Strategy
*   **Unit Tests**: Run against `src/test/mocks.ts` (in-memory client).
*   **Integration Tests**: Live-network smoke tests belong in `tools/testing` or per-app suites, not in this package.
*   **No External RPC**: Tests must not rely on live RPC endpoints to ensure CI stability.
