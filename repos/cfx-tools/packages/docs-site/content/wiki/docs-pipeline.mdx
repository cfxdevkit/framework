# Docs Pipeline

The **Docs Pipeline** module automates documentation generation, synchronization, and validation across the `@cfxdevkit` monorepo. It transforms package metadata, READMEs, API docs, and wiki content into a consistent Nextra-based documentation site while maintaining integrity through content hashing and validation.

## Overview

This module provides:

- **Package documentation sync**: Generates MDX pages from `package.json`, `README.md`, and `API.md` files
- **Wiki synchronization**: Converts wiki markdown files to MDX with Mermaid diagram support
- **Architecture documentation**: Auto-generates layered architecture diagrams and package maps
- **Coverage reporting**: Aggregates test coverage data into a summary page
- **Content validation**: Validates MDX syntax and Mermaid diagram correctness

## CLI Usage

```bash
pnpm --filter @cfxdevkit/docs-pipeline run docs -- <command> [target]
```

### Commands

| Command | Target | Description |
|---------|--------|-------------|
| `sync` | `all` | Sync all documentation sources |
| `sync` | `packages` | Sync package documentation pages |
| `sync` | `wiki` | Sync wiki content from `.gitnexus/wiki` |
| `sync` | `architecture` | Generate architecture documentation |
| `sync` | `coverage` | Generate coverage summary page |
| `validate` | `content` | Validate all generated MDX files |
| `validate` | `wiki` | Validate wiki Mermaid diagrams |
| `validate` | `wiki-fix` | Validate and sanitize broken diagrams |

### Examples

```bash
# Sync all documentation
pnpm --filter @cfxdevkit/docs-pipeline run docs -- sync all

# Sync only package pages
pnpm --filter @cfxdevkit/docs-pipeline run docs -- sync packages

# Validate wiki diagrams and fix broken ones
pnpm --filter @cfxdevkit/docs-pipeline run docs -- validate wiki-fix
```

## Core Components

### Package Discovery

**`discover-packages.ts`** scans the monorepo structure to identify public packages eligible for documentation.

```typescript
// Key exports
discoverDocsPackages(repoRoot: string): Promise<DocsPackageRecord[]>
discoverDocsPagePackages(options: { repoRoot?: string; packageName?: string }): Promise<DocsPackagePageRecord[]>
```

**Package Discovery Flow**:
1. Scans `repos/*/packages` and `repos/*/infra` directories
2. Reads `package.json` to identify public packages
3. Filters out skipped packages (config packages, private packages)
4. Collects README.md and API.md content

**Skip List**: Config packages like `@cfxdevkit/biome-config`, `@cfxdevkit/tsconfig` are excluded from documentation.

### Content Generation

**`package-content.ts`** handles MDX skeleton generation and content hashing.

#### Content Hashing

Each package page includes an embedded hash that tracks content changes:

```typescript
// Hash embedded in generated MDX
{/* readme-hash: <sha256> */}
```

**Functions**:
- `computeSkeletonHash()`: Generates SHA256 hash from package metadata
- `embedHash()`: Inserts hash into content
- `readEmbeddedHash()`: Extracts hash from existing content
- `stripEmbeddedHash()`: Removes hash for comparison

#### MDX Skeleton Generation

```typescript
generateMdxSkeleton({
  name,
  description,
  exports,
  readme,
  api,
}): string
```

The generated skeleton includes:
- Frontmatter with title and description
- Installation tabs (pnpm/npm/yarn)
- README body (stripped of install sections)
- API reference section
- Sub-paths table from `package.json` exports

**Content Processing**:
1. Strips H1 headers and blockquotes from README
2. Removes "Install" sections from README
3. Removes "Sub-paths" sections from API docs
4. Converts JSDoc links to backtick-wrapped text
5. Replaces HexAddress with `HexAddress`

### Package Page Synchronization

**`package-pages.ts`** syncs package documentation to the docs site.

```typescript
// Key exports
syncPackages(): Promise<SyncPackagesResult>
```

**Sync Logic**:
1. Discovers all public packages
2. For each package:
   - Generates MDX skeleton
   - Computes content hash
   - Compares with existing file
   - Updates if hash changed or MDX invalid
3. Updates `_meta.js` with package list
4. Fixes `index.mdx` card links

**Update Triggers**:
- Content hash mismatch (package metadata changed)
- Invalid MDX syntax detected
- Stale content markers (`<!--` or `{@link`)

### Wiki Synchronization

**`wiki-sync.ts`** converts wiki markdown files from `.gitnexus/wiki` to MDX.

```typescript
// Key exports
syncWiki(): Promise<number>
fixMermaidLabels(diagram: string): string
simplifyMermaidDiagram(diagram: string): string
```

**Mermaid Processing**:
1. **Label Quoting**: Fixes unquoted node labels with special characters
   - `[@cfxdevkit/cdk]` → `["@cfxdevkit/cdk"]`
   - `[src/index.ts]` → `["src/index.ts"]`
2. **Diagram Simplification**: Reduces diagrams over 30 lines to top-level connections
3. **Content Conversion**: Wraps Mermaid blocks in `<Mermaid>` component

**Stale File Cleanup**: Removes `.mdx` files whose source `.md` no longer exists.

### Architecture Documentation

**`sync-architecture.ts`** generates layered architecture documentation.

```typescript
// Key exports
syncArchitecturePage(): Promise<number>
```

**Tier Model**:
- **Tier 0**: Framework packages (cfx-core, cfx-keys, cfx-ui, cfx-solidity)
- **Tier 1**: Developer Platform (cfx-tools)
- **Tier 2**: Domain Logic (cfx-domain)
- **Tier 3**: Projects & Apps (not included in scan)

**Output**:
- ASCII diagram showing tier relationships
- Package map table with tier and repo slice
- Dependency rules documentation

### Coverage Reporting

**`sync-coverage.ts`** aggregates test coverage data from `coverage-summary.json`.

```typescript
// Key exports
syncCoveragePage(): Promise<{ withCoverage: number; total: number }>
```

**Data Sources**:
1. `coverage/coverage-summary.json` (Vitest output)
2. `coverage-summary.json` (alternative location)

**Output**:
- Summary table with statement/branch/function/line coverage
- List of packages without coverage data
- Last updated timestamp

### Validation

#### MDX Validation

**`validate-content.ts`** validates all generated MDX files compile correctly.

```typescript
// Key exports
validateGeneratedContent(): Promise<ValidationResult>
```

Uses `@mdx-js/mdx` with `remark-gfm` and `remark-frontmatter` to verify syntax.

#### Mermaid Validation

**`wiki-validate.ts`** validates Mermaid diagrams in wiki content.

```typescript
// Key exports
validateWikiMermaid(options: { fix?: boolean }): Promise<{
  totalBlocks: number
  brokenBlocks: number
  fixedBlocks: number
}>
```

**Validation Methods**:
1. **Heuristic Check**: Validates diagram type, direction, bracket balance, unclosed subgraphs
2. **Mermaid Parser**: Attempts to parse with `mermaid.parse()` (browser-dependent)
3. **Sanitization**: Downgrades broken diagrams to plain text fences

**Fix Mode**: When `fix: true`, broken diagrams are replaced with warning messages.

## Command Execution Flow

```typescript
// scripts.ts orchestrates command execution
runCommand(command: DocsCommandName): Promise<void>
```

**Command Map**:
- `sync:all` → sync:wiki + sync:packages + sync:architecture + sync:coverage
- `sync:packages` → syncPackages()
- `sync:wiki` → syncWiki()
- `sync:architecture` → syncArchitecturePage()
- `sync:coverage` → syncCoveragePage()
- `validate:content` → validateGeneratedContent()
- `validate:wiki` → validateWikiMermaid()
- `validate:wiki-fix` → validateWikiMermaid({ fix: true })

## Contributing

### Adding New Package Types

1. Update `REPO_DIRS` in `discover-packages.ts` if new repository directories are added
2. Add to `SKIP_PACKAGES` if certain packages should not be documented
3. Ensure packages have `package.json` with `name` and `exports` fields

### Modifying MDX Generation

1. Edit `generateMdxSkeleton()` in `package-content.ts`
2. Update `SYNC_VERSION` constant when hash format changes
3. Test with `pnpm sync:packages`
4. Validate with `pnpm validate:content`

### Fixing Mermaid Diagrams

1. Add new special characters to `NEEDS_QUOTE` in `wiki-sync.ts`
2. Adjust `MERMAID_MAX_LINES` threshold in `wiki-sync.ts`
3. Add diagram types to `VALID_TYPES` in `wiki-validate.ts`
4. Test with `pnpm validate:wiki-fix`

## Error Handling

### Common Errors

| Error | Cause | Resolution |
|-------|-------|------------|
| `Unknown docs-pipeline command` | Invalid command name | Check command list in `commands.ts` |
| `Invalid target` | Invalid sync/validate target | Use valid targets from usage |
| `Cannot find workspace root` | Missing `pnpm-workspace.yaml` | Run from monorepo root |
| `mermaid.parse() unavailable` | DOMPurify requires browser | Use heuristic validation only |

### Debugging

Enable verbose logging by running commands from the package directory:

```bash
cd repos/cfx-tools/packages/docs-pipeline
pnpm run docs -- sync packages
```
