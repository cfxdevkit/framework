# @cfxdevkit/defi-react

## Overview
`@cfxdevkit/defi-react` is a Tier 0c React UI surface designed for decentralized finance interfaces. It provides opinionated widgets, low-level UI primitives, and data-fetching hooks for Conflux eSpace. The library is styled entirely via CSS custom properties from `@cfxdevkit/theme/css`, carries zero external CSS/framework dependencies, and is structured as a carve-out target per ADR-0003.

## Architecture & Module Layout
The package is published as a single entry point but exposes granular subpaths for tree-shaking and modular imports:

| Subpath | Purpose |
|---------|---------|
| `./primitives` | Unstyled UI building blocks (buttons, inputs, layouts, feedback, navbar, trade helpers) |
| `./swap` | Swap UI (`SwapWidget`, `TokenPicker`, `useSwap`, `createSwappiAdapter`) |
| `./balance` | Portfolio display (`PortfolioTable`, `usePortfolio`) |
| `./pool` | Pool & pricing hooks (`usePoolTokens`, `usePools`, `useTokenPrice`) |
| `./lp` | Liquidity provision (`AddLiquidityWidget`, `PoolShareBadge`) |
| `./tx-status` | Transaction tracking (`TxStatusList`, `TxStatusToast`) |
| `./service` | DEX abstraction (`SwapService`, `DexAdapter` interface) |
| `./types` | Shared interfaces (`TokenInfo`, `Quote`, `TokenRegistry`, `TrackedTx`) |

All components and hooks rely on:
- `@cfxdevkit/cdk` for `EspaceClient`, `Address`, `Wei`, and `Hash` types
- `@cfxdevkit/react` for account, context, contract, and transaction hooks
- `@tanstack/react-query` for caching, background refetching, and mutation management
- `viem` for ABI encoding/decoding and low-level RPC calls

## Core Primitives
`@cfxdevkit/defi-react/primitives` exports a comprehensive set of CSS-variable-driven components. They are intentionally unopinionated beyond the design token system and require `@cfxdevkit/theme/css` to be imported as a side-effect in your application entry point.

Key exported primitives:
- **Layout & Structure**: `AppShell`, `MainGrid`, `Panel`, `PanelBody`, `SectionHeader`, `StatusBanner`, `MetricCard`, `StatusGrid`, `Metric`
- **Navigation**: `AppNavBar`, `NavBrand`, `NavWalletActions`
- **Forms & Controls**: `Button`, `Input`, `SelectMenu`, `SegmentedControl`, `Tabs`, `Badge`, `Field`, `Notice`, `IconButton`
- **Feedback & Devkit**: `AppToaster`, `SelectableListItem`, `FaucetWidget`, `DevkitStatus`, `NetworkBadge`
- **Trade UI**: `TradeTokenField`, `TradeActionBar`, `TradeSummaryGrid`

All components accept inline `style` props and forward standard DOM attributes where applicable.

## DeFi Widgets & Hooks

### Swap & Liquidity
- **`SwapWidget`**: Headless-styled swap UI that manages token selection, amount input, quote display, and swap lifecycle. Uses `useSwap` under the hood and renders a modal `TokenPicker` when token selection is triggered.
- **`AddLiquidityWidget`**: Dual-token input widget for Swappi V2 liquidity provision. Auto-fills the second token amount based on pool reserves, displays price ratios, and estimates pool share percentage via `PoolShareBadge`.
- **`TokenPicker`**: Searchable, chain-filtered token selection component. Consumes a `TokenRegistry` and performs client-side filtering via `registry.search(query, chainId)`.
- **`useSwap`**: Manages the full swap flow:
  1. Queries `adapter.getQuote()` via `useQuery`
  2. Checks ERC-20 allowance via `useReadContract`
  3. Triggers `approveAsync` (ERC-20 only) via `useMutation`
  4. Executes `swapAsync` via `useSendTransaction` or `useWriteContract`
  5. Tracks receipt via `useWaitForTransaction`

### Portfolio & Balances
- **`usePortfolio`**: Fetches native CFX + ERC-20 balances in parallel using `useQueries`. Automatically merges wrapped-native (WCFX) balances into the native CFX row using `resolveDisplayTokenAddress`. Returns `rows` sorted by balance descending, with human-readable formatting via `formatBigInt`.
- **`PortfolioTable`**: Renders the balance rows. Handles loading, error, and empty states. Accepts a `renderRow` override for custom cell rendering.

### Pool & Pricing
- **`usePoolTokens`** / **`usePools`**: React Query wrappers around `SwapService.getPoolTokens()`. The bulk hook (`usePools`) returns a `Map<Address, PoolData>` for efficient multi-pair polling.
- **`useTokenPrice`**: Queries the price of a token relative to a quote token via `SwapService.getTokenPrice()`. Returns the raw `bigint` output for 1e18 units of input.

### Transaction Tracking
- **`TxStatusList`** & **`TxStatusToast`**: Tracks submitted transaction hashes and polls for receipts using `useWaitForTransaction`. Managed via `TxListProvider` context. Use `useTxList().track(hash)` to register transactions for live status updates.

## Service Layer & DEX Integration
The module abstracts DEX interactions through the `DexAdapter` interface:

```ts
interface DexAdapter {
  getSpenderAddress(): Address;
  getQuote(params: { tokenIn, tokenOut, amountIn, slippageBps?, deadlineMs? }): Promise<Quote>;
  buildCalldata(quote: Quote, options?): Promise<SwapCalldata>;
  getPoolTokens(pairAddress): Promise<PoolTokens>;
  getTokenPrice(tokenAddress, quoteToken): Promise<bigint>;
  getPair(tokenA, tokenB): Promise<Address>;
}
```

**`SwapService`** implements `DexAdapter` for Swappi V2:
- Hardcodes factory, router, and WCFX addresses per chain (`1030` mainnet, `71` testnet)
- Handles native/WCFX routing logic (single-hop vs multi-hop via WCFX)
- Encodes calldata for `swapExactETHForTokens`, `swapExactTokensForETH`, and `swapExactTokensForTokens`
- Exposed via factory function `createSwapService(config: SwapServiceConfig)`

<Mermaid chart={`flowchart TD

  C --> D[viem encodeFunctionData + eth_call]

  G --> B
`} />

1. **User Interaction**: `SwapWidget` or `AddLiquidityWidget` captures inputs and token selection.
2. **Hook Execution**: `useSwap` or `usePortfolio` triggers React Query hooks.
3. **Service Abstraction**: `SwapService` resolves routing paths, handles native/WCFX conversion, and builds calldata via `viem`.
4. **On-Chain Execution**: Transactions are signed via `@cfxdevkit/react`'s `useSendTransaction`/`useWriteContract` and submitted to the RPC.
5. **State Sync**: `useWaitForTransaction` and `useReadContract` keep the UI in sync with on-chain state and allowances.

## Integration Requirements

### Prerequisites
- `@cfxdevkit/theme/css` must be imported as a side-effect in your app's root layout so CSS custom properties resolve.
- Peer dependencies: `react ^19`, `viem ^2`, `@tanstack/react-query ^5`, `@cfxdevkit/theme ^`
- Workspace dependencies (if using monorepo): `@cfxdevkit/abis`, `@cfxdevkit/cdk`, `@cfxdevkit/react`, `@cfxdevkit/ui-core`

### Typical Setup
```tsx
import '@cfxdevkit/theme/css'; // Required for CSS variables
import { AppNavBar, AppShell, Panel, Button } from '@cfxdevkit/defi-react/primitives';
import { SwapWidget, createSwappiAdapter, TxStatusList } from '@cfxdevkit/defi-react/swap';
import { TxListProvider } from '@cfxdevkit/defi-react/tx-status';
import { useClient } from '@cfxdevkit/react/context';

function App() {
  const client = useClient();
  const adapter = createSwappiAdapter({ chainId: 1030, client });

  return (
    <TxListProvider>
      <AppShell>
        <AppNavBar brand={<NavBrand name="DeFi App" />} links={[]} />
        <SwapWidget adapter={adapter} tokens={MY_TOKENS} onSwapSubmitted={tx => 
          useTxList().track(tx.hash, 'Swap')
        } />
        <TxStatusList recent={10} />
      </AppShell>
    </TxListProvider>
  );
}
```

### Key Patterns
- **Controlled State**: Widgets like `SwapWidget` manage internal UI state but expose lifecycle callbacks (`onSwapSubmitted`, `onAddLiquidity`) for external transaction handling.
- **Registry-Based Token Picker**: Use `createTokenRegistry(tokens)` to generate a searchable index compatible with `TokenPicker` and `SwapWidget`.
- **CSS Variable Styling**: All primitives and widgets consume `var(--cfx-*)` tokens. Override them in your root stylesheet to customize appearance without modifying component internals.
- **Error Boundaries**: The module does not implement error boundaries. Wrap widgets in your own error handling strategy or use React 18+ `ErrorBoundary` components.
