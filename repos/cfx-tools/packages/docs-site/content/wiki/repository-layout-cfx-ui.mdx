# Repository Layout — cfx-ui

# cfx-ui

**Tier 0c — React UI surface.**

This module contains packages that provide React-based UI surfaces for the `@cfxdevkit/*` stack. Packages are published as scoped npm modules and consumed by applications in `projects/` and external consumers.

UI moves fastest (weekly releases). It is coupled to `cfx-core` and `cfx-keys` through stable npm ranges, ensuring chain code is not blocked by UI iteration.

## Architecture

The `cfx-ui` repository is organized into four distinct packages, layered by abstraction level.

### Package Boundaries

| Package | Responsibility | Dependencies |
| :--- | :--- | :--- |
| **`theme`** | Design tokens, CSS variables, Tailwind preset. | None (Runtime) |
| **`react`** | Headless hooks, providers, contract interaction. | `@cfxdevkit/cdk`, `@tanstack/react-query` |
| **`wallet-connect`** | Wallet adapters, Wagmi config, SIWE. | `@cfxdevkit/cdk`, `wagmi`, `viem` |
| **`defi-react`** | DeFi widgets (swap, portfolio, pools). | `@cfxdevkit/react`, `@cfxdevkit/services` |

### Constraints

- **MAY** depend on `@cfxdevkit/cdk`, `@cfxdevkit/wallet`.
- **MUST NOT** depend on `cfx-domain`, `cfx-tools`.
- React + Tailwind allowed; no chain-primitive duplication.

## @cfxdevkit/theme

**Scope:** Shared design tokens (colors, spacing, typography, shadows, motion) usable from any UI library.

No React, no CSS-in-JS dependency. Outputs JSON + CSS variables + (optional) Tailwind preset.

### Installation

```bash
npm install @cfxdevkit/theme
```

### Usage

#### Design Tokens
Access raw tokens for non-React environments or custom styling.

```ts
import { colors, spacing, radius } from '@cfxdevkit/theme/tokens';

const buttonStyle = {
  backgroundColor: colors.primary.default,
  padding: spacing.md,
  borderRadius: radius.md,
};
```

#### React Integration
Manage theme state (light/dark/system) via context.

```tsx
import { ThemeProvider, useTheme } from '@cfxdevkit/theme';

function App() {
  return (
    <ThemeProvider defaultTheme="system" storageKey="cfx-theme">
      <Main />
    </ThemeProvider>
  );
}

function Main() {
  const { resolvedTheme } = useTheme();
  return <div>Current: {resolvedTheme}</div>;
}
```

### API Reference

| Export | Description |
| :--- | :--- |
| `colors`, `spacing`, `radius`, `typography`, `shadow`, `motion` | Raw design token objects. |
| `ThemeProvider` | React context provider for theme switching. |
| `useTheme` | Hook to access `theme` (preference) and `resolvedTheme` (effective). |
| `Theme` | Type: `'light' | 'dark' | 'system'`. |
| `ResolvedTheme` | Type: `'light' | 'dark'`. |

## @cfxdevkit/react

**Scope:** Headless React hooks and components built on top of `@cfxdevkit/cdk`.

Fully headless — no styling assumptions or UI dependencies. Built on React Query for caching, background updates, and state management.

### Installation

```bash
npm install @cfxdevkit/react @tanstack/react-query
```

### Setup

Wrap your application with `CfxProvider` to inject client, signer, and query context.

```tsx
import { CfxProvider } from '@cfxdevkit/react';
import { QueryClient } from '@tanstack/react-query';

const queryClient = new QueryClient();

<CfxProvider client={client} signer={signer} queryClient={queryClient}>
  <App />
</CfxProvider>
```

### Core Hooks

#### Account & Balance
```ts
const { address, chainId } = useAccount();
const { data: balance } = useNativeBalance({ address });
const { data: tokenBalance } = useTokenBalance({ address, tokenAddress });
```

#### Contract Interaction
```ts
// Read
const { data } = useReadContract({ abi, address, functionName, args });

// Batch Read
const { data } = useReadContracts({ contracts: [...] });

// Write
const { writeContract, isPending } = useWriteContract();
writeContract({ abi, address, functionName, args });

// Simulate
const { data: result } = useSimulateContract({ abi, address, functionName, args });
```

#### Transactions & Events
```ts
// Send Raw Tx
const { sendTransaction } = useSendTransaction();
sendTransaction({ to, value, data });

// Wait for Confirmation
const { data } = useWaitForTransaction({ hash });

// Watch Events
useWatchEvent({ abi, address, eventName, onLogs });
```

### Keystore Integration
Optional hooks for keystore management (requires `@cfxdevkit/wallet-connect` or local support).

```ts
const { wallets, addWallet, unlockWallet } = useKeystoreLifecycle();
const { data: accounts } = useKeystoreAccounts();
```

### API Reference

| Sub-path | Exports |
| :--- | :--- |
| `.` | `useAccount`, `useNativeBalance`, `useReadContract`, `CfxProvider`, etc. |
| `./account` | `useAccount` |
| `./balance` | `useNativeBalance`, `useTokenBalance`, `useTokenMetadata` |
| `./context` | `CfxProvider`, `useClient`, `useChain`, `useSigner` |
| `./contract` | `useReadContract`, `useWriteContract`, `useSimulateContract` |
| `./tx` | `useSendTransaction`, `useWaitForTransaction` |
| `./keystore` | `useKeystoreLifecycle`, `useKeystoreAccounts`, `KeystoreProvider` |

## @cfxdevkit/wallet-connect

**Scope:** Opinionated wallet-connection bundle for web apps targeting Conflux networks (Core & eSpace), built on Wagmi v2 and Viem.

### Installation

```bash
npm install @cfxdevkit/wallet-connect wagmi viem
```

### Setup

Configure Wagmi providers specifically for Conflux chains.

```tsx
import { ConfluxWagmiProviders, createConfluxWagmiConfig } from '@cfxdevkit/wallet-connect';

const config = createConfluxWagmiConfig();

<ConfluxWagmiProviders config={config}>
  <App />
</ConfluxWagmiProviders>
```

### Features

#### Chain Support
Pre-configured chains for Conflux Core and eSpace.

```ts
import { espaceMainnet, espaceTestnet, CORE_CHAIN_CONFIGS } from '@cfxdevkit/wallet-connect';
```

#### Wallet Detection
Helpers for FluentCore and standard EIP-1193 providers.

```ts
const provider = getFluentCoreProvider();
const isConnected = await detectFluentCore();
```

#### SIWE (Sign-In With Ethereum)
Message generation and verification helpers.

```ts
import { createSiweMessage, verifySiweMessage } from '@cfxdevkit/wallet-connect/siwe';

const message = createSiweMessage({ domain, uri, address });
const result = await verifySiweMessage({ message, signature });
```

#### UI Components
Basic connection UI components.

```tsx
import { ConnectButton, WalletPickerModal } from '@cfxdevkit/wallet-connect/ui';

<ConnectButton onConnect={handleConnect} onDisconnect={handleDisconnect} />
```

### API Reference

| Sub-path | Exports |
| :--- | :--- |
| `.` | `ConfluxWagmiProviders`, `useCoreWallet`, `switchEspaceChain`, `detectFluentCore` |
| `./config` | `createConfluxWagmiConfig`, `createSupportedEspaceChains` |
| `./hooks` | `useCoreWallet`, `useEspaceConnectors` |
| `./siwe` | `createSiweMessage`, `verifySiweMessage`, `generateSiweNonce` |
| `./ui` | `ConnectButton`, `WalletPickerModal` |
| `./auth` | `AuthProvider`, `useAuth` |

## @cfxdevkit/defi-react

**Scope:** React helpers for common DeFi UX patterns (token balance, swap quote, allowance, pool participation, transaction status).

Depends on `@cfxdevkit/react`, `@cfxdevkit/services`, `@cfxdevkit/cdk`.

### Installation

```bash
npm install @cfxdevkit/defi-react
```

### Usage

#### Swap Widget
A full swap interface requiring a `SwapServiceConfig`.

```tsx
import { SwapWidget, createSwapService } from '@cfxdevkit/defi-react';

const service = createSwapService({ rpcUrl, chainId, dexAdapters });

<SwapWidget 
  adapter={service} 
  tokens={tokenList} 
  onSwapSubmitted={handleTx} 
/>
```

#### Portfolio & Pools
Hooks for displaying user assets and liquidity pool data.

```ts
const { data: portfolio } = usePortfolio({ address, tokens });
const { data: pools } = usePools({ chainId });
const { data: prices } = useTokenPrice({ tokens });
```

#### Transaction Status
Track transaction lifecycle.

```tsx
import { TxStatusList, TxStatusToast } from '@cfxdevkit/defi-react/tx-status';

<TxStatusList recent={txHistory} onConfirm={handleConfirm} />
```

### API Reference

| Sub-path | Exports |
| :--- | :--- |
| `.` | `SwapWidget`, `useSwap`, `PortfolioTable`, `usePortfolio`, `TokenPicker` |
| `./swap` | `SwapWidget`, `useSwap`, `createSwappiAdapter` |
| `./balance` | `PortfolioTable`, `usePortfolio` |
| `./token-picker` | `TokenPicker` |

### Primitives
The package exports a set of UI primitives (`./primitives`) that can be used independently of DeFi logic.

```tsx
import { Button, Card, Badge, Tabs } from '@cfxdevkit/defi-react/primitives';

<Button variant="primary">Action</Button>
<Card elevated><Badge variant="success">Success</Badge></Card>
```

## Contribution Guidelines

### Adding New Packages
New packages must be added to `repos/cfx-ui/packages/`. Ensure `package.json` follows the scoped naming convention `@cfxdevkit/<name>`.

### Dependency Management
- **Internal:** Use workspace links or version ranges compatible with `cfx-core`.
- **External:** Pin peer dependencies (e.g., `react ^19`, `@tanstack/react-query ^5`).
- **Forbidden:** Do not import from `cfx-domain` or `cfx-tools`.

### Build & Publish
Packages are built using Vite (`vite.config.ts`) and managed via `moon.yml`.

```bash
# Build all packages
moon run cfx-ui:build

# Publish to npm
moon run cfx-ui:publish
```

### Testing
Ensure hooks are tested with `@testing-library/react` and mock providers (`CfxProvider`, `ConfluxWagmiProviders`).
