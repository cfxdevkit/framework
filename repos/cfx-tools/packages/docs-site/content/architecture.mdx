# Architecture

cfxdevkit follows a **four-tier layered architecture**. Dependencies flow downward only вҖ” no upward or circular imports are allowed.

## Key principles

**One-way dependency graph** вҖ” Tiers cannot create cycles. A tier-0 package must never import from tier-1 or higher.

**Domain extraction by demand** вҖ” Logic moves `projects/ вҶ’ domains/ вҶ’ framework/` once it proves reusable across вүҘ 2 projects.

**Stable public surface** вҖ” Tier-0 packages follow semver and are published to npm. Anything in `platform/devtools/` or `projects/` is internal and may change freely.

## Key execution flows

### Smart contract deployment

```
CompilerPanel (UI)
  вҶ’ useCompilerCatalog (react)
  вҶ’ deployContract (contracts)
  вҶ’ deployCoreContract (contracts)
  вҶ’ waitForReceipt (contracts)
  вҶ’ assertSuccessfulReceipt (contracts)
```

### Wallet initialization with Argon2id key derivation

```
InitializeWallet (wallet)
  вҶ’ addWallet (wallet)
  вҶ’ initFileKeystore (services)
  вҶ’ deriveKekFromEnvelope (services)
  вҶ’ deriveKeyArgon2id (services)
  вҶ’ brandKey (services)
```
