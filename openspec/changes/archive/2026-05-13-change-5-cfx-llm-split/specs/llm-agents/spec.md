## ADDED Requirements

### Requirement: Commit agent
`llm-agents` SHALL export `runCommit(args)` and `runPrecommit(args)` that orchestrate the commit pipeline: run quality gates, gather changeset context, generate a commit message via `LlmProvider`, and write a review report.

#### Scenario: Precommit runs gates without LLM call
- **WHEN** `runPrecommit(args)` is called
- **THEN** quality gates (lint, typecheck, test) are executed and results are printed; no LLM completion is performed

#### Scenario: Commit generates message via provider
- **WHEN** `runCommit(args)` is called and quality gates pass
- **THEN** the commit message is generated by calling `LlmProvider.complete()` with a structured prompt and the result is written to the review report

---

### Requirement: Docs-upkeep agent
`llm-agents` SHALL export `runDocsUpkeep(args)` that runs the docs alignment check via `@cfxdevkit/arch-check`, then delegates doc-maintenance recommendations to the LLM provider.

#### Scenario: Docs upkeep completes when LLM responds
- **WHEN** `runDocsUpkeep(args)` is called
- **THEN** the docs-alignment report is gathered and passed as context to `LlmProvider.complete()`; the recommendations are written to `artifacts/llm/reports/docs-alignment.md`

---

### Requirement: Test-upkeep agent
`llm-agents` SHALL export `runTestUpkeep(args)` that runs the test audit context gathering and delegates test coverage recommendations to the LLM provider.

#### Scenario: Test upkeep produces a report
- **WHEN** `runTestUpkeep(args)` is called
- **THEN** changed files and test coverage context are gathered and passed to `LlmProvider.complete()`; the result is written to `artifacts/llm/reports/review.md`

---

### Requirement: Review agent
`llm-agents` SHALL export `runReviewAgent(opts?)` that performs deterministic code review: identifies generated-path changes, security-sensitive changes, hotspot violations (via `@cfxdevkit/arch-check` CLI), and produces a typed `ReviewResult`.

#### Scenario: Review agent reports hotspot violations
- **WHEN** `runReviewAgent()` is called and the hotspot check returns hard violations
- **THEN** the review result SHALL include error-severity findings for each violating file

#### Scenario: Review agent runs without LLM call
- **WHEN** `runReviewAgent()` is called
- **THEN** no `LlmProvider.complete()` call is made; all analysis is deterministic

---

### Requirement: Actions registry
`llm-agents` SHALL export `listActions()` and `runAction(args)` backed by the `repoActions` registry. Each action specifies a title, default prompt, and context files.

#### Scenario: List actions prints registry entries
- **WHEN** `listActions()` is called
- **THEN** all action keys and titles from the registry are printed to stdout

#### Scenario: Run action delegates to LLM
- **WHEN** `runAction(['docs-upkeep'])` is called
- **THEN** the docs-upkeep action context is gathered and passed to `LlmProvider.complete()`

---

### Requirement: Provider injection
`llm-agents` SHALL accept an `LlmProvider` instance at the top of each agent entry point. Agents SHALL NOT directly import concrete provider classes such as `LemonadeProvider`, `OpenAICompatProvider`, or `GitHubModelsProvider`.

#### Scenario: Agents can be called with a mock provider in tests
- **WHEN** a test passes a mock `LlmProvider` to any agent entry point
- **THEN** the agent uses the mock's `complete()` method and no real HTTP requests are made

---

### Requirement: No @ts-nocheck in llm-agents
All TypeScript source files in `@cfxdevkit/llm-agents` SHALL have no `// @ts-nocheck` suppressors. All exported symbols SHALL have explicit type annotations.

#### Scenario: Typecheck passes without suppressors
- **WHEN** `tsc --noEmit` is run on `llm-agents`
- **THEN** it exits 0 with no errors
